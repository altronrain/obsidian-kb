Если какой-либо *пакет* представлен в нескольких репозиториях, то можно принудительно указать пакетному менеджеру `apt` какой репозиторий предпочтительнее. 

Особенно если речь идет о подключении репы более свежей версии дистрибутива, откуда требуется один пакет

Так на примере установки на `deb10` пакета `prosody` из репозитория `deb11` рассмотрим последовательность действий:

Создать в конфигах `apt` (`/etc/apt/apt.conf.d/`) правило (например `01debian`) , указывающее релиз дистрибутива по умолчанию (что устанавливает всем пакетам репозиториев данного релиза приоритет `990`)
Пример:
```
APT
{
Default-Release "buster";
}
```

Создать файл с указанием репозиториев для обновляемого пакета в `/etc/apt/sources.list.d/` (например `prosody.list`)
Пример:
```
deb http://deb.debian.org/debian bullseye main
```

Создать файл `/etc/apt/preferences`, в котором указать имя пакета, предпочитаемый релиз и приоритет.
Пример:
```
Package: prosody
Pin: release n=bullseye
Pin-Priority: 995
```

После чего можно не опасаясь случайного обновления всех системных пакетов на следующий релиз выполнить процедуры `apt update` и `apt upgrade` или `apt install <packagename>` (например `prosody`, который будет взят из `deb11`)

Тут главное в зависимость по библиотекам не улететь.
