## Владение файлом
Любой файл в UNIX имеет двух владельцев: *пользователя* и *группу*.
При этом пользователь-владелец может не являться членом группы-владельца, что в результате дает очень гибкую систему настройки прав доступа к файлам.

> [!NOTE] Группы пользователя
> Пользователь может быть участником нескольких групп одновременно, при этом одна будет считаться основной, а остальные дополнительными.

Владельцем-пользователем только что созданного файла является пользователь, который его создал. Владелец-группа могут назначаться по-разному:
* Linux-like: по первичной группе пользователя
* BSD-like: по группе-владельцу текущего каталога
Современные nix-системы позволяют дополнительно указывать, какой механизм следует использовать.

Для смены владельца файла используется команда `chown`.  Она позволяет сменить как владельца-пользователя, так и владельца-группу:
```
chown sergey:staff file1 file2 file3 ...
```

Для смены только владельца-группы используется команда `chgrp`:
```
chgrp staff file1 file2 ...
```

## Атрибуты файлов
С каждым файлом в Unix-подобных ОС связзано *12-битное слово*, определяющее права доступа к файлу.
### Младшие девять бит (базовые права)

#### Для обычных файлов
Предстваляют собой три группы по три бита. Группы определяют права:
* user access (u) -- для пользователя-владельца файла
* group access (g) -- для группы, являющейся владельцем файла
* other access (o) -- для всех остальных пользователей (кроме суперпользователя)

Биты, соответственно, определяют три типа прав (применительно к файлам):
* `r`, `4` -- чтение
* `w`, `2` -- запись
* `x`, `1` -- исполнение

Для группы бит совокупность прав можно указать числом, являющимся суммой отдельных бит.

Права доступа могут быть изменены владельцем файла или суперпользователем. Для изменения прав доступа используется команда `chmod`:
![[Pasted image 20260614193840.png]]
Может использоваться как буквенная нотация для добавления, усечения или установки прав доступа, либо числовая нотация.

#### Для каталогов
Также имеем три группы по три бита. Ровно с тем же разделением. Однако сами биты имеют иную интерпретацию.

Интерпретация бит прав доступа для каталогов будет:
* `r`, `4` -- просмотр содержимого (получение имен файлов, содержащихся в каталоге)
* `w`, `2` -- модификация каталога (удаление и создание файлов, в том числе чужих и тех, на которые не хватает прав доступа, так как достаточно прав на каталог)
* `x`, `1` -- использование содержимого каталога (открытие, запуск, получение информации о метаданных файлов, находящихся в каталоге, переход в каталог)

Так, комбинируя `r` и `x` можно добиться интересных эффектов, когда нет прав на просмотр содержимого, но файлы внутри использовать можно. А с правами `w` вообще следует быть осторожным, ибо оно дает право удалять любые файлы в каталоге, независимо от прав на сами файлы.

### Старшие три бита (дополнительные атрибуты)
Помимо младших девяти бит, существуют три особые старших бита: `SetUid Bit`, `SetGid Bit`, `Sticky Bit`

Данные атрибуты также устанавлюваются утилитой `chmod`, только вместо `r`, `w` и `x` необходимо указывать:
* `t` -- Sticky bit
* `s` -- Set UID, SUID
* `s` -- Set GID, SGID
* `l` -- Блокирование

Для установки дополнительных атрибутов чаще всего необходимо быть владельцем файла/каталога (помимо суперпользователя):
![[Pasted image 20260614200846.png]]

#### Для обычных файлов
Интерпретация для файлов:
* `SetUid Bit (04000)` -- исполнение файла с правами своего владельца независимо от прав доступа к файлу у запустившего файл пользователя
* `SetGid Bit (02000)` -- аналогично, исполнение от имени группы владельца файла вместо группы запустившего файл пользователя
* `Sticky Bit (01000)` -- игнорируется (ранее: сохранение образа выполняемого файла в памяти, для уменьшения времени загрузки часто используемых программ)

![[Pasted image 20260614195723.png]]

SUID и SGID обычно используются для кратковременного повышения прав с целью запуска той или иной утилиты. Следует использовать с осторожностью, так как все порождаемые процессы так же будут наследовать расширенные права. Так, например, SUID используется для запуска утилиты `passwd`.

Блокирование устраняет возможные конфликты, когда несколько программ работают с одним и тем же файлом.

#### Для каталогов
Интерпретация для каталогов:
* `SetUid Bit (04000)` -- игнорируется
* `SetGid Bit (02000)` -- при создании файлов в каталоге в качестве группы владельца будет установлена группа самого каталога (BSD-like режим наследования)
* `Sticky Bit (01000)` -- означает, что даже если пользователь имеет право записи в каталог, удалить он может только свои (принадлежащие ему) файлы (так например устроена работа `/tmp`)

![[Pasted image 20260614200409.png]]

`Sticky bit` полезен, так как права на запись для каталога дают очень много привилегий, а данный атрибут позволяет ограничить возможности пользователя.

## Скрытые специфические атрибуты файла (ext2fs)

Помимо атрибутов, принятых во всех реализациях Unix и дополнительных атрибутов, файлы могут наделяться *специфическими атрибутами*, обусловненными конкретной *реализацией файловой системы*.

Такие атрибуты предоставляют весьма широкие возможности, а также имеют *более высокий приоритет* по сравнению с обычными и дополнительными атрибутами, в случае конфликта прав.

В Linux для управления специфическими атрибутами `ext2fs` применяется команда `chattr`, которой передаются опции со знаком `+` (снять опцию) или `-` (добавить опцию)

Так, неизменяемость `i` (immutable) применяется к файлу командой:
```
chattr +i file1
```
Опция `a` разрешает открытие файла для записи только *в режиме дополнения* (append)

Специфические атрибуты `a`, `i` и `j` назначаются и снимаются только суперпользователем. Для остальных атрибутов достаточно быть владельцем файла.
Самих атрибутов довольно много. См `man chattr`.

В BSD скрытые специфические атрибуты называются *флагами* и устанавливаются командой `chflags` с указанием имени атрибута. См `man chflags` в BSD.
