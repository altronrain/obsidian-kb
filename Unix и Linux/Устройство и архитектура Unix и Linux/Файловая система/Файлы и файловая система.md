В Unix/Linux файлы играют ключевую роль.
* все данные хранятся в виде файлов
* файлы определяют привилегии пользователей в системе (посредством прав доступа к файлам)
* файлы обеспечивают доступ к периферийным устройствам
* все программы, выполняющиеся в системе, являются исполняемыи *файлами*

Файлы организованы в древовидную структуру, называемую *файловой системой*.
Каждый файл имеет *имя*, определяющее его местоположение в дереве файловой системы. *Корнем* дерева является корневой каталог `/`, а имена всех остальных файлов содержат *путь*/*маршрут* -- список каталогов, который необходимо пройти от корня, чтобы достичь файла.

Все доступное пользователям пространство объединено в единое дерево каталогов, то есть имеет один корень `/`. По этому:
* все имена файлов начинаются от корня и не содержат идентификаторов устройства (диска)
* единое дерево может быть составлено из нескольких физически различных устройств с различной внутренней архитектурой, но они объединены в единое дерево для удобства пользователя.

Исходя из вышесказанного: *имя файла* является атрибутом *файловой системы*, а не набора данных на диске, который не имеет имени. Каждый файл имеет связанные с ним *метаданные* (хранящиеся в *индексных дескрипторах* -- `inode`).
*Метаданные* уже содержат все характеристики файла, позволяющие операционной системе выполнять над файлом различные операции.

Таким образом:
* Имя файла в системе является указателем на его метаданные
* Метаданные не содержат указателя на имя файла
