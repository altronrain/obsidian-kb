В системе Unix/Linux в основном можно выделить 6 типов файлов, различающихся по функциональному назначению и действиям операционной системы при выполнении операций над файлами:
* обычный файл
* каталог
* специальный файл устройства
* именованный канал (FIFO)
* символическая связь
* сокет

## Обычный файл
Наиболее общий тип файлов. Содержит данные в некотором формате. Для операционной системы такое файлы содержат *просто набор битов*, а вся задача интерпретации содержимого файла ложится на различные прикладные программы.

## Каталог
С помощью каталогов формируется дерево файловой системы. Файл каталога содержит имена всех находящихся в нем файлов, а также указатели на дополнительную информацию (метаданные, содержащиеся в индексном дескрипторе).

Каталог определяет положение файла в файловой системе, поскольку сам файл не содержит информации о своем расположении.

Фактически *каталог* -- таблица, каждая запись в которой соответствует одному файлу.
* первое поле записи содержит номер индексного дескриптора (указатель на метаданные)
* второе поле записи определяет имя файла

## Специальный файл устройства
Данный тип файлов обеспечивает доступ к физическому устройству. Специальные файлы устройств бывают двух видов:
* символьные файлы устройств
* блочные файлы устройств

Символьные файлы устройств используются для небуферизированного обмена данными, а блочные -- для буферизированного (общение блоками определенной длины/пакетами).

## FIFO (First-in-First-Out) или именованый канал
Используется для межпроцессорного взаимодействия (появился в Unix семейства System V, но существует до сих пор). Выглядят устаревшим механизмом.

Создать именованый канал можно командой `mknod` или `mkfifo`.

Принцип работы следующий:
* создаем файл именованного канала
* направляет вывод одного процесса в файл именованного канала
* подаем содержимое именованного канала на вход другого процесса

## Символическая связь
См. также [[Связь файлов]]

Является особым типом файла, в отличие от жесткой связи (которую можно рассматривать как еще один обычный файл).

Символическая связь содерджит расположение  (путь до) файла, на который связь указывает. Она не связана с индексным дескриптором файла-оригинала. Это позволяет делать символьные связи между файлами на различных физических устройствах/дисках.

## Сокет
Предназначены для взаимодействия между процессами (в том числе между процессами на разных ПК). Появились в семействе BSD Unix, где изначально и применялись для межпроцессного взаимодействия. На сокетах работают множество сервисов и подсистем. Де факто являются современный стандартом.
