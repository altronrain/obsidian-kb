Методы управления выполнением программ

## Комбинации клавиш
Управление программой с помощью комбинаций клавиш

### Сообщение "конец файла"
Комбинация клавиш: `Ctrl-D`
В Unix-подобных ОС часто программы принимают "поток данных" на вход, до тех пор пока не возникнет *ситуация конца файла*.
Также это самый корректный способ завершить работу в терминале.

### Принудительное завершение задачи
Комбинация клавиш: `Ctrl-C`
Отправляет процессу сигнал `SIGINT`

Если данная комбинация не помогла, то можно попробовать нажать `Ctrl-\`
Если это поможет, то в текущем директории может появиться файл с именем `core` или суффиксом `.core` (в BSD системах). Он может быть довольно большим.
Отправляет процессу сигнал `SIGQUIT`

Не следует использовать комбинацию `Ctrl-Z` в попытке завершить выполнение программы

### Приостановка выполнения программы
Комбинация клавиш: `Ctrl-Z`
Не убивает активную задачу, а лишь временно приостанавливает ее выполнение

### Остановка вывода на терминал и ее отмена
Комбинация клавиш для остановки вывода на терминал: `Ctrl-S`
Может быть полезно при чтении текста, чтобы он не улетел за пределы экрана, но в современных терминалах довольно большая полоса прокрутки вверх.
Комбинация клавиш для отмены "паузы": `Ctrl-Q`

## Работа с процессами
*Процесс* -- программа, которая запущена и в настоящий момент выполняется в системе. При запуске программы появляется процесс, при завершении работы программы процесс исчезает.

Просмотр выполняющихся процессов (утилиты): `ps` и `top` (и его разновидности)

Попробовать завершить зависший процесс можно отправкой сигналов `SIGTERM` или `SIGKILL`. Стоит отметить, что `INT`, `QUIT`, `TERM` -- перехватываемые сигналы, то есть выполняемая программа может их перехватить и выполнить (или не выполнить) какие-либо действия. `KILL` не является перехватываемым сигналом и однозначно завершит процесс.

Для отправки произвольных сигналов программе применяется команда `kill`, которой нужно указать сигнал флагом и передать PID (числовой идентификатор процесса) в качестве аргумента. PID процесса можно получить из вывода утилиты `ps`
По умолчанию без флагов `kill` отправляет `SIGTERM`. Для передачи `SIGKILL` нужно указать флаг `-KILL` или `-9`
Пример:
```sh
kill -9 2763
kill -KILL 2763
```

В редких случаях процесс не исчезат даже после этого. Это возможно в двух случаях:
* процесс-зомби
* процесс в состоянии "непрерываемого сна"

### Процесс-зомби
Ситуация не страшная сама по себе. Означает что процесс на самом деле завершился, но остается в системе, потому что его предок (родительский процесс, который его запустил) не затребовал у ОС информацию об обстоятельствах завершения своего потомка (ребенка).

Процесс-зомби убить нельзя, он сам и так уже мертв. Поможет завершение работы родительского процесса. 
Ресурсов не потребляет, только занимает место в таблице процессов.
Пример процесса зомби в выводе `ps ax` (в основном ориентируемся на Z)
```sh
	3159 pts/6   Z+    0:00 [prog] <defunct>
```

### Состояние непрерываемого сна
Такое состояние процесса может возникнуть, когда он выполнил системный вызов (обратился к ОС), в ходе выполнения которого ОС перевела его в это состояние, а он в нем и остался. 
Обычно в такое состояние процессы переводятся на доли секунды. Если процесс остался в таком виде на длительное время -- с ПК скорее всего что-то не так.

Завершению процесса не поможет ничего, нельзя принудительно вывести его из этого состояния.

Пример процесса в состоянии непрерываемого сна:
```sh
	4711 pts/6   D     0:01 badblocks /dev/sdc1
```

## Запуск программ в фоновом режиме
Запуск команд в фоновом режиме выполняется символом `&`. Запуск времязатратной команды в фоновом режме не блокирует ввод других команд в текущем экземпляре терминала.
Пример:
```sh
altron@host:~$ updatedb &
[1] 2437
```
где `1` -- номер фоновой задачи, `2437` -- PID процесса

Получить список выполняющихся фоновых задач можно командой `jobs`
```sh
altron@host:~$ jobs
[1]+   Running    updatedb &
```
По завершении работы процесс сообщит об этом в терминал
Примеры успешного и не успешного запуска, и завершения сигналом `SIGTERM`:
```sh
[1]+   Done       updatedb &
[1]+   Exit 1     updatedb &
[1]+   Terminated updatedb &
```

Отправка сигналов фоновым процессам выполняется также через утилиту `kill`. В данном случае удобно ссылаться на *номер фоновой задачи*. Номер фоновой задачи указывается через символ `%`
Пример завершения второй фоновой задачи:
```sh
kill %2
```
Без указания номера сигналы будут отправлены *последней* фоновой задаче.

Чтобы отправить в фон уже запущенную программу, нужно приостановить выполнение текущей программы комбинацией `Ctrl-Z`. Затем с помощью команды `bg` можнго возобновить выполнение приостановленной задачи, но ужэе в фоновом режиме.
Обратное действие возврата команды из фонового режима в текущей (выполняющейся в терминале) выполняется командой `fg`.
