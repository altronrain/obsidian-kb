Запущенные программы общаются с окружением через потоки ввода-вывода. Каждый поток позволяет получаить извне или передавать вовне информацию (последовательность байтов).
Программа может одновременно работать с несколькими потоками ввода-вывода, различая их по номерам. Эти номера называются *дескрипторами*.
Работая с потоками, программы не делают предположением, с чем на самом деле связан тот или иной поток, это удобная абстракция, позволяющая одни и те же программы использовать и для работы с терминалом и для чтения из файлов или записи в них.

### Стандартные дескрипторы
Стандартные дескрипторы и потоки ввода-вывода в Unix-подобных ОС:
* `0` -- поток стандартного ввода
* `1` -- поток стандартного вывода
* `2` -- поток вывода сообщений об ошибках (и диагностических сообщений)
### Управление потоками ввода-вывода
Для управления потоками ввода-вывода используются символы:
```
<,>,>>,>&,|
```
Примеры операций управления потоками ввода-вывода:
```
cmd1 > file1
```
направление вывода программы `cmd1` в файл `file1`; если файл существует -- он будет перезаписан, если не существует -- будет создан.
```
cmd1 >> file1
```
направление вывода программы `cmd1` в файл `file1`; если файл существует -- вывод будет дописан в конец файла, если не существует -- будет создан.
```
cmd2 < file2
```
запуск программы `cmd2`, подав ей содержимое файла `file2` в качестве стандартного ввода (если файла не существует -- будет ошибка)
```
cmd3 > file1 < file2
```
запуск программы `cmd3` одновременно перенаправив ее ввод и вывод (ввод из файла `file2`, вывод в файл `file1`)
```
cmd1 | cmd2
```
запустить одновременно программы `cmd1` и `cmd2`, подав стандартный вывод первой на стандартный ввод второй (*конвейер*)
```
cmd4 2> errfile
```
перенаправление потока ошибок в файл
```
cmd5 2>&1 | cmd6
```
объединение потоков стандартного вывода и потока ошибок программы `cmd5` и направление их на стандартный ввод `cmd6`
