Иногда определенному запросу может соответствовать несколько страниц документации из разных разделов, например существует системный вызов `write` и команда `write`. При запросе информации командой `man write` скорее всего будет выдана справка по команде. В таком случае можно конкретизировать раздел документации, чтобы добиться вывода нужной справки.

Базовые секции системного справочника `man`:
* `1` -- пользовательские команды ОС
* `2` -- системные вызовы ядра ОС
* `3` -- библиотечные функции языка C
* `4` -- описания файлов устройств
* `5` -- описания форматов системных конфигурационных файлов
* `6` -- игровые программы
* `7` -- общие понятия, теория
* `8` -- команды системного администрирования ОС
В справочнике могут быть также и другие секции. При установке различных программ они могут добавлять справку о себе в существующие разделы или создавать новые разделы

Для вывода всех страниц справочной документации, строго соответствующих указанному значению, используется ключ `-f`:
```sh
altron@debian:~$ man -f write  
write (1)            - send a message to another user  
write (2)            - write to a file descriptor
```

Для поиска страниц по регулярному выражению можно использовать `apropos` без аргументов или его эквивалент `man -k`:
```sh
altron@debian:~$ apropos fopen  
fopen (3)            - stream open functions  
fopencookie (3)      - opening a custom stream  
pcap_dump_fopen (3pcap) - open a file to which to write packets  
pcap_fopen_offline (3pcap) - open a saved capture file for reading  
pcap_fopen_offline_with_tstamp_precision (3pcap) - open a saved capture file for reading  
```

Для получения списка всех секций `man` в системе можно использовать комбайн:
```sh
man -k . | cut -d ' ' -f2 | sort -u
```
Для получения списка всех cтраниц в секции используется утилита `apropos`:
```sh
apropos -s <имя_секции> .
```
или
```sh
apropos -s <имя_секции> -w '*'
```
Вывод идентичен и будет представлять собой список страниц и краткое описание
