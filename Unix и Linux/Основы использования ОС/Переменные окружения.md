Выполняющиеся команды в Unix-подобных ОС могут получать информацию о неких глобальных настройках из *переменных окружения*. Окружение представляет собой множество строк вида:
```
VAR=VALUE
```
где `VAR` -- имя переменной, а `VALUE` -- ее значение

Окружение свое у каждой запущенной программы (процесса). Процесс может менять своё окружение. Когда один процесс запускает другой процесс, порожденный процесс обычно наследует окружение родителя.

Примеры переменных окружения:
* `PATH` -- содержит список каталогов для поиска исполняемых файлов, если пользователь дал команду без указания каталога
* `HOME` -- путь к домашнему каталогу пользователя
* `LANG` -- определяет предпочтительный язык для многоязычных приложений
* `EDITOR` -- предпочтительный текстовый редактор 

### Работа с переменными окружения в терминале
Интерпреатор командной строки *копирует* все окружение *в свои переменные*.
Организованы они таким же образом `VAR=VALUE`.
С переменными окружения в терминале можно выполнять следующие действия:

Вызывать:
```sh
echo $PATH
```

Модифицировать командой `export`:
```sh
export PATH=$PATH:/sbin:/usr/sbin
```

Убирать из окружения командами `unset` и `export`:
```sh
unset MYVAR
export MYVAR
```

Запускать отдельные команды, модифицируя их переменные окружения:
```sh
VAR=value command
```
так, например можно указать редактор для отдельной команды:
```sh
EDITOR=nano chfn
```
