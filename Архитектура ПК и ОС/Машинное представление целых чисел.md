## Хранение чисел в ячейках
Представление целых чисел в компьютере ограничено. На хранение целого числа выделяется фиксированное количество разрядов (бит):
* 8 бит (одна ячейка)
* 16 бит (две ячейки)
* 32 бита (четыре ячейки)
* 64 бита (восемь ячеек)

## Перенос в и заем из несуществующего разряда
При использовании для пердставления целых положительных чисел стемепи счисления по основанию *N* и ограничения разрядов числом *k* максимальное число, которое можно представить составляет *N^k - 1*

Если попытаться добавить к числу *N^k - 1* еще единицу произойдет переполнение ячеек -- не найдется еще одного разряда чтобы записать туда единицу, а все текущие ячейки "сбросятся" в нулевое значение. Эторт процесс называется *переносом в нресуществующий разряд*.

*Заем из несущестующего разряда* представляет собой обратную ситуацию. При вычитании единицы из минимального состояния (все нули) получим число *N^k - 1*, так, как буд-то бы вычитание произошло из числа, имеющего единицу в несуществующем большем разряде.

## Порядок расположения чисел в нескольких ячейках
Если для записи числа используется несколько ячеек, возникает вопрос организации следования байтов числе в памяти. В данной ситуации используется два подхода:
* little-endian
* big-endian

Подход little-endian предполагает, что *первым* в памяти всегда идет *младший байт* числа и далее байты располагаются в порядке возрастания.

Подхоб big-endian полностью противоположен. *Первым* в памяти идет *старший байт* и далее байты располагаются в порядке убывания.

Пример записи 32-битного (4 байта) целого числа 7500 (в hex: 1D4C)
```little-endian
4C 1D 00 00
```
```big-endian
00 00 1D 4C
```

## Запись беззнаковых чисел в ячейках
Собственно, кратко, уже рассматривали на примерах выше. Для записи самого числа используются все биты ячеек, сколько бы их не было отведено.
Таким образом *n*-бит позволяют хранить числа от 0 до 2^*n* -1.
Так например 8-бит позволит хранить числа от 0 до 255.
## Запись знаковых чисел в ячейках
Изначальный подход основывается на заеме из несуществующего разряда. Так, число -1 будет представлено всеми единицами во всех разрядах (в двоичной системе счисления).
Однако, при использовании такого подхода необходимо определиться, когда же начинать считать числа положительными, а то на примере 8-бит чисел `00000001` может тогда означать `-255` а не `1`.

С целью исключить путаницу для записи знаковых чисел, *отрицательными* считаются комбинации, *старший бит которых равен 1*.

Тогда на примере 8-битных чисел наибольшее по модулю отрицательное число будет составлять: `10000000`, то есть `-128`. Наибольшее положительное число: `01111111`, то есть `127`.

Одно получается из другого прибавлением или вычитанием единицы. Это простейшие случаи *переполнения* (что по природе своей идентично переносу и займу в/из несуществующего разряда).

Плюсы такого подхода:
* можно легко определить знак числа по его старшему биту
* можно легко сменить знак числа, инвертировав значения во всех разрядах и прибавив 1.
Пример изменения знака (для числа 5):
```
00000101 -- 5
11111010 -- -6
11111011 -- -5
```
При этом для 0 и для максимального по модулю отрицательного данная операция приведет к самому же этому числу:
```
00000000 -> 11111111 -> 00000000
10000000 -> 01111111 -> 10000000
```
что опять же связано с переполнением.

С помощью подобного подхода можно реализовать вычитание чисел путем сложения с отрицательным слагаемым (соответственно, инвертированным). При этом от результата нужно отбросить "несуществующий разряд".
Пример на 8-бит: `75-19`
```
75 - 19
01001011 - 00010011
75 + (-19), а точнее: 75 + (256 - 19) - 256
01001011 + 11101101 = 100111000 -> 00111000
```
