## Общая шина
Основа компьютера -- общая шина. Фактически общая шина представляет собой дорожки.
К шине подключаются:
* центральный процессор (ЦП, CPU)
* оперативное запоминающее устройство (ОЗУ, RAM -- память произвольного доступа)
* контроллеры (для управления остальными устройствами компьютера)

Среди общей шины можно различить следующие типы дорожек:
* шина адресов
* шина данных
* шина управления

С точки зрения схемотехники каждая дорожка может быть в положении 1 или 0.
Так, комбинации нулей и единиц на шине адресов составляют адрес. Все устройства кроме CPU включаются в работу только тогда, когда состояние шины адресов соответствует их адресу. В остальное время они не обращают внимания на шины.

## Память
Память состоит из одинаковых ячеек памяти, каждая из которых имеет уникальный адрес, отличающий эту ячейку от других. Все адреса ячеек, возможные на данном компьютере образуют а*дресное пространство*. Его размер определяется количеством дорожек на шине адресов: 
Если дорожек N, то возможных адресов будет 2^N

Вообще такое адресное пространство называют *физическим*. Также имеются *виртуальные* адреса и виртуальное адресное пространство.

### Принцип работы ячейки памяти
С ячейкой памяти можно выполнить всего две операции:
* запись (нового значения в ячейку)
* чтение (значения из ячейки)
CPU устанавливает на *адресной шине* адрес ячейки, затем, используя *шину управления*, передает импульс, заставляя ячейку либо перетать свое содержимое, либо установить новое содержимое в соответствии с состояние *шины данных*.\
При этом старое содержимое ячейки теряется.

По шине данных информация передается параллельно, если ячейка 8-разрядная, то потребуется 8 дорожек для записи или считывания из ячейки.

CPU может работать с памятью напрямую через шину. Для обращения к иным устройствам необходимо делать это через контроллеры, выполняя программы-драйверы.

Память может быть оперативной и *постоянной*. Первая очищается при отключении электропитания, вторая -- нет. В Постоянную обычно записывается программа тестирования аппаратуры и поиска операционной системы (БСВВ)

## Дисковая подсистема
Информация на дисках хранится в виде файлов с именами, удобными для человека. Файлы существуют только на дисках, в памяти никаких файлов нет, память не оперирует такими понятиями.

## Принцип работы CPU
Центральный процуессор -- электронная схема (обычно в виде микропроцессора, т.е. в виде одной микросхемы).
Выполняет простейшие действия, заданные командами, из которых состоит программа.
В составе CPU обычно присутствуют *регистры* -- запоминающие устройства, спрособные хранить от нескольких до нескольких десятков двоичных разрядов. 
Процессор основную работу проводит над информацией, хранящейся в регистрах.

Примеры операций, поддерживаемых CPU:
* чтение и запись ячеек памяти (обязательно)
* арифметика (сложение и вычитание --минимум, деление и умножение)
* копирование информации
* логические операции
* служебные действия
Все это образует систему команд конктретного процессора.

Каждое действие, выполняемое процессором -- машинная команда.
Машинная команда обозначается кодом операции, или *машинным кодом*.

В процессоре присутсвуют электронные схемы для хранения информации, подобные ячейкам памяти; они называются *регистрами*.
Регистры бывают:
* общего назначения (для краткосрочного хранения переменных и промежуточных результатов; по окончанию рассчетов результаты заносятся в оперативную память)
* служебные (содержат информацию, необходимую самому процессору)

Программа, состоящая из машинных кодов, располагается в ячейках памяти. 
Один из служебных регистров процессора (называемый счетчиком команд, или *указатель инструкции*) содержит адрес ячейки памяти, в которой располагается слдедующая инструкция к выполнению.

Собственно, процессор работает, выполняя *цикл обработки команд*.
Пример:
* берется адрес из указателя инструкции (также назыввается счетчиком команд)
* из ячеек памяти по этому адресу считывается код команды
* увеличивается значение указателя инструкции на длину извлеченного кода (длину команды), после чего регистр будет содержать адрес *следующей за текущей* инструкции
* процессор выполняет действия, предписанные командой из памяти.
Далее цикл повторяется.

КОгда указатель инструкции содержит адрес того или иного места в памяти, говорят, что в этом месте в памяти *находится управление*

Сущестуют машинные *команды перехода* (передачи управления), которые изменяют последовательность выполнения команд, предписывая процессору перейти в другое место программы, то есть они меняют значение указателя инструкции.
Переходы бывают *условные* и *безусловные*. Для первых сначала проверяется истинность какого-либо выражения. Именно условные переходы позволяют организовать ветвления и циклы.
Также процессоры поддерживают переходы с запоминанием *точки (адреса) возврата*, что используется для вызова подпрограмм. При выполнении такого перехода адрес, находящийся в указателе инструкции (адрес *следующей команды* обычного цикла), сначала сохраняется в памяти (в аппаратном стеке), и лишь затем перезаписывается.
По завершении работы подпрограммы  она производит *возврат управления*, то есть помещает в указатель инструкции то значение, которое было запомнено при её вызове, что позволяет продолжитль выполнение с комманды, *следующей за командой перехода с вызовом*.

## Внешние устройства
Подключаются к общей шине через контроллер. Все контроллеры общаются с CPU одинаково (для процессора нет разницы между ними). Общаются такими же командами чтения и записи.
В некоторых архитектурах они даже находятся в общем адресном пространстве с памятью, однако чаще в своем обособленном адресном пространстве -- *адресах портов ввода-вывода*.
Один контролле может содержать несколько портов (отвечать на несколько адресов).
В отличие от памяти, при записи контроллер данные не запоминает, а воспринимает их как предписание к действию. При чтении передает информацию, соответственно (информацию, которая связана с состоянием самого контроллера).

CPU ничего не знает о командах контроллеровв, что какие коды означают в общении с ними. Для работы с контроллерами нужны специальные программы -- драйверы. Они, выполняясь в CPU выполняют общение с контроллером. Обычно драйверы являются частью ОС.

## Иерархия запоминающих устройств
Можно выстроить следующим образом:
* регистры CPU
* кеш
* оперативная память
* диски
* ленты
В данной градации объем памяти обратно пропорционален скорости доступа к информации и стоимости решения. Где регистры CPU будут самыми быстрыми но самыми малыми и дорогими (проектирование новых регистров очень трудозатратно и ресурсоемко).

Кеш-память хранит *копию* данных из оперативной памяти. Выигрыш по сравнению с ОЗУ в том, что для доступа к данным не нужно задействовать шину. Кеш обычно многоуровневый, в котором почти все ближние к CPU уровни выполнены с ним в одной микросхеме.

Оперативная память хранит выполняемые программы и данные для их работы.

Кроме того, все что до уровня оперативной памяти включительно является энергозависимой памятью и теряет информацию при отключении питания.

Диски позволяют долгно хранить информацию. Являются энергонезависимым устройством. Позволяют выполнять доступ к данным в произвольном порядке. Работают гораздо медленнее ОЗУ. Не могут напрямую работать с CPU, по этому программы с них сначала загружаются в память. 

Для совсем долговременного хранения (в хранилищах) применяются магнитные ленты. Самый дешевый способ хранения. Недостаток -- невозможность произвольного доступа к данным.
