См. также [[Особенности разработки в мультизадачных ОС]]
Так уже говорилось о:
* защите памяти
* разделении машинных команд на обычные и привилегированные
Однако одного этого недостаточно для реализации мультизадачного режима работы системы

При мультизадачном режиме работы в момент завершения операции ввода-вывода процессор занят выполнением другой задачи. Между тем ОС должна выполнить целый набор операций:
* перевести задачу из состояния блокировки в состояние готовности (как минимум)
\+ потенциально:
* копировать данные из буфера контроллера
* сброс контроллера
* инициирование другой операции ввода-вывода для другой задачи

ОС должна узнать о завершении операции ввода-вывода, но процессор выполняет другую задачу и не производит непрерывного опроса контроллера. Для решения этой задачи придуман аппарат *прерываний*.
* в момент завершения операции ввода-вывода *контроллер* подает CPU определенный сигнал (электрический импульс), называемый *запросом прерывания*.
* CPU прерывает выполнение активной задачи и передает управление процедуре ОС, которая выполняет все необходимые действия по окончании операции ввода-вывода. Такая процедура называется *обработчиком прерывания*.
* после завершения процедуры-обработчика управление возвращается активной задаче

Для реализации пакетного мультизадачного режима достаточно, чтобы на уровне аппаратуры были реализованы:
* прерывания
* защита памяти
* два режима работы процессора

Для реализации разделения времени или систем реального времени требуется, чтобы на уровне аппаратуры были реализованы:
* прерывания
* защита памяти
* два режима работы процессора
* *таймеры*
Планировщику ОС систем разделения времени требуется возможность отслеживать истечение квантов времени, выделенных программам. Для ОС реального времени это тоже нужно, и требования еще жестче.
*Таймер* представляет собой простое устройство, которое *генерирует прерывания* через равные промежутки времени. Эти прерывания дают возможность ОС:
* получить управление
* проанализировать текущее состояние имеющихся задач
* сменить активную задачу при необходимости

Таймер представляет собой отдельное устройство, остальные пункты -- особенности CPU.
