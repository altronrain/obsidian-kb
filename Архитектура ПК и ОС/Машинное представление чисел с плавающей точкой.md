Компьютеры умеют обрабатывать и хранить дробные числа, назваемыми числами с плавающей точкой.

## Хранение чисел с плавающей точкой в памяти
Такие числа предполагают хранение:
* знакового бита *s* (так же старший бит)
* мантиссы *M* -- двоичная дробь из интервала `1 <= M < 2`
* машинного порядка *P* -- целое число, означающее степень двойки, на которую следует умножить мантиссу.

Итоговое хранимое число считается равным ![[Pasted image 20241102183240.png]]
Целая часть мантиссы всегда равна 1, потому ее обычно не хранят, используя все разряды для хранения цифр дробной части.
Машинный порядок хранится в виде *смещенного* беззнакового целого числа: разряды, где хранится машинный порядок, рассматриваются как беззнаковое целое из которого для получения машинного порядка вычитается некоторая константа (*смещение машинного порядка*). 
Количество бит, отведенных на мантиссу и машинный порядок зависит от размера всего числа.

## Другое объяснение на примере 32-bit float
Но гораздо проще следующее объяснение хранения чисел с плавающей точкой (на примере 32-битных чисел и типа float):
![[Pasted image 20241102193313.png]]
Окно (window) сообщает, между какими двумя последовательными степенями двойки будет число
```
[0,1], [1,2], [2,4], [4,8], ..., [2^127,2^128]
```
Смешение (offset) разделяет окно на 2^23 сегментов. С помощью окна и смещения можно аппроксимировать число.
Нетрудно заметить, почти каждое следующее окно в два раза больше предыдущего, а соответственно с увеличением числа сильно падает точность представления (проблемы со смещением на целое могут возникнуть уже на числах порядка 2^24, так как интервал разбивается на 2^23 сегментов)

Пример для числа 3.14:
* число 3.14 положительно: *s* = 0
* число 3.14 находится в интервале \[2,4\], то есть окно должно начинаться с 2^1, то есть (*P*=1), в 32-bit float это P=E-127, то есть E=128 -- это и будет хранимое беззнаковое целое
* есть 2^23 смещений, 3.14 находится в (3.14 - 2)/(4 - 2) ~= 0.57 внутри интервала, что дает смещение 2^23 * 0.57 = 4781507
Итого число в памяти 3.14 компьютера в виде 32-bit float будет:
```
S = 0 = 0b
E = 128 = 10000000b
M = 4781507 = 10010001111010111000011b
```
![[Pasted image 20241102194841.png]]
То есть значение 3,14 аппроксимируется как 3,1400001049041748046875

## Операции над числами с плавающей точкой
Для выполнения операций сложения, вычитания, умножения и деления над такими числами, сначала их необходимо *привести к одному порядку*. 
* мантиссу числа, машинный порядок которого меньше, сдвигают вправо на нужное количество позиций
* производится выполняемая операция над числами
* если результат оказался меньше 1 или больше 2, то выполняется *нормализация*, то есть изменяют порядок и сдвигают мантиссу так, чтобы значение числа не изменилось, но мантисса снова стала удовлетворять условию

При сдвиге вправо младшие биты мантиссы отбрасываются. Возникает *ошибка округления* (она в целом неизбежна в операциях над дробями, взять те же периодические дроби).

*Вычисления над числами с плавающей точкой практически всегда дают не точный результат, а приблизительный*. **Сравнивать на равенство такие числа крайне не рекомендуется**.
