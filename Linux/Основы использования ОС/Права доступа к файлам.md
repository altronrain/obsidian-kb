С каждым файлом в Unix-подобных ОС связзано *12-битное слово*, определяющее права доступа к файлу.

## Младшие девять бит
Предстваляют собой три группы по три бита.
Группы определяют права для:
* владельца
* группы, которой принадлежит владелец файла
* всех остальных

Биты, соответственно (интерпретация для файлов):
* `r`, `4` -- чтение
* `w`, `2` -- запись
* `x`, `1` -- исполнение
Для группы бит совокупность прав можно указать числом, являющимся суммой отдельных бит.

Интерпретация бит прав доступ для каталогов будет:
* `r`, `4` -- просмотр содержимого
* `w`, `2` -- модификация каталога (удаление и создание файлов, в том числе чужих и тех, на которые не хватает прав доступа, так как достаточно прав на каталог)
* `x`, `1` -- использование содержимого каталога (открытие, запуск и так далее)

## Старшие три бита
Помимо младших девяти бит, существуют три особые старших бита: `SetUid Bit`, `SetGid Bit`, `Sticky Bit`
Интерпретация для файлов:
* `SetUid Bit (04000)` -- исполнение файла с правами своего владельца независимо от прав доступа к файлу у запустившего файл пользователя
* `SetGid Bit (02000)` -- аналогично, исполнение от имени группы владельца файла вместо группы запустившего файл пользователя
* `Sticky Bit (01000)` -- игнорируется

Интерпретация для каталогов:
* `SetUid Bit (04000)` -- игнорируется
* `SetGid Bit (02000)` -- при создании файлов в каталоге в качестве группы владельца будет установлена группа самого каталога
* `Sticky Bit (01000)` -- означает, что даже если пользователь имеет право записи в каталог, удалить он может только свои (принадлежащие ему) файлы (так например устроена работа `/tmp`)

Для установки прав доступа используется команда `chmod`
