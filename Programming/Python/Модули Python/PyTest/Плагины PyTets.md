Для PyTest существует большое количество плагинов, добавляющих или изменяющих его функционал, либо облегчающих работу со смежными пакетами/фреймворками.
Далее представлены популярные плагины PyTets в разных категориях.

### Изменяющие ход выполнения тестов
* `pytest-order` -- указание порядка выполнения тестов метками
* `pytyest-randomly` -- рандомайзер порядка запуска тестов
* `pytest-repeat` -- запуск тестов повторно указанное количество раз
* `pytest-rerunfailures` -- перезапуск провальных тестов указанное количество раз. Полезно в случае тестов с плавающими-ошибками (например в веб-тестировании)
* `pytest-xdist` -- запуск тестов параллельно в несколько потоков

### Изменяющие/улучшающие вывод
* `pytest-instafail` -- добавляет флаг `--instafail`. Репортит `traceback` сразу после упавшего теста а не по завершении серии тестов
* `pytest-sugar` -- отображает зеленые галки вместо точек для прошедших тестов. Добавляет прогресс-бар + как `pytest-instafail` выводит `traceback` сразу после упавшего теста
* `pytest-html` -- позволяет генерировать HTML отчеты по результатам тестирования

### Плагины для тестирования web
* `pytest-playwright` -- упрощение работы с [[Playwright]] (подготовка браузера, модификация параметров запуска)
* `pytest-django` -- упрощает тестирование Django приложений
* `pytest-flask` -- упрощает тестирование Flask приложений

### Плагины генерации фейковых данных для тестов
* `Faker` -- генерирует данные по заданным параметрам. Добавляет фикстуру `faker`.
* `model-bakery` -- генерирует Django model объекты с данными
* `pytest-factoryboy` -- включает в себя фикстуры для Factory Boy и database model data генератор
* `pytest-mimesis` -- аналогичен Faker (но вроде как побыстрее)

### Плагины для расширения функциональности
* `pytest-cov` -- функционал оценки [[Покрытие исходного кода тестами]]
* `pytest-benchmark` -- замер времени выполнения кода в тестах
* `pytest-timeout` -- возможность указать `timeout` для тестов
* `pytets-asyncio` -- тестирование асинхронных функций
* `pytest-bdd` -- плагин для написания тестов в парадигме [[BDD (behavior-driven-development)]]
* `pytest-freezegun` -- остановка времени для тестов. Если в теле тестов считывается время, то будет считываться одно и то же значение
* `pytest-mock` -- обёртка над unittest mock. Упрощает [[Мокинг (mocking)]], добавляя фикстуру `mocker`
