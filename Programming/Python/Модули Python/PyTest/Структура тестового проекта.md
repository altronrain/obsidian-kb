Любому тестовому проекту присуще наличие определенного набора конфигурационных файлов, среди которых можно выделить:
* `pytest.ini`
* `conftest.py`
* `tox.ini`, `pyproject.toml`, `setup.cfg` (могут присутствовать, могут заменять собой `pytest.ini`)

Особняком стоит отметить `__init__.py`, который можно использовать в поддиректориях, что позволит иметь идентичные наименования тестовых файлов в разных поддиректориях (так как поддиректории начинают восприниматься как отдельные пакеты).

Тесты обычно хранятся в отдельном директории `tests`.

### pytest.ini
Основной конфигурационный файл, позволяет изменить поведение PyTest по умолчанию. Директорий, в котором расположен данный файл, является *корневым директорием* тестового проекта (rootdir).

Начинается с секции `[pytest]`. далее содержит настройки в формате
```
setting = <value>
```
Если значений может быть несколько, то позволяется вводить их в одну строку, но не всегда. Например *метки обязательно регистрировать по одной на строку*. Для лучшей читабельности лучше придерживаться стиля разбиения множественных значений на отдельные строки. 
Пример часто используемых настроек:
* `addopts = ` -- позволяет перечислить флаги запуска PyTest, которые хотим использовать автоматически по умолчанию
* `testpaths = ` -- указывает директории, в которых содержатся тесты (что может сократить время запуска PyTest, минуя процедуру их поиска)
* `markers = ` -- секция, в которой регистрируются пользовательские маркеры (по одному на строку)

Хорошей практикой будет создание в любом случае файла `pytest.ini` или его аналогов-заменителей в корневом директории проекта, даже если специфически настроек не требуется и файл будет пустым. В таком случае PyTest пометит данный директорий как корневой и не будет искать его выше (вплоть до корневого диретория системы).

### conftest.py
Файл для размещения пользовательских фикстур и хуков. Обычно размещается в корневом директории проекта. Таких файлов, однако, может быть несколько, с размещением на разных уровнях (например. в поддиректориях), при этом информация из файла нижнего уровня (ближе к запускаемому тестовому сценарию) будет более приоритетной, что позволяет в специфических случаях переписывать фикстуры для определенных тестов.
На практике же часто рекомендуется не использовать более одного файла `conftest.py` с целью облегчить поиск фикстур и работу с ними. Наличие флага `--fixtures` не облегчит работу настолько, сколько понимание, что фикстуры могут быть только в корневом файле `conftest.py` или в файле с самими тестовыми сценариями.

### tox.ini, pyproject.toml и setup.cfg
Файл `tox.ini` присутствует, если в проекте используется инструмент командной строки для автоматизированного тестирования [[tox]].

Так как является ini-файлом, то все настройки PyTest записываются аналогично `pytest.ini` в секции `[pytest]`

Файлы `pyproject.toml` и `setup.cfg` используются различными инструментами Python по созданию пакетов и служат для хранения настроек.

Для `pyproject.toml` секция с настройками PyTest называется `[tool.pytest.ini_options]`
* единичные значения опций указываются в кавычках
* множественные зачения опций указываются в виде списка и в кавычках каждое

Для `setup.cfg` секция с настройками Pytest называется `[tool: pytest]`. Формат указания опций аналогичен `pytest.ini`
