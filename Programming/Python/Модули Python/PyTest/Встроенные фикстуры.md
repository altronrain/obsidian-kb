Помимо возможности создавать свои [[Фикстуры в PyTest]] фреймворк имеет ряд встроенных фикстур, для решения типовых задач. Рассмотрим некоторые из них.
### tmp_path и tmp-path_factory
Фикстуры для создания временных директорий.

Фикстура `tmp_path` имеет область применения (scope) `function` и возвращает объект/экземпляр `pathlib.Path`, указывающий на временный директорий (существует грубо-говоря на время выполнения тестового сценария)/

Фискстура `tmp_path_factory` имеет область применения (scope) `session`, возвращает объект `TempPathFactory`. У этого объекта есть метод `mktemp()`, возвращающий `pathlib.Path` объект. Можно создавать множество временных директориев.

### capsys
Служит для перехвата вывода `stdout` и `stderr`
Метод `readouterr()` возвращает кортеж, содержащий как `out` так и `err` части.
Пример перехвата стандартного вывода:
```python
def test_version_v2(capsys):
	cards.cli.version()
	output = capsys.readouterr().out().rstrip()
	assert output == cards.__version__
```
Где:
	`cards.cli.version()` -- команда, которая что-то выводит
	`capsys.readouterr().out().rstrip()` -- перехват и обработка вывода

Для вывода всей информации по тестам, в том числе и успешным, например `print` выражения, можно использовать в вызове `pytest` флаг `-s (--capture=no)`
По умолчанию выводится только текст к провалившимся тестам.

Еще один вариант всегда выводить `print` в теле теста, делать это в следующем менеджере контекста:
```python
with capsys.disabled():
	print(...)
```

Похожие встроенные фикстуры:
* `capfd` -- перехватывает файловые дескрипторы `1` и` 2`, что обычно соответствует `stdout` и `stderr`
* `capsysbinary` -- в отличие от `capsys` перехватывает байты а не текст
* `capfdbinary` -- перехватывает байты на дескрипторах `1` и `2`
* `caplog` -- перехватывает вывод, записанный пакетом `logging`

### monkeypatch
Фикстура позволяет выполнять динамическую модификацию тестируемого класса или модуля во время выполнения. Основной способ применения -- облегчить тестирование путем замены каких-то входных или выходных зависимостей тестового окружения.
Фикстура имеет область применения (scope) `function` и используется для модификации объектов, словарей, переменных окружения, текущих директорий и т.д. Мини-версия `mocking`.

Перечень методов `monkeypatch`:
* `setattr(target, name, value, raising=True)` -- установить атрибут
* `delattr(target, name, raising=True)` -- удалить атрибут
* `setitem(dic, name, value)` -- задать значение для словаря
* `delitem(dic, name, raising=True)` -- удалить значение словаря
* `setenv(name, value, pretend=None)` -- установить переменную окружения
* `delenv(name, raising=True)` -- удалить переменную окружения
* `syspath_pretend(path)` -- установить `path` как `sys.path`, который используется Python как список импорта
* `chdir(path)` -- смена текущего рабочего директория

Основной принцип, который следует запомнить при использовании `monkeypatch` и `mocking`, заменяемый объект должен заменяться на *аналогичный* объект.
Если мы подменяем вызываемую функцию/метод, то и заменять ее необходимо функцией. нельзя подставить конкретное значение в таком случае.

Пример (подмена метода в исследуемом классе):
```python
def test_patch_get_path(monkeypatch, tmp_path):
	def fake_get_path():
		return tmp_path
	
	monkeypatch.setattr(cards.cli, "get_path", fake_get_path)
	assert run_cards("config") == str(tmp_path)
```
Так как подменяем *функцию* `get_path()`, то и заменяем ее целиком на *функцию* `fake_get_path()`

Пример (подмена переменной окружения):
```python
def test_patch_env_var(monkeypatch, tmp_path):
	monkeypatch.setenv("CARDS_DB_DIR", str(tmp_path))
	assert run_cards("config") == str(tmp_path)
```
