Хуки PyTest -- это функции, позволяющие изменить поведение PyTest изнутри.
Это также некие входные точки для разработчиков [[Плагины PyTets]] по модификации его поведения.

В основном используется следующий функционал:
* `pytest_addoption` -- регистрирует новые опций запуска PyTest аналогично [[argparser]]
* `pytest_configure` -- позволяет плагинам и файлам `conftest` выполнять начальную настройку
* `pytest_collection_modifyitems()` -- вызывается после формирования списка тестов для выполнения и позволяет этот списак модифицировать
Помимо указанных хуков могут использоваться различие *плагин-специфичные хуки*.

Располагаются в файле `conftest.py` совместно с пользовательскими фикстурами.

### Добавление пользовательских опций
Для PyTest можно указать свои опции запуска. Механизм добавления идентичен `argparser`. В основном это делается с целью передать те или иные значения, которые затем могут быть использованы в фикстурах и иных местах.
Пример из реального проекта (добавление опций запуска):
```python
def pytest_addoption(parser):
	parser.addoption('--edge-ip', action='store', default=None,
		help='IP-адрес Numa Edge для подключения к WEbUI', required=False)
	parser.addoption('--edge-port', action='store', default=None,
		help='Порт Numa Edge для подключения по telnet', required=False)
	parser.addoption('--gns-ip', action='store', default=None,
		help='IP-адрес сервера GNS3', required=False)
```
Пример использования значений из пользовательских опций в фикстуре:
```python
if request.config.getoption('--edge-ip'):
	return request.config.getoption('--edge-ip')
else:
	return cfg['params']['EDGE_IP']
```

### Изменение поведения посредством `pytest_configure`
Хук `pytest_configure` вызывается после того как собраны все аргументы. Вызывается для каждого плагина и файла `conftest.py`. 
Позволяет задать конфигурацию, соответственно можно вызвать этот хук самостоятельно пользователем и указать необходимые параметры до начала тестирования.
Пример из реального проекта:
```python
@pytest.hookimpl(tryfirst=True)
def pytest_configure(config: pytest.Config):
"""
Хук для сохранения файла отчета в динамически созданный директорий

Args:
	config (pytest.Config): Объект, содержащий настройки и параметры
"""
global directory
timestamp = date.strftime(date.today(), '%Y-%m-%d')
directory = Path('results') / f'{timestamp}'
directory.mkdir(parents=True, exist_ok=True)
if config.pluginmanager.getplugin('html') and config.option.htmlpath is None:
	config.option.self_contained_html = True
	config.option.htmlpath = directory / 'report.html'
elif config.pluginmanager.getplugin('html') and config.option.htmlpath:
	config.option.self_contained_html = True
	name = config.option.htmlpath
	config.option.htmlpath = directory / f'{name}_report.html'
```
где подменяем директорий по умолчанию для HTML отчета и его имя.
### Плагин-специфичные хуки
На практике применял хуки плагина `pytest-html` для генерации HTML-отчетов, где таким образом редактируется набор полей итогового отчета.
Пример из документации `pytest-html`:
```python
def pytest_html_results_table_header(cells):
    cells.insert(2, "<th>Description</th>")
    cells.insert(1, '<th class="sortable time" data-column-type="time">Time</th>')

def pytest_html_results_table_row(report, cells):
    cells.insert(2, f"<td>{report.description}</td>")
    cells.insert(1, f'<td class="col-time">{datetime.utcnow()}</td>')

@pytest.hookimpl(hookwrapper=True)
def pytest_runtest_makereport(item, call):
    outcome = yield
    report = outcome.get_result()
    report.description = str(item.function.__doc__)
```
