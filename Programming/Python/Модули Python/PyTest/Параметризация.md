Параметризация очень полезный инструмент с целью сокращения кодовой базы. Когда на одной тестовой функции требуется проверить множество равнозначных вариантов (вспоминаем разделение на эквивалентные классы данных) входных параметров, то вместо дублирования кода пригождается параметризация тестового сценария.

К вопросу о параметризации можно подходить по разному:
1. Параметризировать тестовые функции (самый распространенный вариант)
2. Параметризировать фикстуры
3. Параметризировать хук-функцию pytest_generate_tests (экзотика)

### Параметризация тестовых функций
Для параметризации тестовых функций используется декоратор `@pytest.mark.parametrize`
Синтаксис:
```python
@pytest.mark.parametrize(
	'param1, param2, ...',
	[
		('value1', 'value2', ...),
		('valueX', 'valueY', ...),
		...,
	],
)
def test_function(..., param1, param2, ...)
```
Следует обратить внимание на компоновку и запятые.
* Перечисление параметров в первой части выражения (до квадратных скобок) выполняется целиком в кавычках. Внутри параметры перечисляются через запятую
* В квадратных скобках указываются *наборы значений* объявленных параметров. Они указываются в виде *кортежей*. Значения указываются отдельно в кавычках, разделителем выступает запятая.
Тест будет выполнен с каждым набором параметров, соответственно.

Случай для одного параметра:
```python
@pytest.mark.parametrize('param1', ['value1', 'value2', ...])
```
в таком случае нет кортежей, так как в них нет никакой необходимости

Как альтернативный вариант указания множественных параметров, можно указать несколько маркеров параметризации, по одному параметру в каждом, тогда PyTest будет *каскадно* перебирать все варианты (так количество проводимых итераций будет больше)
Пример:
```python
@pytest.mark.parametrize('param1', ['value1', 'value2', ...])
@pytest.mark.parametrize('param2', ['valueX', 'valueY', ...])
def test_function(..., param1, param2, ...)
```

Также для каждого набора значений позволяется указывать [[Маркеры (метки)]] в случае необходимости. Самый частый сценарий -- пометить один из наборов как `xfail`
Пример:
```python
@pytest.mark.parametrize(
	'param1, param2, ...',
	[
		('value1', 'value2'),
		('valueX', 'valueY'),
		pytest.param('valueA', 'valueB', marks=pytest.mark.xfail(reason="issue"))
		...,
	],
)
def test_function(..., param1, param2, ...)
```
Используется конструкция `pytest.param` для которой указывается атрибут `marks`
Аналогично вместо `xfail` можно указать и иные метки, например `skip` или `skipif`

Если требуется указать несколько меток для набора параметров, то в атрибут `marks` передается список.

Источник: https://docs.pytest.org/en/latest/how-to/skipping.html#skip-xfail-with-parametrize

### Параметризация фикстур
При выносе параметризации в фикстуру PyTest вызовет фикстуру столько раз, сколько значений передано. 
*Каждый* тест, использующий данную фикстуру, тоже будет вызван *столько же* раз.
Синтаксис:
```python
@pytest.fixture(params=['value1', 'value2', ...])
def some_fixture(request):
	return request.param

def test_something(..., some_fixture):
```
Используется `requests`. Кода относительно чуть больше.

Отличия от параметризации тестовой функции:
* В отличие от параметризации тестовой функции тут можно реализовать условное поведение в фикстуре в зависимости от переданного параметра.
* Можно одну фикстуру использовать для множества тестов (будет выполняться один и тот же набор параметров для них).
* Так можно реализовать сложные `setup` и `teardown` (например с разными подключениями к БД, разным *стартовым состоянием системы* и т.д.)

В целом можно описать разницу в подходах между двумя этими видами параметризации следующим образом:
* "Тот же тест, разные данные" -- используем параметризацию тестовой функции
* "Тот же тест, разное стартовое состояние" -- используем параметризацию фикстуры

### Параметризация через хук pytest_generate_tests
Хуки в целом часто используются плагинами PyTest, изменяя его стандартное поведение.
Но большинство из них так же можно использовать в `conftest.py` и файлах тестов напрямую.
Синтаксис:
```python
def pytest_generate_tests(metafunc):
	if some_fixture in metafunc.fixturenames:
		metafunc.parametrize('some_fixture', ['value1', 'value2', ...])

def test_something(..., some_fixture):
```
Используется объект `metafunс`, содержащий множество информации, но мы используем его здесь для параметризации по имени фикстуры.
Источник: https://docs.pytest.org/en/latest/reference/reference.html#metafunc

Функция `pytest_generate_tests` будет вызвана во время составления списка тестов к выполнению.


В такой связке может получиться крайне мощный инструмент. Примеры нетривиального использования:
* Сделать зависимыми параметры от флагов, так как `metafunc` имеет доступ к опциям через `metafunc.config.getoption('--someflag')`
* Сделать параметризацию параметров зависимой от наличия другого параметра, например если тест запрашивает оба параметра, то выдавать один набор, если один параметр, то другой набор данных
