Если код должен вызывать исключение, для его проверки следует использовать `pytest.raises()`

Пример:
```python
def test_no_path_raises_error():
	with pytest.raises(TypeError):
		cards.CardsDB()
```

Если в блоке контекст менеджера возникнет указанное исключение, то тест пройдет (его статус будет `PASSED`). Если исключения не возникнет -- не пройдет (статус будет `FAILED`).

Для `pytest.raises()` можно также указать regular expression в параметре `match=` по тексту ошибки в исключении.
Еще эффективнее менеджер контекста завернуть в переменную, таким образом получим объект `ExceptionInfo`, у которого можно проверять различные атрибуты (например с `.value`) можно сделать альтернативу `match=`

Пример:
```python
def test_no_path_raises_error():
	with pytest.raises(TypeError) as error:
		cards.CardsDB()
	assert 'Error Text' in str(error.value)
```
