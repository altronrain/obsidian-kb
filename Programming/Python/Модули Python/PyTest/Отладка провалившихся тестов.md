PyTest имеет специальные флаги (опции запуска) для упрощения отладки упавших тестов.

### Флаги для указания запуска тестов и поведения
* `--lf` `--last-failed` -- запуск только тех тестов, которые провалились во время предыдущего запуска
* `--ff` `--failed first` -- запуск всех тестов, но провалившиеся на предыдущем запуске будут первыми
* `-x` `--exitfirst` -- прекратить тесты на первом провалившемся
* `--maxfail=num` -- задает порог неуспешных тестов. Выполнение тестов останавливается при превышении порога
* `--nf` `--new-first` -- запуск всех тестов согласно времени их модификации (наиболее свежие будут выполнены в первую очередь)
* `--sw` `--stepwise` -- останов тестов на первом провале. Запуск тестов с последнего неудачного в следующий раз
* `--sw-skip` `--stepwise-skip` -- то же что и `--sw`, но пропускает неудачный тест при последующем запуске

### Флаги для контроля вывода PyTest
* `-v` `--verbose` -- подробный вывод
* `--tb=[auto/long/short/line/native/no]` -- контроль вывода traceback
* `-l` `--showlocals` -- отображать локальные переменные вместе с traceback

### Флаги для запуска отладчика
* `--pdb` -- запуск интерактивного отладчика в месте ошибки
* `--trace` -- запуск отладчика `pdb` незамедлительно на прогоне каждого теста (в интерактивном режиме)
* `--pdbcls=debugger` -- запуск альтернативного отладчика

Для `pdb` в исходном коде программы или коде тестовых сценариев можно добавлять `breakpoint()`
