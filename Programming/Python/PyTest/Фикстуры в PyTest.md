Фикстуры в pytest -- это вспомогательные функции, которые служат для выполнения некоего кода ДО и ПОСЛЕ кода теста.

Применяются для:
* подготовки тестового окружения
* передачи данных в тест
* эмуляции состояния системы
* возврата в первоначальное состояние после завершения теста

Чтобы превратить объявить обычную функцию как фикстуру применяется декоратор:
```python
@pytest.fixture()
def some_funtion(...)
```
После чего, имя функции можно передавать в тестовые функции как параметр. Для PyTest это будет указанием выполнить код вспомогательной функции до тела тестовой функции.

### Строение фикстуры
Общая максимально возможная структура фикстуры приведена ниже:
```python
@pytest.fixture()
def do_something():
	<setup_code>
	yield <object> #-- или просто yield
	<teardown_code>
```
Код функции до оператора `yield` или `return` называется `SETUP`
Код функции после оператора `yield` называется `TEARDOWN`

В минимальном варианте фикстура может вовсе не содержать `TEARDOWN` части и завершаться оператором `return`:
```python
@pytest.fixture()
def do_something():
	<setup_code>
	return <object> #-- или просто return
```

### Область применения/видимости фикстур (scope)
Для фикстуры можно указать атрибут `scope=`, который по сути определяет *частоту* применения фикстуры к тестам, которые ее вызывают.
Возможные значения атрибута:
1. `scope=function` 
	Запуск фикстуры на каждый тест (каждую тестовую функцию/метод).
	`SETUP` перед *каждым* тестом, `TEARDOWN` после *каждого* теста.
2. `scope=class`
	Запуск фикстуры разово на класс тестов, независимо от количества тестовых методов в классе. Как пример: разовая подготовка окружения для группы тестов 
	При этом вызов фикстуры все же правильнее указывать для *каждого* тестового сценария, ибо порядок их выполнения может быть рандомизирован)
3. `scope=module`
	Запуск фикстуры разово на модуль (файл), независимо от количества тестовых классов и функций в модуле
4. `scope=packege`
	Запуск разово на пакет или тестовый директорий, независимо от количества тестовых модулей
5. `scope=session`
	Запуск разово за весь текущий вызов pytest

По умолчанию, если атрибут `scope=` не задан явно, используется значение `function`

Плюсом к перечисленным штатным уровням позволяется указать в качестве значения атрибута `scope=` функцию, тем самым можно *динамически* переопределять область видимости фикстур.

Пример (динамический scope в зависимости от флага вызова `pytets`):
```python
def dynamic_db_scope(fixture_name, config):
	if config.getoption('--func-db', None):
		return 'function'
	return 'session'

@pytest.fixture(scope=dynamic_db_scope)
def db():
	...
```

### Многоуровневое использование фикстур
Помимо того, что тестовый метод (тестовый сценарий) может использовать несколько фикстур одновременно, фикстуры сами по себе могут использовать, вызывать другие фикстуры (зависеть от других фикстур).

При использовании этой возможности необходимо придерживаться правила:
Фикстуры могут опираться только на фикстуры *аналогичного* или *большего* уровня.

Примеры:
1. Фикстура `scope=funcion` может использовать фикстуры `scope=function` и выше (class, module, package, session)
2. Фикстура `scope=module` *не* может использовать фикстуры `scope=class` или `scope=funcion`, это приведет к ошибке
### Размещение и просмотр доступных фикстур
Чтобы сделать фикстуры общедоступными их следует помещать в файлы `conftest.py` (той же директории или выше).

Чтобы посмотреть все доступные фикстуры и в каком файле они находятся можно воспользоваться ключом `--fixtures`. 

Ключ `--fixtures-per-test` позволяет посмотреть все используемые тестом/тестами фикстуры и файлы, где они определены. Собственно, полезно запустить для одного теста.

Пример:
```
pytest --fixtures-per-test tests/test_auth::TestAuth::test_guest_should_see_login
```

### Автовызов фикстур
Можно использовать автовызов фикстур параметром `auto=True`
Это подходит для кода, который нужно запускать каждый раз, но при этом тесты не зависят от состояния системы или каких-либо данных от фикстуры. Используется довольно редко.

### Переименование фикстур
Для удобства можно переименовывать фикстуры параметром `name=`. Если имя было изменено с помощью данного параметра, то в тестах следует вызывать фикстуру по *новому* заданному имени.
