Проверка может потребоваться не только приложению, но и для:
* скриптов, запускаемых непосредственно из Python как `python3 some_script.py`
* импортируемых скриптов, чей код выполняется только при запуске
* приложений, для которых необходима установка зависимостей

### Тестирование скриптов
При тестировании скриптов необходимо решить две задачи:
* инициация запуска скрипта из тела тестовой функции
* сбор вывода запускаемого скрипта
Для этого можно использовать [[Модули стандартной библиотеки]] `subprocess`
Пример:
```python
from subprocess import run

def test_hello():
	result = run(["python", "hello.py"], capture_output=True, text=True)
	output = result.stdout
	assert output == "Hello, World!\n"
```
Минусы использования `subprocess.run()`:
* нет возможности проверить отдельные части большого скрипта (если только его функциональность не разбита на функции)
* сложно разнести код тестов и скрипты по разным директориям

### Тестирование импортируемых скриптов
Для решения обозначенных выше проблем скрипты можно делать импортируемыми (без ущерба их функциональности).
Для этого достаточно вынести код тела скрипта в функцию `main()`
```python
def main():
	print("Hello, World!")

if __name__ == "__main__":
	main()
```
Его все также можно будет вызвать напрямую через Python, но при этом его теперь можно импортировать. При импортировании код не будет выполнен. Он будет выполнен при вызове функции `main()`
```
$ python
>>> import hello
>>> hello.main()
Hello, World!
```
Данный скрипт можно будет импортировать и в тестовый код
```python
import hello

def test_main(capsys):
	hello.main()
	output = capsys.readouterr().out
	assert output == "Hello, World!\n"
```

> [!NOTE] Решение проблемы с разным расположением файлов
> Если скрипты и тестовые сценарии находятся в разных директориях, то необходимо в `pytest.ini` добавить директиву `pythonpath=src_dir`, где `src_dir` -- путь до файлов скриптов. В таком случае PyTest сам добавит папку со скриптами в переменную `sys.path`

### Тестирование приложения с зависимостями
В случае использования одного PyTest придется поставить зависимости в тестовое окружение из файла `requirements.txt` командой:
```
pip install -r requirements.txt
```
Если используется [[tox]], то можно указать `-rrequirements.txt` для параметра `deps` в `tox.ini`
```tox.ini
[tox]
envlist = py39, py310
skipsdist = true

[testenv]
deps = pytest
	pytest-srcpaths
	-rrequirements.txt
commands = pytest

[pytest]
addopts = -ra
...
```
Тогда при запуске он установит зависимости из файла `requirements.txt` в тестовые окружения указанных версий Python.
