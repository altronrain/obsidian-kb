PyTest является одним из наиболее популярных фреймворков для автоматизированного тестирования на языке программирования Python. Широко применяется как в тестировании web так и для иных приложений и задач.
### Именование файлов

Необходимо придерживаться следующих принципов именования файлов, функций и классов, содержащих тестовые сценарии:
1. Файлы тестовых сценариев должны называться  `test_*.py` или `*_test.py`
2. Классы, содержащие тестовые сценарии, должны называться `Test*`
3. Методы класса/функции с тестовыми сценариями должны называться `test_*`
## Возможные результаты тестов

Запуск тестового сценария может завершиться различными статусами. 
Список статусов и их отображение в полной и (сокращенной) нотациях:
1.  PASSED (.) -- тест прошел успешно
2. FAILED (F) -- тест не прошел, ошибка на этапе выполнения тестовой функции
3. SKIPPED (s) -- тест был пропущен и не выполнялся (указывается декораторами skip() skipif())
4. XFAIL (x) -- тест *не должен был* пройти и *не прошел* (указывается декоратором xfail())
5. XPASS (X) -- тест *не должен был* пройти, но *прошел* (указывается декоратором xfail())
6. ERROR (E) -- возникло исключение во время выполнения фикстур или хуков (на этапе setup или teardown), но не в тестовой функции.
