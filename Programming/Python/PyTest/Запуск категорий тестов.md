PyTest позволяет запускать определенные наборы тестов, что удобно для быстрых проверок

Примеры:
### Запуск отдельного теста
```
pytest tests/test_auth.py::TestAuth::test_guest_should_see_auth_page
```
### Запуск класса тестов
```
pytest tests/test_dashboard::TestSystemInfoWidget
```
### Запуск модуля тестов (отдельный файл)
```
pytest tests/test_interfaces_table.py
```
### Запуск отдельной функции-теста вне класса
```
pytest tests/test_card.py::test_defaults
```
### Запуск всех тестов в директории
Так можно обойти ограничение на имя директория при желании (лучше именовать tests):
```
pytest some_tests_dir
```
### Запуск тестов, удовлетворяющих паттерну (подстроке)
```
pytest -k <подстрока>
```
Вхождение ищется в именах тестов и тестовых классов.

С ключом `-k` можно применять ключевые слова `and`, `not` и `or` и их группировку. В таком случае искомое выражение помещается в кавычки:
```
pytest -k "<подстрока1> or <подстрока2>"
```
### Запуск промаркированных тестов
```
pytest -m <метка>
```
Метка заранее должна быть определена в файле `pytest.ini`

С ключом `-m` можно применять ключевые слова `and`, `not` и `or` и их группировку. В таком случае искомое выражение помещается в кавычки:
```
pytest -m "not <метка>"
```
