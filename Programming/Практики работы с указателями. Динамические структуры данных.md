Если контекст решаемой задачи предполагает начало работы с пустого списка, то есть списка, не содержащего ни одного элемента, обязательно следует превратить указатель в корректный пустой список, занеся в него "ничего" 
Например, в Pascal: `nil`

## Односвязные списки
Динамическая структура данных, строится из *звеньев*, каждое из которых, представляя собой переменную типа запись, в качестве одного из своих полей имеет *указатель на следующий элемент списка* (поле должно иметь тип "указатель на запись того же типа"). Последний элемент содержит в этом поле значение `nil`

Для обращения с таким списком достаточно в каком-нибудь указателе хранить адрес первого элемента, а далее мы можем добраться до любого элемента списка.

Преимуществом данного списка как динамической структуры данных являетсяего *нефиксированный размер*

*Односвязный* означает, что:
* на каждый из элементов списка указывает *один* указатель
* каждая запись содержит одно поле, предназначенное для сохранения связности списка и содержащая указатель
* в списке выстроена одна цепочка связей из указателей
Все эти утверждения эквивалентны

На практике обычно используют один из двух способов:
* заносят элементы всегда в начало с помощью вспомогательного указателя
* хранят два указателя (на начало и конец списка) и заносят элементы в конец

### Работа при помощи вспомогательного указателя
Решаемые типичные задачи:
* составление односвязного списка с записью элементов в начало (получаем перевернуютую последовательность)

Добавление элемента в начало списка производится в три этапа:
* используя вспомогательный указатель мы создаем *новый элемент списка*
* заполняем новый элемент значением и делаем так, чтобы его *указатель на следующий элемент стал указывать на текущий* (который пока является первым, а станет вторым) -- это адрес текущего указателя
* признание нового элемента первым элементом списка путем занесения его адреса в указатель, хранящий адрес первого элемента
![[Pasted image 20241114205322.png]]
На примере синтаксиса Pascal:
```pascal
    while not SeekEof do
    begin
        new(tmp);
        tmp^.data := {some_data};
        tmp^.next := first;
        first := tmp;
    end;
```

Для случая *пустого списка* *последовательность действий* по добавлению первого элемента *полностью совпадает* с последовательностью действий по добавелнию элемента в начало списка, уже содержащего элементы, при условии что перед началом работы указатель `first` указывает на "ничего" (`nil`).

#### Вывод элементов
При занесении чисел все время в начало, располагаться они будут в обратном порядке
Пример вывода чисел в обратном порядке *без очистки памяти*:
```pascal
tmp := first;
while tmp <> nil do
begin
	writeln(tmp^.data);
	tmp := tmp^.next;
end;
```

Чаще на практике требуется сразу же удалять элементы, высвобождая память. Выполнить это можно несколькими способами:
* Способ 1:
	* запомнить во временный указатель адрес следующего элемента
	* удалить первый элемент
	* указателю на первый элемент присвоить адрес следующего элемента
Пример способа 1 *с очисткой памяти*:
```pascal
while first <> nil do
begin
	tmp := first^.next;
	dispose(first);
	first := tmp;
end;
```

* Способ 2:
	* запомнить адрес первого элемента во вспомогательном указателе
	* исключить этот элемент из списка изменив указатель на первый элемент
	* удалить исключенный первый элемент
Пример способа 2 *с очисткой памяти*:
```pascal
while first <> nil do
begin
	tmp := first;
	first := first^.next;
	dispose(tmp);
end;
```

### Работа с хранением вспомогательного указателя
Решаемые типичные задачи:
* составление односвязного списка с записью элементов в конец (получаемпризнание нового элемента первым элементом списка путем занесения его адреса в указатель, хранящий адрес первого элемента прямую последовательность + можем обратиться/вернуться к первому элементу)

Когда список пуст, оба указателя имеют значение "ничего" `nil`
Добавление элемента в конец списка производится в три этапа:
* создаем *новый элемент списка* в конце (сразу связанный с текущим последним)
* сдвигаем указатель на последний элемент в только что добавленный элемент
* заполняем поля созданного элемента
![[Pasted image 20241114221449.png]]
На примере синтаксиса Pascal:
```pascal
new(last^.next);
last := last^.next;
last^.data := {some_data};
last^.next := nil;
```

Для случая *пустого списка* *последовательность действий* по добавлению первого элемента *является частным случаем*. Ведь `last` никуда не указывает
В таком случае последовательность действий такая:
```pascal
new(first);
last := first;
last^.data := {some_data};
last^.next := nil;
```

Итого, объединяя частный случай и общий с помощью условия получаем:
```pascal
    while not SeekEof do
    begin
        if first = nil then
        begin
	        new(first);
	        last := first
	    end
	    else
	    begin
		    new(last^.next);
		    last := last^.next
		end;
		last^.data := {some_data};
		last^.next := nil;
    end;
```
