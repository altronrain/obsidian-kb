Перечисленные объекты могут быть организованы с помощью различных структур данных (например односвязных или двусвязных списков). Их отличает *определенный набор поддерживаемых операций*, а не структура данных

## Стек и очередь
*Не являются* сами по себе *структурами данных*. Могут быть организованы с помощью совершенно различных структур данных (например односвязных списков)
*Стек* и *очередь* определяются не структурой данных, а тем, каковы доступные пользователю *операции* и как эти операции работают.

Стек и очередь представляют собой абстрактный *объект*, хранящий значения определенного типа. Для этого объекта определены *две базовые операции*:
* добавление нового значения
* извлечение значения

Разница между стеком и очередью лишь в том, в каком порядке происходит извлечение объектов:
* очередь работает по принципу FIFO (первым вошёл -- первым вышел)
* стек работает по принципу LIFO (последним вошел -- первым вышел)

Операции над стеком обычно называют:
* *push* (добавление значения в стек)
* *pop* (извлечение значения) 
Операции над очередью обычно называют:
* *put* (добавление значения в очередь)
* *get* (извлечение значения)

## Дека
Дека -- объект, который создается из двусвязного списка (от *double-ended queue*). 
Дека поддерживает 4 операции:
* добавление в начало -- *pushfront*
* добавление в конец -- *pushback*
* извлечение из начала -- *popfront*
* извлечение из конца -- *popback*

Собственно, дека может выступать как в роли *стека*, так и в роли *очереди*, в зависимости от применяемых операций, но лучше *не использовать* ее для этого.
