## Базовые сведения о рекурсии
Решаемые типичные задачи:
* сопоставление строк с шаблоном (с масками, такими как '\*' или '?')
* работа с двоичным деревом поиска (очень важно)
* работа с односвязными списками (базис рекурсии -- пустой список)
* поиск по каталогу с подкаталогами

Применимо как к процедурам так и к функциям (Pascal)

Подпрограммы (функции), при необходимости, могут вызвать сама себя. Это возможно:
* напрямую из тела самой подпрограммы
* косвенно (в цепочке различных подпрограмм)

При рассмотрении косвенного случая стоит отметить, что рекурсия может быть *взаимной*, то есть в простейшем случае одна подпрограмма вызывает другую, а вторая -- снова первую

При этом при каждом рекурсивном вызове будет создан *новый комплект локальных переменных*, так как они создаются в момент вызова подпрограммы и исчезают при возврате управления, потому с ними проблем не возникает.

При работе с рекурсивными вызовами важно не уйти в бесконечное выполнение сродни зависанию, пока не отвалимся по памяти. 
То есть рекурсия должна рано или поздно закончиться, а для этого необходимо, чтобы каждый следующий вывод решал туже самую задачу в чуть более простом варианте.

Соответственно, нужно  также определить *базис рекурсии* -- тривиальный или вырожденный случай, при котором дальнейшие рекурсивные вызовы уже не нужны.
Пример:
```pascal
procedure PrintChars(ch: char; count: integer);
begin
	if count > 0 then
	begin
		write(ch);
		PrintChars(ch, count - 1)
	end
end;
```
Здесь для печати уменьшающегося количества символов применен не цикл а рекурсивынй вызов процедуры. Каждый раз на вход вызываемой процедуры будет попадать меньшее значение count, до тех пор пока последняя процедура просто не завершится по условию, а соответственно завершатся и остальные процедуры.

Также используется *обратный ход рекурсии*, когда подпрограмма *сначала делает рекурсивный вызов, а затем какие-то действия еще*. 
Тогда *выполнение этих действий* будет идти *от последнего* вызова подпрограммы *к первому*, так как возвраты из рекурсий происходят в порядке, обратном заходу в рекурсию.
Пример с печатью цифр числа:
```pascal
procedure PrintDigitsOfNumber(n : integer);
begin
	if n > 0 then
	begin
		PrintDigitsOfNumber(n div 10);
		write(n mod 10, ' ')
	end
end;
```
В таком случае будет выведен корректный порядок, если переставить рекурсивынй вызов и печать местами, то получим обратный порядок цифр

> [!NOTE] 
> Рекурсию всегда можно заменить циклом, вопрос лишь в том. насколько это сложно. Иногда лучше отдать предпочтение им.

## Примеры рекурсии

### Работа с односвязными списками
При обработке односвязных списков можно использовать рекурсию (если хватит памяти для вызовов)

Решение задачи нахождения суммы чисел односвязного списка с использованием традиционного цикла:
```pascal
function ItemListSum(p: itemptr) : integer;
var
	sum: integer;
	tmp: itemptr;
begin
	tmp := p;
	sum := 0;
	while tmp <> nil do
	begin
		sum := su8m + tmp^.data;
		tmp := tmp^.next
	end;
	ItemListSum := sum
end;
```
А вот решение с рекурсивным вызовом функции:
```pascal
function ItemListSum(p: itemptr) : integer;
begin
	if p = nil then
		ItemListSum := 0
	else
		ItemListSum := p^.data + ItemListSum(p^.next)
end;
```

Удаление элементов из списка также легко читается:
```pascal
procedure DisposeItemList(p: itemptr);
begin
	if p = nil then
		exit;
	DisposeItemList(p^.next);
	dispose(p)
end;
```

Добавление элемента в отсортированный список чуть сложнее, отметим 3 состояния:
* список пустой -- нужно вставить элемент первым
* список непустой, первый элемент больше вставляемого -- вставка элемента первым
* список непустой, первый элемент меньше или равен вставляемому -- вставка элемента в *остаток списка*
ну а указатель на первый элемент остатка списка содержится в поле `next` первого элемента исходного списка
```pascal
procedure AddNumIntoSortedList(var p: itemptr; n: integer);
var
	tmp: itemptr;
begin
	if (p = nil) or (p^.data > n) then
	begin
		new(tmp);
		tmp^.data := n;
		tmp^.next := p;
		p := tmp;
	end
	else
		AddNumIntoSortedList(p^.next, n)
end;
```

### Работа с двоичным деревом поиска
Рекурсия *делает возможной* нормальню работу с двоичным деревом поиска

Представим дерево со следующими записями:
```pascal
type
	TreeNodePtr = ^TreeNode;
	TreeNode = record
		data: longint;
		left, right: TreeNodePtr;
	end;
```
Корневой указатель:
```pascal
var
	root: TreeNodePtr = nil;
```

Задача нахождения суммы элементов дерева заключается в нахождении:
* суммы левой половины
* суммы правой половины
* сложения сумм и прибавления к ним значения текущего узла
так как левое и правое поддеревья также являются аналогичными деревьями, то для работы с ними можно использовать те же вычисления (то есть ту же функцию)
```pascal
function SumTree(p: TreeNodePtr): longint;
begin
	if p = nil then
		SumTree := 0
	else
		SumTree := SumTree(p^.left) + p^.data + SumTree(p^.right)
end;
```
Почти вся работа с деревьями строится по той же схеме:
* базис рекурсии при работе с деревьями -- пустое дерево
* далее производится рекурсивный вызов для правого и левого поддеревьев

*Особый случай* при добавлении элементов для двоичных деревьев -- обработка числа *равного* текущему, в такихз случаях возможны следующие варианты действий:
* выдача ошибки о невозможности добавления
* заведение и увеличение счетчика, чтобы показать что вхождение было более одного раза
* не делать ничего

Поиск по дереву имеет смысл выделить в отдельную подпрограмму (которая может применяться как при добавлении, так и при поиске заданного элемента). Такая подпрограмма будет возвращать *указатель* на то место, где *находится* или *должен находиться* элемент
```pascal
type
	TreeNodePos = ^TreeNodePtr;

funtion SearchTree(var p: TreeNodePtr; val: longint) : TreeNodePos;
begin
	if (p = nil) ort (p^.data = val) then { объединяем случай пустого дерева и совпадения с корнем дерева }
		SearchTree := @p { взятие адреса вернет nil или адрес корня}
	else
	if val < p^.data then
		SearchTree := SearchTree(p^.left, val)
	else
		SearchTree := SearchTree(p^.right, val)
end;
```
