GNU debugger (`gdb`) является наиболее популярным отладчиком в Unix-подобных ОС.
Применяется при отладке компилируемых программ

## Добавление отладочной информации в программы
По умолчанию для исходного текста программы отладчик может ограничиться показом машинного кода, получившегося из программы при компиляции, но использовать такой режим практически нереально на практике.
Дело в точ что исполняемый код программы по умолчанию не включает в себя информации об именах переменных и подпрограмм, строках исходного текста и так далее.

Для того чтобы отладчик, работающий с исполняемым файлом, имел доступ к этой информации, необходимо содействие компилятора. То есть необходимо скомпилировать исходные тексты программы с добавлением *отладочной информации*
Пример для FreePascal:
```pascal
fpc -g someprog.pas
```
При этом размер итогового исполняемого файла становится значительно больше, но появляется возможность полноценно использовать отладчик на полную мощность.

## Запуск программы в отладчике и передача аргументов
Для запуска исследуемой программы в режиме отладки необходимо передать имя исполняемого файла в качестве параметра:
```shell
gdb ./someprog
```
Если требуется передача аргументов, то это можно сделать с ключом `--args`:
```shell
gdb --args ./someprog one two three
```
Отладчик можно прикреплять к уже запущенным процессам/программам, указав имя исполняемого файла и PID запущенного процесса:
```shell
gdb ./someprog 2765
```
После запуска отладчик выведет служебную информацию и приглашение на ввод *своей* командной строки:
```gdb
(dgb)
```

## Операторы управления выполнением программы
Список команд для управления ходом выполнения команды в отладчике `gdb`:
* `start` -- запуск выполнения программы в "ручном режиме"
* `run` -- запуск выполнения программы в автоматическом режиме (до ближайшей точки останова или успешного/неуспешного завершения программы)
* `next` -- шаг в одну строку текста без входа в подпрограммы
* `step` -- шаг в одну строку текста с заходом в подпрограммы
* `list [<param>]` -- просмотр окрестностей текущей строки (можно указать количество строк)
Работа с точками останова:
* `break <param>` -- установка точки останова (требует параметра)
Примеры `break`:
```gdb
break 12     break myfile.c:442     break MYFUNCTION
```
где MYFUNCTION -- имя подпрограммы (для FreePascal придется указывать заглавными буквами)
При создании точки останова отладчик покажет ее номер, что позволит в дальнейшем управлять ею следующими командами:
* `disable <br_num>` -- временное отключение точки останова с указанным номером
* `ignore <br_num> <count>` -- игнорирование точки останова указанное число раз
* `cond <br_num> <condition>` -- указание условия останова в виде логического выражения. Точка останова будет учтена при выполнении условия
* `info breakpoints` -- вывод информации о точках останова в программе и условиях для них
Прервать ход выполнения программы в отадчике можно также комбинацией `Ctrl-C`

Работа с переменными:
* `inspect` -- просмотр текущего значения переменных
* `set var <var>=<val>` -- принудительное изменение значения переменной
Работа с подпрограммами:
* `bt` или `backtrace` -- отображение вызванных (но еще не завершенных) подпрограмм, параметры их вызова и места вызова. Вначале указывается самый последний вызов, также вызовы пронумерованы *номерами фреймов*
* `frame <num>` -- просмотр места вызова подпрограммы
Так `frame 1` позволит посмотреть место вызова подпрограммы, указанной с номером `#0`. После команды `frame` можно воспользоваться `list` и `inspect`
* `call` -- позволяет вызвать любую подпрограмму с указанными аргументами, но есть ограничения

Выход из отладчика производится командой `quit`, или отправкой `Ctrl-D`
Также имеется справка, вызываемая командой `help`
