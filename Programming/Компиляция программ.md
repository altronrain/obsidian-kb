Материал по большей части относится к кормпилируемым ЯП
Процесс компиляции зависит от ЯП и используемого компилятора, но в общем случае процесс построения исполняемого файла программы из кода можно разбить на два этапа. Представлено краткое описание процесса.

## Первый этап: построение объектных модулей
Объектный модуль еще не является исполняемым файлом. Он содержит объектный код, которому для превращения в машинный код требуется еще одна стадия обработки.
Объектные файлы формирутся и для каждого используемого модуля или библиотеки отдельно.
Объектный файл (модуль) пока все еще содержит метк вместо адресов и точек подключения библиотек и других модулей. Их необходимо собрать все вместе.

Пример создания объектного модуля программы на NASM:
```shell
nasm -f elf hello5.asm
```
где указывается, что на выходе ожидаем объектный файл в формате ELF
## Второй этап: компоновка объектных файлов
Для получения исполняемого файла необходимо соединить вместе все объектные файлы модулей, подключить к ним библиотеки, расставить все ссылки (адреса), тем самым превратив объектный код в машинный. Это задача *компоновщика* (редактора связей, линкера)

Отсутствие сторонних модулей и библиотек не исключает стадии компоновки программ

Пример создания 32-битного исполняемого файла из объектного файла:
```shell
ld -m elf_i386 hello5.o -o hello5
```
где ключ:
* `-m` -- указывает архитектуру
* `-o`  -- указывает имя исполняемого файла (от output)
