Выполняются над целыми числами *одинакового типа*, однако числа при этом рассматриваются не как числа, а как *строки отдельных битов*, составляющих их машинное представление.

Можно разделить на два вида:
* логические операции над отдельными битами (*всеми одновременно*)
* сдвиги

## Логические операции
Логические операции  -- все те же `not`, `and`, `or`, `xor`.
Так `not`, примененный к числу типа `integer` даст число с инвертированными битами:
```pascal
x := 75;
y := not x; -> -76
```
а если бы это был `word`, т.е. беззнаковое целое, то результат был бы
```pascal
x := 75;
y := not x; -> 65460
```
что в двоичной записи такая же 16-ти битная последовательность

Результат логических операций будет того же типа, что и операнды.

## Сдвиги
Операции побитового сдвига *сдвигают* битовое представление на определенное число возиций влево `shl` или вправо `shr`.
Обе операции требуют двух операндов (исходного числа и количества позиций для сдвига), ключевое слово при этом ставится посередине.

При рассмотрении операций считаем запись числа в двоичном виде справа налево:
```
1  -> 00000001
21 -> 00010101
```

При сдвиге влево на `k` позиций, *старшие* `k` разрядов *пропадают*, а *справа* (младшие разряды) дописываются *нули*.
Для *беззнаковых целых* сдвиг влево эквивалентен домножению числа на $2^k$
Пример
```pascal
1 shl 5 -> 32
00000001 -> 00100000

21 shl 3 -> 168
00010101 -> 10101000
```

При сдвиге вправо на `k` позиций, *младшие* `k` разрядов *пропадают*, а *слева* (старшие разряды) дописываются *нули*.
Для *беззнаковых целых* и *положительных знаковых* сдвиг вправо эквивалентен делению на $2^k$ с отбрасыванием остатка
Для знаковых отрицательных `shr` не даст такого результата (как минимум потому что число станет положительным). Необходимо использовать встроенные функции `SarShortint`, `SarSmallint`, `SarLongint`, `SarInt64`
Пример
```pascal
133 shr 5 -> 4
10000101 -> 00000100

64 shr 3 -> 8
01000000 -> 00001000
```
