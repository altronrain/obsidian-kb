## Базовые сведения о рекурсии
Применимо как к процедурам так и к функциям

[[Подпрограммы]], при необходимости, могут вызвать сама себя. Это возможно:
* напрямую из тела самой подпрограммы
* косвенно (в цепочке различных подпрограмм)

При этом при каждом рекурсивном вызове будет создан *новый комплект локальных переменных*, так как они создаются в момент вызова подпрограммы и исчезают при возврате управления, потому с ними проблем не возникает.

При работе с рекурсивными вызовами важно не уйти в бесконечное выполнение сродни зависанию, пока не отвалимся по памяти. 
То есть рекурсия должна рано или поздно закончиться, а для этого необходимо, чтобы каждый следующий вывод решал туже самую задачу в чуть более простом варианте.

Соответственно, нужно  также определить *базис рекурсии* -- тривиальный или вырожденный случай, при котором дальнейшие рекурсивные вызовы уже не нужны.
Пример:
```pascal
procedure PrintChars(ch: char; count: integer);
begin
	if count > 0 then
	begin
		write(ch);
		PrintChars(ch, count - 1)
	end
end;
```
Здесь для печати уменьшающегося количества символов применен не цикл а рекурсивынй вызов процедуры. Каждый раз на вход вызываемой процедуры будет попадать меньшее значение count, до тех пор пока последняя процедура просто не завершится по условию, а соответственно завершатся и остальные процедуры.

Также используется *обратный ход рекурсии*, когда подпрограмма *сначала делает рекурсивный вызов, а затем какие-то действия еще*. 
Тогда *выполнение этих действий* будет идти *от последнего* вызова подпрограммы *к первому*, так как возвраты из рекурсий происходят в порядке, обратном заходу в рекурсию.
Пример с печатью цифр числа:
```pascal
procedure PrintDigitsOfNumber(n : integer);
begin
	if n > 0 then
	begin
		PrintDigitsOfNumber(n div 10);
		write(n mod 10, ' ')
	end
end;
```
В таком случае будет выведен корректный порядок, если переставить рекурсивынй вызов и печать местами, то получим обратный порядок цифр

## Взаимная рекурсия
Рекурсия может быть *взаимной*, то есть в простейшем случае одна подпрограмма вызывает другую, а вторая -- снова первую

Для реализации подобного в Pascal необходимо прибегать к *предварительному объявлению (декларации)* подпрограмм, иначе компилятор выдаст ошибку, так как на время обработки одной подпрограммы он встретит имя/вызов другой подпрограммы, о которой он еще ничего не знает. Потому необходимо *заранее объявить подпрограмму

При декларации подпрограммы указываются:
* имя подпрограммы
* параметры вызова
* для функции -- тип возвращаемого значения
* ключевое слово `forward` c точкой с запятой

Пример декларации процедуры и функции:
```pascal
procedure TraverseTree(var p: NodePtr); forward;
function CountValues(p: NodePtr; lim: longint): integer; forward;
```

Таким образом порядок действий при взаимной рекурсии такой:
* декларируется первая из подпрограмм
* описывается (полностью, с телом) вторая из подпрограмм
* описывается (полностью, с  телом) первая из подпрограмм

Если задекларированная подпрограммма не будет описана компилятор выдаст ошибку
