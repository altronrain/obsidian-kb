## Управление обработкой ошибок
Указание компилятору, что мы будем сами обрабатывать ошибки, выполняется последовательностью `{$I-}`, (включается, соответственно `{$I+}`).
По умолчанию включено.

Пример использования совместно со встроенной функцией `IOResult`, возвращающей код очередной операции ввода:
```pascal
{$I-}
read(x, y);
if IOResult = 0 then
	writeln(x * y)
else
	writeln('I couldn''t pasrse your input')
```

## Изменение операции взятия адреса
По умолчанию результатом операции взятия адреса (см. [[Programming/Pascal/Типы данных/Использование указателей/Адреса и указатели#Операция взятия адреса]]) является *нетипизированный адрес*.
Поведение можно отключить директивой `{$T+}`. Соответственно:
* `{$T-}` -- по умолчанию, соответстует нетипизированному адресу
* `{$T+}` -- соответствует типизированному адресу

## Изменение режима вычислений условий
По умолчанию FreePascal использует "ленивые вычисления", то есть в случае, если заведомо известно, что выражение примет то или иное булево значение, то оставшаяся часть условия не вычисляется. Что позволяет избежать исключений в некоторых случаях.
Поведение регулируется директивой `{$B-}`. Соответственно:
* `{$B-}` -- по умолчанию, соответствует "ленивым вычислениям"
* `{$B+}` -- соответсвует полному вычислению всех элементов логического выражения

## Условная компиляция
Какие-то фрагменты программы можно обрабатывать и компилировать по условию. 
Для этого необходимо директивой `{$DEFINE}` объявить какое-нибудь слово, и далее директивами `{$IFDEF}` и `{$ENDIF}` оборачивать фрагменты кода, которые будут выполняться по условию (только тогда, когда указана директива `{$DEFINE}`)
Пример:
```pascal
{$DEFINE DEBUG}
{...}
{$IFDEF DEBUG}
writeln('DEBUG: x=', x, ' y = ', y);
{$ENDIF}
```
