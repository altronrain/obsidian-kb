Любые типы, вводимые программистом (не являющиеся встроенными), называются пользовательскими

Для указания пользовательских типов используется секция `type`, которую можно расположить как в глобальной области, так и локально в подпрограммах.

Рассмотрение массивов и записей см. в [[Массивы]]

## Типы синонимы
Самый простой вариант применения пользовательских типов, создать тип, являющийся синонимом имеющегося:
```pascal
type
	MyNumber = real;
```
После можно использовать `MyNumber` как тип переменных
```pascal
var
	x, y, z: MyNumber;
```

Почему это может быть полезно:
Появляется *единая точка для внесения изменений*. При начале разработки программы мы можем и не знать, хватит ли нам того или иного типа (размерности). Подход с типом-синонимом существенно облегчает правки при переходе на тип большей или меньшей размерности

## Диапазоны и перечислимые типы
### Диапазоны
Простейший пример действительно новых типов -- *диапазоны*. 
Для диапазонов четко задаются границы литералами. Переменная такого типа может принимать целочисленные значения из заданного диапазона.

В остальном с такими переменными можно работать также как и с другими значениями подобного типа, из которого выделен диапазон (как целочисленные числа или символы).
```pascal
type
	digit10 = 0..9 { цифры от 0 до 9 }
	LatinCaps = 'A'..'Z' { заглавные латинские буквы }
```
Стоит отметить, что для *хранения* переменных типа диапазона *используется* такой же *размер*, как у *наименьшей* переменной *встроенного типа*, способной *вместить все значения диапазона*.
```pascal
digit10 = 0..9 -> 1 байт
digit 15000..15010 -> 2 байта
```
Так получается потому, что перерасход памяти по потерям менее накладный чем преобразования (лишние операции) на процессоре при работе с числами.

В качестве элементов диапазона могут выступать только константы времени компиляции (то есть значение которых определяет компилятор до преобразования текста программы).
### Перечисляемые типы
Выражение такого типа может принимать *одно из значений, перечисленных явным образом* при описании типа.
```pascal
type
	TrafficLights = (red, yellow, green);
var
	light: TrafficLights;
```
Такой переменной по тексту программы может быть присвоено значение из перечисленных при объявлении типа.

Имена константа в перечислениях не могут пересекаться между собой, должны быть уникальными, иначе компилятор не поймет, к какому перечислению она относится. По этому их часто снабжают префиксами:
```pascal
type
	Signals = (SigRed, SigYellow, SigGreen);
	Colors = (ColRed, ColOrange, ColYellow, ColGreen, ColBlue);
```

Для значения перечислимого типа определены операции сравнения. Также можно узнать предыдущее и следующее значение функциями `pred` и `succ`, правда их *небезопасно* применять к уже пограничным значениям. С помощью функции `ord` можно узнать порядковый номер элемента (нумерация с 0).

## Порядковый тип
Обобщая информацию, под *порядковым типом* в Паскале понимается тип, удовлетворяющий следующим условиям:
* всем возможным значениям этого типа сопоставлены целочисленные *номера*
* над значениями типа определны операции сравнения, где элемент с *меньшим номером считается меньшим*
* для типа имеется возможность указать наименьшее и наибольшее значения

Для определения порядковых номеров текущего, предыдущего и следующего элементов с такими типами и используются встроенные функции `ord`, `pred`, `succ`

Примеры порядковых типов в Pascal:
* `boolean`, где `false` имеет номер 0, а `true` -- номер 1
* `char`, номера элементов соответствуют кодам символов (от 0 до 255)
* целочисленные типы знаковые и беззнаковые (кроме `int64` и `qword`)
* любой диапазонный тип
* любой перечислимый тип
