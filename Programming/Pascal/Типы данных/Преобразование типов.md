Преобразование типов имеет место, когда представление числа в памяти компьютера меняется. Преобразование типов может быть:
* неявным
* явным

## Неявное преобразование типов
Имеет место быть когда *компилятор сам*, без прямых указаний, преобразует тип выражения.
В Pascal набор таких преобразований довольно ограничен:
* преобразования целочисленных типов друг в друга
* преобразование целочисленных типов в числа с плавающей точкой
* преобразование чисел с плавающей точкой друг в друга
* преобразование символа в строку

Пример преобразования целочисленного типа в число с плавающей точкой:
```pascal
var
	r: real;
begin
	{...}
	r := 15; -> 15.0
```
Помимо присваивания, такое преобразование возможно и при вызове подпрограммы (ожидающей переменную типа `real`)

Также любая операция деления вернет число с плавающей точкой, независимо от типов операндов:
```pascal
10 / 5 -> 2.0
```

> [!NOTE] Ограничения неявного преобразования
> Неявное преобразование возможно только над значениями выражений, но не переменными. По этому в подпрограммы, ожидающие *параметр-переменную* необходимо *передавать именно* тот конкретный *тип, что указан в подпрограмме*

## Операторы преобразования дробного в целое
Если компилятор неявно сам преобразует целое в дробное, то обратную операцию он выполнять не будет. Дело в том что существует несколько методов выполнения такой операции:
* `trunc` -- преобразование с отбрасыванием дробной части
* `round` -- преобразование с округлением

Собственно, функции `trunk` и `round` выполняют вычисления а не преобразование как таковое.
Пример:
```pascal
r := 15.75;
i := trunk(r); -> 15
j := round(r); -> 16
```

## Явное преобразование типов
Позволяет преобразование как значений так и переменных.
Для значений возможны преобразования:
* порядковых типов друг в друга (целых числе, `char`, `boolean`, перечислимых типов)
* типов, имеющих одинаковый размер машинного представления
Для переменных обязательно требуется совпадение размеров, и п.1 не рабоатет.

При явном преобразовании синтаксис напоминает вызов функции, но вместо имени функции указывается имя типа
Пример:
```pascal
integer('5') -> 53 {но двухбайтное знаковое}
byte('5') -> 53 {но однобайтное беззнаковое}
```

Без явного преобразования типов можно обойтись всегда и лучше обходиться без него
