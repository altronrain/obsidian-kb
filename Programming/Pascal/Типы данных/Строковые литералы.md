Представляют собой типы:
* `string` -- для строк из множества символов
* `char` --для одиночных символов

Над строковыми литералами допустимы следующие операции:
* сложение `+` -- а точнее конкатенация (соединение нескольких строк в одну)
* строки также можно *сравнивать*, но результатом будет *булево значение*

## Строка как массив char
Работу со строками можно представить и в виде массива `char`, при этом допускается выполнять присваивание не только того же типа в такой массив, но и строкового литерала:
```pascal
var
	hello: array [1..30] of char;
begin
	hello := 'Hello, world!';
end.
```
При этом программа при печати выведет еще 17 символов `\0`, что не есть удобное и корректное поведение для текста. Можно было бы решить пробелму обработкой посимвольно в цикле, но это неудобно. 

Необходимо иметь *более гибкое средство* для работы со строками. Фундаментальное *свойство строки* -- *иметь непредсказуемую длину*. К тому же над строками очень часто выполняется операция конкатенации.

## string
Частный случай массива из элементов типа `char`, введенный для удобства.
При объявлении можно указать предельный размер строки (а можно и не указывать). Максимальная длина при этом ограничена 255 символами:
```pascal
s1: string[15]; { строка в 15 символов }
s2: string; { строка в 255 символов }
```
Попытка указать число больше 255 вызовет ошибку.
Тип `string` хранит длину строки в *отдельном* байте, по этому максимальная длина строки -- 255, а сам `string` занимает на 1 байт больше чем предельная длина хранимой строки.

С переменной типа `string` можно работать как с массивом `char`, индексация самих элементов при этом идет с единицы.
В элементе с индексом 0 содержится длина массива, но при этом элемент тоже имеет тип `char`, так что там фактически содержится *символ с кодом соответствующим длине строки*.
Можно получить длину строки из нулевого элемента как:
```pascal
ord(s1[0])
```
Но лучше использовать встроенную функцию `length`

Иные особенности `string`:
* можно присваивать строки любых размеров друг в друга (строка может быть обрезана по длине в процессе)
* при *передаче по значению* в подпрограмму передается *только значащая часть* переменной, а не вся длина (строка из 3 символов передаст 4 байта)
* при передаче как параметр-переменная не действует ограничение на совпадение типов, можно передавать `string` любой длины в подпрограмму (строка также может быть обрезана по длине)

Строки могут быть пустыми (нулевой длины), обозначаются как `''`
