*Динамические переменные* в отличие от обычных создаются по время выполнения программы, а не описываются в тексте, как обычные переменные.
Для работы с такими переменными необходимо освоить *адреса* и *указатели*

## Теоретическая основа указателей
Повторение ранее изученных тезисов:
Память компьютера состоит из *одинаковых ячеек*, каждая из которых имеет свой *адрес*. Адрес *позволяет однозначно идентифицировать* ячейку памяти.

Переменные часто занимают не одну ячейку, а *несколько идущих подряд* -- *область памяти* (частный случай это `char` и `byte`, занимающие ровно 1 ячейку). *Адресом области памяти* считается *адрес первой ячейки* этой области.

Адреса можно хранить в памяти, то есть одна область памяти содержит (хранит) адрес другой области памяти. Многие ЯП предусматривают специальный тип или семейство типов выражений, которые соответствуют адресам. Переменная такого типа, то есть переменная, в которой хранится адрес (другой переменной) называется *указателем*.

Таким образом:
* указатель -- переменная, в которой хранится адрес
* утверждение вида "A указывает на B" означает "A *содержит адрес* B"

## Указатели в Pascal
### Типизированные указатели
В большинстве случаев в Pascal применяются *типизированные указатели*. Это переменные, *хранящие адрес переменной определенного типа*.
Для обозначения указателя используется символ `^`
Пример:
```pascal
var
	p: ^integer;
	q: ^real;
```
где `p` может хранить адрес `integer`-переменных, а `q` -- хранить адрес `real`-переменных.

### Нетипизированные указатели
В Pascal также предусмотрен отдельный вид указателей, называющийся *нетипизированными указателями* (и адресами соответственно). Этот тип называется `pointer`.
Адреса этого типа компилятор рассматривает просто как *абстрактные адреса* ячеек памяти, *не делая предположений* о том какого *типа значения* находятся по указанному адресу.
*Указатели* данного типа *могут хранить произвольный адрес* в памяти, *независимо* от того *что по этому адресу находится*.
Пример:
```pascal
var
	ap: pointer;
```
Более того, значение такой переменной-указателя можно присвоить любой переменной типизированного-указателя, что может быть *чревато ошибками*. 
Например, если забыть что нетипизированный указатель указывает на `string` и присвоить значение переменной `pointer` типизированному указателю типа `integer`).

### Нулевой указатель или недействительный адрес
Частным случаем нетипизированного адресного выражения является встроенная константа `nil`, обозначающая *недействительный адрес*. Такой, по которому в памяти заведомо не может находиться ни одна переменная.
Эта константа присваивается переменным указательных типов чтобы показать, что *данный указатель сейчас никуда не указывает*.
Константу `nil` также иногда называют *нулевым указателем*, хотя фактически это константа (а указатель это переменная)

### Операция взятия адреса
Адрес переменной можно получить через *операцию взятия адреса* (обозначается знаком `@`)
Пример:
```pascal
var
	r: real;
	p: ^real;
begin
	{...}
	p := @r;
```
Операцию взятия адреса можно применить *к любой переменной*, а не только к имеющим имя-идентификатор (например к элементу массива или полю записи)

Результатом операции взятия адреса по умолчанию является *нетипизированный адрес* (в Си по другому). Это поведение можно исправить директивой `{$T+}` -- тогда будет адрес типа переменной.

### Разыменование
*Разыменование* (от английского dereference) на самом деле более подходящий перевод -- *обращение по адресу*. Обозначается символом `^` *после имени указателя* или произвольного *адресного выражения* (возвращающего адрес).
Пример:
```pascal
var
	r: real;
	p: ^real;
begin
	{...}
	p := @r;
	p^ := 25.7;
```
где указанное выражение будет означать "то, на что указывает `p`", то есть на переменную `r`
Таким образом, *в память по адресу*, хранящемуся в `p` (то есть *в переменную* `r`) занесли значение `25.7`
