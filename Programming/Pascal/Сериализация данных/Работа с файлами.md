При открытии файла, программе наряду со штатными потоками ввода/вывода становятся доступен поток для работы с файлом. Операции с таким потоком в целом похожи на работу с обычными потоками (на уровне ОС вообще нет различий).

Этот поток нужно как-то отличать от стандартных потоков. Для этого в Pascal предусмотрены *файловые перменные*, для описания которых используются *файловые типы*.

Передача файловых переменных в подпрограммы возможна только через var-параметры.

## Файловые типы
Типы файловых переменных (файловые типы):
* `text` -- работа с файлом, предполагая что он текстовый
* `file` -- работа с файлом как абстрактной последовательностью байтов, имея возможность записать и прочитать любой его фрагмент операциями *блочного чтения* и *блочной записи*
* `file of <type>` -- работа с файлом, состоящим из блоков информации фиксированной длины, соответствующих машинному представению в памяти какого-то типа. Например `file of real` или чаще на практике `file of myrecord`, где `myrecord` описанный заранее тип-запись

## Привязка имени файла к файловому типу
В блоке переменных мы объявляем файловую переменную указанного файлового типа, но она сама по себе никак не связана с существующими или пока еще не существующими файлами на диске
Для выполнения привязки используется процедура `assign`, принимающая на вход имя файловой переменной и имя файла (относительное или абсолютное, существующее или еще не существующее)
Пример:
```pascal
var
	file1: text;
begin
	{...}
	assign(file1, 'data.txt');
```

## Открытие  и закрытие файлов
Открытие файла выполняется после привязки, и может быть выполнено различными способами, в зависимости от решаемой задачи:
* `reset` -- используется для чтения из файла (файл должен существовать)
* `rewrite` -- перезапись имеющегося файла с нуля или создание нового файла
* `append` -- открытие файла для *записи на добавление*. Возможно *только для текстовых файлов*
Во всех случаях в качестве аргумента процедуре передается имя файловой-переменной

Закрытие файла выполняется процедурой `close` с передачей имени файловой перменной (после чего можно вновь открыть файл с тем же именем, передав ту же файловую переменную, или переназначить файловую переменную на другое имя файла)

## Процедуры работы с файлами
Для обработки содержимого файлов могут использоваться различные процедуры.
* `read` и `write` -- чтение и запись для *текстовых* и *типизированных* файлов
* `readln` и `writeln` -- чтение и запись только для текстовых файлов
* `eof` -- выполнение действия до наступления ситуации конца файла для *текстовых* и *типизированных* файлов
* `SeekEof` -- выполнение действия до наступления ситуации конца файла *только* для *текстовых* файлов
* `BlockRead` и `BlockWrite` -- чтение и запись для блочных файлов

Отличие от обычного вызова процедур будет составлять указание файловой-переменной в качестве первого аргумента

Пример процесса создания текстового файла:
```pascal
const
	message = 'Hello, world!';
	filename = 'hello.txt';
var
	f: text;
begin
	assign(f, filename);
	rewrite(f);
	writeln(f, message);
	close(f)
end.
```

## Изменение режима работы с файлами через filemod
По умолчанию процедуры, открывающие *типизированный или блочный* файл, открывают его в режиме "чтение и запись" (как `reset` так и `rewrite`, просто один открывает существующие, другой -- создает новый или перезатирает). Если прав на запись нет, то возникнет ошибка. 
Обойти это ограничение можно с помощью глобальной перменной `filemode`, указав в нее новое значение (`0`), где:
* `0` -- только чтение
* `1` -- только запись
* `2` -- запись и чтение

## Особенности текстовых файлов
При записи в текстовый файл иных типов данных, например чисел, они будут переводиться в свое строковое представление, а операция считывания -- обратное преобразование.
Текстовые файлы не допускаю принудительного изменения текущей позиции и не предполагают попеременных операций чтения и записи.

Для типизированного файла `file of char` в принципе позволит открыть любой файл в качестве типизированного, так как все файлы состоят из байтов

## Особенности типизированных файлов
В отличие от текстовых файлов для типизированных поддерживается перемещение по файлу, потому что все записи имеют известный фиксированный размер. Для этого используется процедура `seek`, которой передается файловая переменная и *позиция* (нумерация с нуля)

## Особенности блочных файлов
При открытии файлов через `reset` и `rewrite` появляется второй параметр -- размер блока, и по умолчанию он равен 128 байтам, но можно задать свой

Процедуры `BlockRead` и `BlockWrite` принимают 4 параметра:
* 1 -- файловая переменная
* 2 -- переменная произвольного типа и размера, в которую будет помещена считанная информация или из которой будет взята информация для записи (в зафисимости от процедуры)
* 3 -- целое число, задающее *количество блоков* (перемножение на размер блока не должно превосходить размер переменной из п.2)
* 4 -- переменная типа `longint`, `int64`, `word` или`integer`, куда ззаписывается *количество блоков*, которое реально удалось прочитать или записать

По поводу п.4 результат может оказаться мегьше ожидаемого, например если при чтении наступил EOF. Собственно для `BlockRead` значение 4 переменной `0` говорит о EOF.

## Операции над файлами как целым
Помимо работы *с содержимым файлов*, над *самими файлами* файлами в Pascal позволдяется выполнение ряда операций, таких как:
* удаление (процедурой `erase`)
* переименование (процедурой `rename`)

Процедура `erase` принимает один параметр -- файловую переменную. Подразумевается что файл должен быть назначен в переменную, но не открыт
Пример:
```pascal
assign(f, 'some_file.txt');
erase(f);
```

Процедура `rename` получает два параметра, файловую переменную и новое имя файла в виде строки. Подразумевается что файл должен быть назначен в переменную, но не открыт
Пример:
```pascal
assign(f, 'some_file.txt');
rename(f, 'new_filename.txt');
```
