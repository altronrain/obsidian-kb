Разнесение кодовой базы по различным файлам полезно с увеличением объема программы
Причины, по которым работа с одним файлом большого объема неудобна:
* длинный файл тяжело перелистывать
* в файле большого объема обычно все равно редактируем/добавляем небольшой кусок кода, обилие лишнего кода мешает/отвлекает
* с разбиением кода по файлам проще найти нужный фрагмент кода

В FreePascal позволяется включать содержимое одного файла в другой во время трансляции
Это выполняется следующей директивой:
```pascal
{$I myfile.pas}
```
что будет работать также, если вместо строки вставить все содержимое указанного файла

Разбиение на отдельные файлы снимает множество проблем, однако не все. Такой набор файлов остается *одной единицей трансляции*. То есть их можно откомпилировать только *все вместе* за один прием.
Когда программы слишком большие, то их компиляция может занять несколько часов или суток, внесение малейших изменений заставляет повторять процесс. Что делает работу невозможной.

Решить эту проблему помогают модули. См [[Работа с модулями Pascal]]
