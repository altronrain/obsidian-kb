В языке Pascal существует два вида подпрограмм:
* процедуры
* функции

Любые подпрограммы описываются дло начала блока основной программы

## Процедуры
Теоретическая основа работы с процедурой:
Процедура *вызывается* (например, из тела основного блока программы).

Компьютер *запоминает* адрес того места памяти, где в программе ему встретилась инстуркция вызова, после чего *передает управление* вызываемой подпрограмме, то есть переходит к выполнению ее машинного кода.

Когда подпрограмма завершает свою работу, *адрес возврата* используется чтобы *вернуть управление* туда, откуда произошел вызов, а точнее. на следующую после вызова инструкцию.

*Вызов процедуры является оператором*

Оформление процедуры в синтаксисе Pascal:
* заголовок `procedure` с именем/идентификатором процуедуры со *списком формальных парметров* и их типов
* секция локальных описаний
* тело `begin`-`end`

Пример процедуры:
```pascal
procedure PrintChars(ch: char; count: integer);
var
	i: integer;
begin
	for i := 1 to count do
	write(ch)
end;

procedure PrintLineOfDiamond(k, n: integer);
begin
	PrintChars(' ', n + 1 - k);
	write('*');
	if k > 1 then
	begin
		PrintChars(' ', 2*k -3);
		write('*')
	end;
	writeln
end;
```

> [!NOTE] Особенности указания параметров
> В *описании* процедуры с несколькими параметрами различных типов они указываются через точку с запятой `;`
> При *вызове* процедуры параметры передаются через запятую `,`

## Функции
Теоретическая основа работы с функцией:
Также является подпрограммой, которой передается управление.
В *отличие от процедур* функции *предназначены* для вычисления значения (но, строго говоря, не ограничены вычислениями).
*Функция* возвращает управление вызывающему фрагменту программы не просто так, а сообщая резщультат вычисления, то есть она *возвращает значение*.

*Вызов функции является выражением, имеющим тип, указанный как возвращаемое значение функции*

Оформление процедуры в синтаксисе Pascal:
* заголовок `function` с именем/идентификатором функции со *списком формальных парметров* и их типов, а также *типом возвращаемого значения* через `:`
* секция локальных описаний
* тело `begin`-`end`

Пример функции:
```pascal
function Cube(x: real): real;
begin
	Cube := x * x * x
end;
```
Так, `Cube(2.7)` есть выражение типа `real`. Его можно использовать в вычислениях, например:
```pascal
a := Cube(b+3.5) - 17.1;
```
По телу функции вычисляемое значение *присваивается* имени самой функции вместо переменной
Функция может возвращать *одно* значение, так как сама есть выражение указанного типа.
### Функция как логическое выражение
Функция может возвращать и `boolean` выражение, а значит может служить условием для ветвления *сама по себе*
Пример:
```pascal
if IsLatinLetter('с') then
```

### Побочные эффекты функций

> [!NOTE] Материал -- особенность Pascal
> Полезно для выработки привычки, что *выражение всегда вычисляется ради его результата*. И лучше программировать без побочных эффектов (или с минимальными, где это неизбежно, например в C)

Функция строго говоря не ограничена *вычислением выражения*.
Помимо этого она может *сделать что-то еще*, например:
* вернуть текущее время в качестве значения
* вывести строку на экран
* изменить значение глобальной переменной
* изменить значение парамерта-переменной

Что-то где-то изменится из-за того, что было вычислено выражение.

Любые изменения в среде выполнения программы, как в памяти (переменные), так и за ее пределами (вывод на стандартные потоки), которые произошли в ходе вычисления выражения, называют *побочным эффектом*.

Функция, задача которой -- вычислить выражение, делает что-то еще. Это контр-интуитивное поведение. Вместо этого следует использовать процедуры.

Пример как делать *не следует*:
```pascal
function Size: integer;
var
	num: integer;
begin
	write('Enter num: ');
	readln(num)
	Size := num
end;
...
n := Size;
```
Пример как *следует* делать в подобной ситуации:
```pascal
procedure Size(var res: integer);
var
	num: integer;
begin
	write('Enter num: ');
	readln(num)
	res := num
end;
...
n := Size(n);
```

## Параметры-значения
Применимо как к процедурам так и к функциям.

Параметры значения представляют собой *локальную переменную,* в которую записывается *значение*, указанное при вызове.
В качестве значения может быть передано вычисляемое выражение, переменная, значение с использованием переменной:
```pascal
procedure p(x: integer);

p(33+8);

a := 15;
p(2*a + 7);

a := 4;
p(a);
```
Изнутри подпрограммы мы никак не можем повлиять на выражение в вызове. 
Соответственно, для случая с переменной, можно считать, что мы работаем в подпрограмме с ее *копией*, которая никак не связана с оригиналом. Действия над копией на оригинал не влияют

## Параметры-переменные
Применимо как к процедурам так и к функциям

В случае, когда необходимо внутри подпрограммы *изменить значения* одной или нескольких *переменных, находящихся в точке вызова*,  используются *параметры-переменные*
В таком случае *в подпрограмму передается* не значение, а *переменная как таковая*.
Имя параметра в объявлении подпрограммы становится синонимом переменной в точке вызова.
В заголовке подпрограммы перед описанием параметров-переменных ставится слово `var` (которое действует до точки с запятой или закрывающей скобки)
Пример:
```pascal
procedure powers(x: real; var quad, cube, fourth, fifth: real);
begin
	quad := x * x;
	cube := quad * x;
...
{ тело программы }
begin
	powers(17.5, p, q, r, t);
```
в переменные `p`, `q`, `r`, `t` будут занесены результаты вычисления степеней переданного числа

> [!NOTE] Оптимизация с параметрами переменными
> При передаче переменной (сколь угодно большой) в подпрограмму через параметр-переменную никакого копирования не происходит, что позволяет использовать параметры-переменные для *оптимизации*

## Передача пользовательских типов в подпрограммы
При передаче [[Пользовательские типы]], [[Массивы]] и [[Запись]] нельзя использовать анонимные типы. Нужно объявлять тип в секции `type` и указывать его в вызовах:
```pascal
type
	MyRange = 1..100;

procedure p1(b: MyRange);
begin
	writelb(b)
end;
```
Справедливо как для параметров-значений так и для параметров-переменных.

Переменные большого размера лучше вообще не передавать в подпрограммы, а если и делать это, то как параметры-переменные, чтобы не производилось копирование (а передавался только адрес).

## Рекурсия
Применимо как к процедурам так и к функциям

Подпрограмма, при необходимости, может вызвать сама себя. Это возможно:
* напрямую из тела самой подпрограммы
* косвенно (в цепочке различных подпрограмм)

При этом при каждом рекурсивном вызове будет создан *новый комплект локальных переменных*, так как они создаются в момент вызова подпрограммы и исчезают при возврате управления, потому с ними проблем не возникает.

При работе с рекурсивными вызовами важно не уйти в бесконечное выполнение сродни зависанию, пока не отвалимся по памяти. 
То есть рекурсия должна рано или поздно закончиться, а для этого необходимо, чтобы каждый следующий вывод решал туже самую задачу в чуть более простом варианте.
Соответственно, нужно  также определить *базис рекурсии* -- тривиальный или вырожденный случай, при котором дальнейшие рекурсивные вызовы уже не нужны.
Пример:
```pascal
procedure PrintChars(ch: char; count: integer);
begin
	if count > 0 then
	begin
		write(ch);
		PrintChars(ch, count - 1)
	end
end;
```
Здесь для печати уменьшающегося количества символов применен не цикл а рекурсивынй вызов процедуры. Каждый раз на вход вызываемой процедуры будет попадать меньшее значение count, до тех пор пока последняя процедура просто не завершится по условию, а соответственно завершатся и остальные процедуры.

Также используется *обратный ход рекурсии*, когда подпрограмма *сначала делает рекурсивынй вызов, а затем какие-то действия еще*. 
Тогда *выполнение этих действий* будет идти *от последнего* вызова подпрограммы *к первому*, так как возвраты из рекурсий происходят в порядке, обратном заходу в рекурсию.
Пример с печатью цифр числа:
```pascal
procedure PrintDigitsOfNumber(n : integer);
begin
	if n > 0 then
	begin
		PrintDigitsOfNumber(n div 10);
		write(n mod 10, ' ')
	end
end;
```
В таком случае будет выведен корректный порядок, если переставить рекурсивынй вызов и печать местами, то получим обратный порядок цифр.
