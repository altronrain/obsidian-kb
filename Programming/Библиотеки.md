Не рассматриваются разделяемые библиотеки (с суффиксом `.so`) с концепцией динамической загрузки

Библиотека в понимании на уровне редактора связей (компоновщика) -- файл, объединяющий в себе некоторое количество объектных модулей и, как правило, содержащий в себе таблицы для ускоренного поиска имен символов (подпрограмм) в этих модулях.

Простыми словами библиотека -- готовый набор подпрограмм, объединенных в модули, представленные в заранее откомпилированном виде

## Свойство объектных файлов / модулей библиотеки
Важным свойством объектных файлов и модулей заключается в том, каждый из них может быть включен в итоговую программу *только целиком* или не включен вообще.
Так, если в одном модуле объединить несколько подпрограмм, затем потребуется в программе лишь одна из них, весь модуль будет включен в итоговый код программы.

Этот факт стоит учитывать при разбиении библиотеки на модули. Например системные библиотеки чаще всего строятся по принципу "одна функция -- один модуль"

## Составление библиотек
Для составления библиотек применяется утилита `ar` (на самом деле ее применение не ограничивалось только библиотеками, это попросту был архиватор)

Пример объединения нескольких объектных файлов в библиотеку:
```shell
ar crs libgreet.a strlen.o getstr.o putstr.o quit.o
```
Для именования файлов библиотек характерны следующие правила:
* наличие суффикса `.a`
* наличие префикса `lib`
так для *имени библиотеки* `greet` *имя файла* получается `libgreet.a`

Для компоновщика затем можно при создании программы указывать имя файла библиотеки:
```shell
ld greet.o libgreet.a
```
или указывая имя библиотеки флагом `-l` и директорий библиотеки флагом `-L`:
```shell
ld greet.o -l greet -L .
```
последний вариант удобен для библиотек, *установленных* в системе, так как системные директории линкер знает сам и флаг `-L` не нужен

При разработке библиотек следует по возможности избегать зависимостей от других библиотек, и избегать в любом виде перекрестных зависимостей между библиотеками
(см. [[Алгоритм работы редактора связей#Свойства редактора связей и принципы построения библиотек]])
