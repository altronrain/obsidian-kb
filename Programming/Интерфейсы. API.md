При реализации механизмов, относительно которых важны *доступные операции*, а не то, как они реализованы (не зависящий от внутренней структуры) то такой набор процедур вместе с пояснениями по применению называется *интерфейсом* (или *API*)

Программисту, использующему далее интерфейс, *нет необходимости* знать как реализован функционал (те самые операции). А внесение изменений в реализацию *не должно* влиять на использование интерфейса.

Обычно, для разработки удачного интерфейса функционал как раз начинают продумывать с него самого. То есть, задумываются о том, какие операции должна предоставлять реализация, затем уже приступают непосредственно к реализации
