Язык Си регистрозависим:
* ключевые слова требуют обязательной записи в нижнем регистре
* идентификаторы принято писать в snake_case

Комментарии языка Си заключаются в `/* */` (начало и конец комментария), причем вложенные комментарии компилятор не понимает. Комментарии в Си нельзя использовать для временного исключения из компиляции кусков кода (нужно использовать директивы условной компиляции)
Стиль строчных комментариев из Си++ `//` не приветствуется

Программа на Си состоит из функций. Нет никакого аналога *головной программы*. Любые операторы могут находиться только в телах функций.
Вне функций могут располагаться:
* описания типов
* описания глобальных переменных и т.п.
они не являются операторами и ничего сами по себе делать не могут

По соглашениям с функции `main` начинается выполнение программы (стандартная библиотека предоставляет код, по которому с метки `_start` вызывается функция `main`, а возвращенное ею значение подается на `_exit`)

Оператор `return` завершает выполнение одной функции. Как частный случай, для функции `main` оператор `return` приведет к завершению работы программы (если там нет рекурсивного вызова `main`), а переданное число будет рассматриваться как код возврата.

Для выхода из программы сущестует библиотечная функция `exit`, описанная в `stdlib.h`, принимающая число от 0 до 255 (который будет рассматриваться как код возврата). Функция `exit` подготавливает программу перед завершением работы, вытесняя всю информацию из буферов, что там могла оставаться

Символ точки с запятой `;` в Си является *частью синтаксиса оператора*, а не разделяет операторы

Значение, возвращаемое функцией `main`, предназначается ОС в роли *кода равершения*
