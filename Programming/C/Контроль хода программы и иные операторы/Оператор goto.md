Оператор `goto` является *оператором безусловного перехода*
Синтаксис:
```c
goto <метка> ;
```

В отличие от Pascal не требует предварительного описания метки, на которую будет дулеться переход. Можно использовать любой произвольный не занятый идентификатор

Область видимости идентификатора -- функция. Между функциями "прыгать" нельзя.
*Внутри* одной функции можно *прыгруть куда угодно*, в том числе внутрь сложного оператора (но делать этого не следует никогда)

Меткой можно пометить *только оператор*.
Если имеется необходимость поставить метку *в конце* составного оператора, *после* нее необходимо предусмотреть *пустой оператор* (обычно `;`)

Метка записывается непосредственно перед оператором и отделяется от него двоеточием:
```c
emerg_exit: return 1;
```

Подобные переходы лучше не использовать на практике, кроме двух случаев:
* выход из нескольких вложенных друг в друга конструкций
* освобождение локально захваченных ресурсов перед досрочным завершением подпрограммы
см. также [[Безусловные переходы#Когда применение goto допустимо и желательно]]
