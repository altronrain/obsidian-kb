Чаще других операторов в C встречается *оператор вычисления выражения ради побочного эффекта*
Синтаксис:
```c
<произвольное арифметическое выражение>;
```
Символ `;` превращает *выражение* в законченную конструкцию -- *оператор*

Роль `;`:
* сообщить компилятору, что конструкция закончилась и можно не ожидать появления знаков операций далее
* показывает, что для данного выражения нас *не интересует его значение*

Конструкции вида `x + 5;` или вовсе `5;` являются легитимными с точки зрения языка, но бессмысленными, ибо в них нет никакого побочного эффекта

Для чего используется этот оператор:
* при присваивании
* вызов функции ради побочного эффекта

При присваиваниях записывается *выражение* (операция) присваивания, которая имеет значение (результат вычислений), но нас оно не интересует, с помощью `;` мы сообщаем что выражение является завершенной конструкцией.
Пример:
```c
x = 17;
```
Это *не* оператор присваивания (в C такого оператора нет)

Пример вызова функции ради побочного эффекта с использованием оператора вычисления выражения:
```c
printf("Hello, world\n");
```
Вычисляемым выражением здесь является бинарная операция вызова функции:
* `printf` -- первый аргумент
* строковый литерал `"Hello, world\n"` -- второй аргумент
вычисленное значение будет равно `13`, но оно нас не интересует, о чем мы сообщаем компилятору, поставив `;`
