Оператор выбора `switch` является самым запутанным в C. 
По сути также является обобщением ветвления на *N* ветвей.
Синтаксис:
```c
switch ( <выражение> ) { <метки и операторы> }
```
при этом:
* `<выражение>` должно иметь произвольный *целочисленный* тип
* `<метки и операторы>` -- произвольная последовательность операторов языка C, *первый* из которых, а также, возможно, *некоторые* другие снабжены *case*-метками

Обыкновенные case-метки состоят из:
* ключевого слова `case`
* целочисленной *константы времени компиляции*
* двоеточия
Синтаксис:
```c
case <константа> :
```

Второй вид case-меток выглядит проще:
```c
default :
```
такая метка обозначает *все варианты, крроме явно перечисленных* (аналог Python `_:`)
Обычно метка `default` ставится в конце оператора `switch`

В качестве констант времени компиляции *нельзя* использовать константы, описываемые как переменные (со словом `const`), так как они *не считаются константами времени компиляции* (ибо она может вообще располагаться в другой единице трансляции)

Что отличает такой оператор в языке C от других ЯП: 
Операторы в теле `switch` представляют собой *единую секцию операторов*, которая выполняется *от метки, соответствующей вычисленному выражению, и до конца тела* `switch`

Иначе говоря, case-метка являет собой точку, откуда выполнение тела `switch` *может начаться*, но *не предписывает завершить выполнение* операторов, если оно началось выше нее.

По этому чаще в операторе `switch` для каждой case-метки используют еще и `break`, так как необходимо разделить вычисления на непересекающиеся ветви:
```
switch(t) {
case 1:
	printf("First\n");
	break;
case 2:
	printf("First\n");
	break;
case 3:
	printf("First\n");
	break;
case 4:
	printf("First\n");
	break;
case 5:
	printf("First\n");
	break;
default:
	printf("First\n");
}
```
Оператор `break` также как и в циклах завершает выполнение тела, передав управление в конец оператора `switch`
Помимо `break` можно испаользовать:
* `goto` наружу
* `return`
* при наличии объемлющего цикла -- `continue` для завершения итерации
в реальности чаще применяется `return`

Подобные опреаторы имеют свойство сильно "распухать", с чем можно бороться следующими способами (применимо ко многим ЯП с подобными конструкциями):
* выносить `switch` целиком в отдельную функцию или оставлять с оператором `switch` как можно меньше другого кода
* если отдельные секции `switch` поддаются оформлению в виде обособленных функций, сделать это, оставив в теле `switch` только вызовы функций и `break`
* *никогда* не размещать один `switch` внутри другого

Про `break` и `continue` смотри так же [[Оператор continue]] и [[Оператор break]]
