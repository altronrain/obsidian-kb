Над всеми базовыми (встроенными) типами языка C определены обычные арифметические операции (все они подразумеваются бинарными):
* `+` сложение
* `-` вычитание
* `*` умножение
* `/` деление:
	* обычное арифметическое, если операнды с плавающей точкой: `5.0/2.0 -> 2.5` (приблизительно)
	* целочисленное, если операнды -- целые: `5/2 -> 2` (точный результат с применением *усечения*)
	* `%` взятие остатка от деления (только для целочисленного деления)

На практике *не надо* применять деление к опреандам *разных* типов, стоит привести опреанды к одному типу. *Остаток* от деления следует брать только при *целых положительных операндах*

При целочисленном делении всегда производится *усечение*, а именно *усечение в сторону нуля* (начиная со стандарта C99).  Например:
```
$ ./div  
Целочисленное деление отрицательных чисел: 19 / 5 = 3  
Целочисленное деление отрицательных чисел: -19 / 5 = -3
```
При этом усечение производится *сразу же* (для остальной части выражения будет использовано усеченное значение)


При *взятии остатка* `%` отделения при целочисленном делении, начиная со стандарта C99 (когда у нас действует усечение в сторону нуля) при вводе отрицательных значений действует следующее правило:
* если первый операнд отрицателен -- будет получен отрицательный результат
* в остальных случаях будет получен положительный результат

Также, взятие остатка от целочисленного деления можно вычислить путем выполнения:
```c
a % b = a - (a / b) * b;
```
Пример на `11 % -5`:
```c
11 - (11 / -5) * (-5) = 11 - (-2) * (-5) = 11 - 10 = 1
```

Про нюансы операции деления над числами с плавающей точкой см. также [[Числа с плавающей точкой#Субнормальные числа]]
Также с числами с плавающей точкой можно наблюдать ошибки округления данных.
