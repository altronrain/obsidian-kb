В C существуют операции *инкремента* и *декремента*:
* `++`
* `--`
это унарные операции, обычно применяемые к целочисленным переменным. Предписывают увеличение (уменьшение) значения переменной на `1`

Существует две формы таких операций:
* префиксная
* суффиксная

Пример префиксной формы:
```c
++i /* эквивалент i += 1 */
--i /* эквивалент i -= 1 */
```
Пример суффиксной формы:
```c
i++ /* эквивалент i += 1 */
i-- /* эквивалент i -= 1 */
```
В обеих формах с переменной `i` происходит в итоге *ровно то же самое*

Подобные операции могут встречаться в выражениях, а следовательно иметь значение. В значении и кроется *разница форм*:

* в *префиксной* форме значением выражения будет *новое* значение переменной
* в *суффиксной* форме значением выражения будет старое значение переменной

Пример:
```c
/* x = 8, y = 8 */
x = 7;
y = ++x;

/* p = 8, q = 7 */
p = 7;
q = p++;
```

Или, перефразируя:

* постфиксная форма -- переменная меняется после использования ее значения
* префиксная форма -- переменная меняется до использования ее значения


> [!NOTE] Где не следует применять инкремент и декремент
>  1. Не применяйте операцию инкремента или декремента к переменной, которая является частью более чем одного аргумента функции
>  2. Не используйте операцию инкремента или декремента к переменной, которая появляется в выражении более одного раза
>  
>  В таких случаях порядок вычислений не определен и не гарантирован.
>  Примеры:  
> ```
> printf("%10d %10d\n", num, num* num++);
> ans = num / 2 + 5 * (1 + num++);
> n = 3;
> y = n++ + n++;
> ```
