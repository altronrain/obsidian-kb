Набор логических операции:
* `&&` конъюнкция (логическое И)
* `||` дизъюнкция (логическое ИЛИ)
* `!` унарная операция отрицания (логическое НЕ)
* `!=` используется для логического исключающего или, так как это то же самое not equal
* отрицание XOR, соответственно можно выразить как `==` (equal)

В C используется схема *ленивых вычислений*, то есть если значение выражения уже определено, второй операнд не вычисляется.
Примеры:
```c
f() || g() /* g() не вычисляется если f() вернула истину */
f() && g() /* g() не вычисляется если f() вернула ноль (ложь) */
```
что доказывает, что операторы `||` и `&&` также являются точками следования, гарантирующими вычисление левой части выражения перед вычислением правой.

Это позволяет выполнять присваивание в левой части и использовать переменную в правой:
```c
while ((c = getchar()) != ' ' && c != '\n')
```

Начиная со стандарта C99 имеется возможность альтернативной формы записи логических операций при включении заголовочного файла `iso646.h`
```
      and       &&
      not       !
      not_eq    !=
      or        ||
      xor       ^
```
