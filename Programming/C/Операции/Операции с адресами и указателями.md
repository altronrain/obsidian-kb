## Взятие адреса и разыменование

Основные операции:
* операция взятия адреса -- `&`
* операция разыменования -- `*`

Пример работы с указателями:
```c
int x;    /* переменная x типа int */
int *p;   /* указатель на тип int p */

p = &x;   /* занесение адреса x в указатель p */
*p = 27;  /* *p -- синоним x. занесение 27 в переменную x  */
```
Пример для пояснения постановки символа `*`:
```c
int x, y;
int *p, *q, *r;

x = 25;
y = 36;
p = &x;
q = &y;
```
Тогда:
```c
r = p;
```
есть присвоение переменной `r` значения переменной `p`, то есть в `r` будет находиться *адрес* переменной `x`
В то время как:
```c
*r = *q;
```
есть присвоение переменной `x` значения переменной `y`, то есть числа `36`. Сами указатели при этом не изменяются

## Арифметические операции над указателями

Помимо операций взятия адреса и разыменования над указателями допустимы арифметические операции с *целыми* числами. При этом получаемый результат будет зависеть от *типов указателей*

Так, в современных системах обычно указатели 64-битные сами по себе (8 байт).
Их адреса выражены в виде 16-ричных значений (hex).

*Сложение* и *вычитание* целочисленных значений с укзателями выполняется путем *умножения целого* числа на *количество байтов* в *типе данных*, на которые указывает указатель. После чего результат добавляется к исходному адресу или вычитается из него, в зависимости от выполняемой операции.

*Инкремент* и *декремент* будут выполнять подобный сдвиг на величину, равную единице данных в объявленном указателе (если int -- чаще всего 4 байта).

С инкрементом, декрементом, сложением и вычитанием нужно быть аккуратным и не выходить за пределы памяти программы (или исследуемого массива). Допускается максимум указание на первый элемент за массивом (по его окончании).

Можно получать *разность* нескольких указателей. Выполнять данную операцию можно над указателями *одного типа*. Обычно выполняется над указателями на элементы одного массива, чтобы понять, каково расстояние между элементами.
Разность выражается в *количестве элементов* типа, соответствующего типу указателя.
```
значение указателя, разыменованный указатель, адрес указателя:
ptr1 = 0x7ffe87519e00, *ptr1 = 100, &ptr1 = 0x7ffe87519df8

сложение значения int с указателем:
ptr1 + 4 = 0x7ffe87519e10, *(ptr1 + 3) = 400

значение после выполнения операции ptr1++:
ptr1 = 0x7ffe87519e04, *ptr1 = 200, &ptr1 = 0x7ffe87519df8

значение после выполнения операции ptr2--:
ptr2 = 0x7ffe87519e04, *ptr2 = 200, &ptr2 = 0x7ffe87519df0

восстановление исходных значений указателей:
ptr1 = 0x7ffe87519e00, ptr2 = 0x7ffe87519e08

вычитание одного указателя из другого:
ptr2 = 0x7ffe87519e08, ptr1 = 0x7ffe87519e00, ptr2 - ptr1 = 2

вычитание из указателя значения типа int:
ptr3 = 0x7ffe87519e10, ptr3 - 2 = 0x7ffe87519e08
```

## Иные операции с указателями

Что логично -- *присваивание*. Указателю можно присвоить:
* имя массива (будет присвоен адрес первого элемента массива)
* переменную, над которой выполняется операция взятия адреса
* другой указатель

При этом требуется, чтобы *адрес был совместим с типом указателя* (адрес переменной `double` не получится присвоить указателю на `int` напрямую)

Над указателями также можно выполнять операции *сравнения* (если они имеет *один и тот же тип*). Большим будет указатель, содержащий больший адрес.

## Ошибки при работе с указателями

*Ни в коем случае нельзя* присваивать значение в неинициализированный указатель.
В процессе чего происходит *разыменование* такого неицилиализированного указателя.
```c
int *ptr;     /* указатель объявлен, но не инициализирован значением */
*ptr = 5;     /* ошибка */
```

Мы пытаемся сохранить значение в ячейке, на которую указывает `ptr`, но так как указатель неинициализован, он имеет *случайное значение*. Что при этом произойдет -- никто не гарантирует.

Создание указателя приводит к выделению памяти только *под сам указатель*.
Для хранения данных память не выделяется. *Перед использованием указателю должен быть присвоен адрес ячейки памяти, что уже была определена*.

Так:
* при применении функции с параметром типа указатель ему присваивается адрес переданной фактической переменной
* можно присвоить указателю другой указатель или адрес переменной перед применением
* можно воспользоваться функцией `malloc()` для предварительного выделения памяти
