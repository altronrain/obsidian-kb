## Простое присваивание
В языке C присваивание является *операцией*, а не оператором, а значит, *может* использоваться в выражениях

Пример:
```c
a = b;
```
является *выражением*, которое имеет *значение* (равно тому значению, которое было только что присвоено)

Также в C возможно подобное:
```c
x = (y = 17) + 1;
/*...*/
a = b = c = d = e + 5;
```

Дополнительно:
Чтобы понять почему, так, следует принять тот факт, что в момент разработки языка еще не бсуществовало оптимизаторов. 

Операция `x = y` в языке ассемблера представляет собой:
```nasm
mov eax, [y]
mov [x], eax
```
и так для каждого присваивания будет создаваться пара. 
Если подобных присваиваний несколько, то повторные операции `mov eax, [y]` оказываются лишними, только что *присвоенное значение остается в аккумуляторе*, и если следующая операция использует его, то можно оптимизировать ассемблерный код.
Пример без оптимизации:
```
a = x;
b = x;
c = x;

mov eax, [x]
mov [a], eax
mov eax, [x]
mov [c], eax
mov eax, [x]
mov [c], eax
```
Пример с оптимизацией:
```
a = b = c = x;

mov eax, [x]
mov [c], eax
mov [b], eax
mov [a], eax
```

По поводу того, что может стоять слева от знака `=` в операции присваивания, см. [[Леводопустимые выражения#Леводопустимые выражения]]
(если кратко, то там могут располагаться сущности, которым есть куда присваивать значения, то есть они указывают/ссылаются/связаны тем или иным образом с определенной ячейкой/областью памяти, в которую можно произвести запись)
Что логично, основное назначение операции присваивания -- записать/сохранить значение в ячейке памяти.

## Присваивание переменной значения, полученного из нее же
Это могут быть операции вида:
```
+= -= *= /= %= &= |= ^= <<= >>=
```
Например:
```c
a += 7
```
переменной `a` присвоить ее же значение, увеличенное на `7`

Следует помнить что запись вида `a += b` не является эквивалентом `a = a + b` в полной мере. Между ними есть разница

Так, выражение слева может быть довольно сложным, и вообще зависеть от вычисления функций, например:
```c
v[f(x)] += 10;
v[f(x)] = v[f(x)] + 10;
```

Первая запись предпочтительнее (можно сказать -- является *единственно корректной* формой), ибо функция `f(x)` вычисляется *один* раз

Во втором случае функция будет вычисляться дважды. Это вхолостую потраченное время CPU, а также возможное двойное выполнение побочных эффектов (и ладно бы безобидных, таких как вывод на экран). А если `f(x)` при одних и тех же вводных возвращает разные значения -- то во второй форме записи возьмем значение одного элемента массива и запишем в совершенно другой.

В каждой записи вида `<операция>=` выполняется:
* извлечение значения из стоящей слева переменной
* выполнение операции над значением и правым операндом
* результат операции заносится обратно в переменную
