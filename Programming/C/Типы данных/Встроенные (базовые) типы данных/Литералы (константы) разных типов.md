Литералы -- значения, записанные в программе в явном виде. 

## Запись целых чисел
Про типы см. [[Целые числа]]

Язык C позволяет записывать целые числа в:
* десятичной -- форма *без лидирующего нуля*
* шестнадцатеричной -- форма *с лидирующим нулем* перед числом
* восьмеричной системах счисления -- форма с префиксом `0x`

Примеры записи чисел в системах счисления:
```c
# dec
123   1024   42
# oct
052   035
# hex
0x2A   0x1d   0x1D   0x7E8
```

### Знаковость литералов. Авто-определение типов
В языке C имеется разница в знаковости чисел *по умолчанию*, в зависимости от формы записи.
Так, числа:
* в десятичной форме *в большинстве случаев* знаковые
* в восьмиричной форме *всегда* беззнаковые
* в шестнадцатиричной форме *всегда* беззнаковые

Также со знаковостью связано и *авто-определение типов*.
Так, для чисел в *десятичной* системе счисления:
* если число умещается в `int` -- `int`
* если число больше `int`, но умещается в `unsigned int` -- `unsigned int`
* если число больше `unsigned int`, но умещается в `long` -- `long`
и так далее, пока число не перестанет умещаться в `long long`, что просто вызовет ошибку

Для чисел в *восьмиричной* и *шестнадцатиричной* системах счисления ситуация выглядит похожим образом, за отсутствием этапов со знаковыми диапазонами:
* если число умещается в `unsigned int` -- `unsigned int`
* если число больше `unsigned int`, но умещается в `unsigned long` -- `unsigned long`
и так далее (аналогично, > `unsigned long long` вызовет ошибку)

> [!NOTE]
> Исходя из описанного, литералов типов `char` и `short` в Си нет, не существует.
> Литералы вида `a`, `Z` имеют тип `int`

### Явное указание типов литералов целых чисел
Можно явно указать тип литерала (больший чем `int`) если это требуется, добавив в конец литерала комбинацию из суффиксов:
* `L` (`long`)
* `LL` (`long long`)
* `U` (`unsigned`)
Так, пример:
```c
15      /* будет иметь тип int */
15L     /* будет иметь тип long */
15ULL   /* будет иметь тип unsigned long long */ 
```
такую форму записи можно применять и к `oct` и `hex` записям чисел

## Запись чисел с плавающей точкой
Числа с плавающей точкой записываются в виде литералов в десятичной системе счисления.
Форма записи в общем случае представляет собой:
* целую часть
* дробную часть, отделенную от целой точкой
* порядок (`e` или `E` и далее степень числа 10)
при этом можно указывать только целую, только дробную, опускать порядок
Пример вариантов записи числа `1500.0`:
```c
1500.0  1500.  15E2  15.E2  15E+2  0.15e4  .15e4  15000e-1
```

Тип по умолчанию -- `double`. Можно использовать суффиксы:
* `f` (или `F`) (`float`)
* `L` (`long`)
для принудительного указания типов `float` или `long double`

## Запись составных литералов (массивов) (С99)
Используется для передачи констант массивов в функции, обрабатывающие массивы. Требует стандарта C99.

Такой литерал выглядит подобно списку инициализации массива, но дополнительно предварен *именем типа*, заключенным в круглые скобки.
Так, например:
```c
int diva[2] = {10, 20}; /* обычное оъявление массива с инициализацией */
(int [2]) {10, 20};     /* составной литерал-массив */
```

Имя типа соответствует конструкции из объявления массива, из которой убрали имя массива и все это заключили в круглые скобки.

При этом размер массива можно и не указывать, как и в объявлениях:
```c
(int []) {10, 20, 30};
```

Для многомерных массивов данный способ также масштабируется:
```c
(int [2][4]) { {1, 2, 3, -9}, {4, 5, 6, -8}};
```

Такие составные литералы имеют гарантированное *время существования внутри выполняемого блока*.


## Запись символов и строк
В C *символ* и *код символа* -- одно и то же.
Для записи кодов символов используются *символьные литералы*, формируемые с помощью *апострофов*. Они имеют тип `int`

### Служебные символы
В первую очередь объявить служебный символ можно по его цифровому коду, так например:
```c
char beep = 7;     /* ASCII-код для звукового сигнала терминала */
```

Также для представления *служебных символов* предусмотрены *escape-последовательности*
```
'\n' '\r' '\t' '\a' '\b' '\f' '\v'
```
если требуется напечатать символ обратного слеша, он, соответственно, тоже экранируется `'\\'`. Аналогично с символом апострофа или кавычек

### Строки и отдельные символы
Для представления *строк* используются *строковые литералы* (обрамляются *двойными кавычками*)
Все имеющиеся escape-последоватлельности работают так же как и в представлении символов.
Пример:
```c
"Never say \"never\""
```

Разница между символьными и строковыми литералами:
* при встрече символьного литерала компилятор заменит его на *целое число* типа `int`
* при встрече строкового литерала компилятор расположит (в секции кода) массив из элементов типа `char`, завершающийся нулевым элементом `0`, а сам литерал будет заменен на *адрес* области памяти, где расположен массив
Примеры:
```c
'Z'      /* 90 */
"Z"      /* [ 90, \0 ] */
"text"   /* [ 116, 101, 120, 116, 0 ] */
```
Для обозначения символа с кодом ноль часто используют запись `\0` для удобного восприятия. Семантически то же самое что обычный `0`

Компилятор умеет *склеивать* несколько строковых литералов разделенных пробелами, что бывает удобно если строка не влезает в 80 колонок):
```c
"This " "is" " a string" " whi" "ch is" " long"
/* равносильно */
"This is a string which is long"
```


> [!NOTE] Про символьные константы (литералы)
> Выше уже говорилось, что компилятор заменяет символьный литерал на целое число типа `int`. 
> 
> В этой связи нужно быть осторожным при инициализации переменных типа `char` или присвоении им значений. Во время этого процесса:
> 
> * строковый литерал --  числовое значение, хранящееся в ячейке памяти, равной по размеру типу `int` (например, 32-бита);
> * присваивается (помещается) в переменную типа `char`, и по итогу хранится в 8-битной единице памяти;
> 
> Таким же образом в C на системе с 32-битным `int` валидно описание символьной константы вида `'FATE'` из 4 отдельных символов, которые будут занимать все 32-бита. Попытка присвоить такой литерал переменной типа `char` поместит в переменную только последнюю букву `E`
