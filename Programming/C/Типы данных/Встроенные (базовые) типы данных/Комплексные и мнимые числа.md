Комплексные и мнимые числа введены в стандарте C99. Они не являются обязательными и их поддержка не гарантирована со стороны компилятора.
На практике применяются довольно редко. Начиная с C11 весь пакет поддержки комплексных числе сделан необязательным.

## Комплексные типы
Комплексные типы:
```c
float _Complex
double _Complex
long double _Complex
```
Каждый из типов содержит по два числа соответствующего типа с плавающей точкой:
* одно число для действительной части (`float`, `double` или `long double`)
* второе число для мнимой части (`float`, `double` или `long double`)

## Мнимые типы
Мнимые типы:
```c
float _Imaginary
double _Imaginary
long double _Imaginary
```

Если включить заголовочный файл `complex.h`, то можно:
* `_Complex` заменить в коде на `complex`
* `_Imaginary` заменить в коде на `imaginary`
* использовать символ `I` для представления квадратного корня из `-1`

Комитет по стандартизации не ввел напрямую ключевые слова `complex` и `imaginary`, так как до появления C99 из соображений обратной совместимости кода (так как многие могли использовать слово `complex` в кастомных структурах для обеспечения работы с комплексными числами, как пример).
