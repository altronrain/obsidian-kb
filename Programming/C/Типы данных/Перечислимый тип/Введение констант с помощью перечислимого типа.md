Напоминание: наличие в программе явным образом указанных чисел крайне нежелательно, за исключением очевидных случаев с использованием `0, 1, 2, -1`. Все остальные числовые константы *следует снабдить именами*

> [!NOTE] Историческая справка по объявлению констант в C
> В C изначально для именования константных значений приходилось использовать макропроцессор. Макропроцессор отрабатывает *после лексического* анализа, но *раньше синтаксического*. В связи с этим константы, введенные как макросы, не подчиняются областям видимости и крайне не удобны в использовании

С появлением перечислимого типа появился единственный способ в C описать корректный идентификатор:
* подчиняющийся соглашениям об областях видимости
* представляющий собой константу времени компиляции

Потому в программах на C можно часто встретить подобные конструкции:
```c
enum { max_buffer_size = 1024 };
```
Очевидно, что конструкция не имеет смысла в качестве "перечислимого типа", а основная задача состоит в создании *имени* `max_buffer_size`, которое в программе будет использоваться далее, а при компиляции преобразовываться в число `1024`

Само средство (перечислимые типы) используется не по прямому назначению, а метод ввода констант представляет собой *хак*

Описанный выше способ обладает одним *существенным ограничением*
Таким способом можно ввести только *целочисленную* константу (для остальных типов все еще придется использовать макропроцессор)

Имена констант в C обычно записываются в *верхнем регистре*, однако этому правилу не стоит следовать, когда речь идет о константах, вводимых через `enum`
См. также [[Макропроцессор]]
