Так как константы-значения перечислимого типа являются *константами времени компиляции* они прекрасно подходят для использования в составе `case-` меток оператора `switch` (см. [[Programming/C/Контроль хода программы и иные операторы/Оператор выбора|Оператор выбора]])

Конечно, в качестве метки может использоваться и целочисленный тип, но такой подход *не рекомендуется никогда*. 

Если использовать в заголовке оператора перечислимый тип `enum` (или функцию, возвращающую значение этого типа), то компилятор будет оказывать дополнительную поддержку, *проверяя, все ли возможные значения мы обработали*.

То есть, *компилятор предполагает*, что если выражение, по которому производится выбор в операторе `switch`, имеет перечислимый тип, то в теле `switch` должны присутствовать `case-`метки для *всех различных значений* перечислимого типа (либо должна присутствовать метка `default`)
Пример:
```c
enum greek { alpha, beta, gamma = beta, delta };

void greek_print(enum greek x)
{
	switch(x) {
	case alpha:
		printf("Alpha\n");
		break;
	case beta:
		printf("Beta\n");
		break;	
	}
}
```
при компиляции получим предупреждение от компилятора, о том что значение `delta` не обработано в операторе `switch` (про `gamma` ничего не скажет, так как оно совпадает с `beta`, который был обработан)

Чтобы избавиться от предупреждения, достаточно добавить метку `default` (и, например, поставить там пустой оператор):
```c
/*...*/
default:
	;
```
но этого делать *не рекомендуется*

Вместо сокрытия ошибки лучше ее проанализировать, зачастую может сократить время отладки, ибо подразумевается, что *все случаи* `enum` должны обрабатываться. Так, можно *добавить* новое значение в перечислимый тип `enum` и *забыть добавить* его в оператор выбора `switch`. С наличием `default` предупреждения не увидим.

Перечислимый тип в общем-то и предназначен для оператора выбора, чтобы избегать подобных конструкций с целочисленными значениями:
```c
switch(n) {
case 1:
	/*...*/
case 2:
	/*...*/
case 3:
	/*...*/
/*...*/
case 13:
	/*...*/	
}
```
где читающему программу будет крайне не понятно *что означают* данные числа.
