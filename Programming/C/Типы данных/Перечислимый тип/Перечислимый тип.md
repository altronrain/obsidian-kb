## Основы перечислимого типа
*Перечислимый тип* -- такой тип данных, множество значений которого *конечно* и *явным* образом задано в виде набора *идентификаторов*.

В языке C обозначается ключевым словом `enum`
Пример перечисляемого типа:
```c
enum colors { red, orange, yellow, green, blue, violet };
```
где:
* `red, orange, ...` -- набор значений нового типа
* `colors` -- *имя перечисления*
* `enum colors` -- *имя* нового (введенного) *типа*
по этому *ввод переменной* такого нового типа будет выглядеть следующим образом:
```c
enum colors one_color;
```
без ключевого слова `enum` компилятор выдаст ошибку

В C *идентификаторы, введенные в качестве возможных значений в описании перечислимого типа, считаются целочисленными константами* (имеют тип `int`)
Нумерация будет начинаться с `0`. Так для примера выше:
```
red    -> 0
orange -> 1
yellow -> 2
...
```
при этом идентификаторы становятся *полноценными синонимами* соответствующих чисел
Так, например осмысленным будет выражение:
```c
violet * green - blue /* 5*3 -4 = 11 */
```
но на практике так делать не стоит

В переменную перечислимого не запрещается занести любое целое число типа `int`, но так делать тоже не надо (да и компилятор выдаст предупреждение)

## Постановка `;` в описании перечислимого типа
*Точка с запятой*  `;` после закрывающей фигурной скобки довольно нетипична для C.
В C подобное сочетание встречается в *двух* случаях:
* при описании пользовательских типов
* при описании переменных со сложными инициализаторами (например инициализаторы массивов)

Точкой с запятой должно заканчиваться любое *объявление* или *описание*, за *исключением* *описания функции* (в котором вместо `;` идет тело функции)

В C *не различаются* конструкции *описания типа* и *описания переменной*. Позволяется совместить два в одном выражении таким образом:
```c
enum states {
	running, blocked, ready
} new_state, last_state;
```
где:
* `enum states` -- вводимый тип
* `running, blocked, ready` -- три целочисленные константы
* `new_state, last_state` -- две переменные типа `enum states`

В любом случае с постановкой символа точки с запятой следует быть внимательным, ибо ошибка от компилятора по этому поводу часто *крайне невразумительная* (сложная для понимания)

> [!NOTE] Точка с запятой в структуре описания
> В продолжение, `;` просто обозначает конец структуры *описания*, которая в общем случае состоит из:
> * спецификатора типа
> * списка описываемых переменных (возможен пустой список)
> Так, формально синтаксис C позволяет оставить список переменных пустым в любом описании (в том числе и с базовыми типами):
> ```
> int ;
> ```
> хоть это и *бессмысленно* (большинство компиляторов выдаст предупреждение)

## Особенности применения перечислимого типа
### Без имени перечисления
Стандартом C разрешено опускать имя перечисления (что лишает возможности описать переменную подобного перечислимого типа). Используется для задания констант
Пример:
```c
enum { alpha, beta, gamma };
```
### Указание значений констант
Также, позволяется указать значения констант самостоятельно:
```c
enum colors {
	red     = 0xff0000,
	green   = 0x00ff00,
	blue    = 0x0000ff,
	yellow  = 0xffff00,
	cyan    = 0x00ffff,
	magenta = 0xff00ff,
	black   = 0,
	white   = 0xffffff,
};
```
что существенно расширяет область применения таких констант.

Позволяется задавать в одном перечислении *несколько* констант с одинаковым значением:
```c
enum colors {
	/* ... */
	cyan     = 0x00ffff,
	aqua     = 0x00ffff,
	/* ... */	
};
```
### Смешивание констант со значением и без значения
Стандарт C позволяет в одном перечислимом типе смешивать константы с указанием значения и без указания значения
Пример:
```c
enum numbers { one = 1, two, three, first = 1, second, third };
```
при этом константа, значение которой не указано, будет на единицу больше предыдущей. 
Так для примера выше `two` и `second` будут иметь значение `2`, а `three` и `third` -- `3`

Справа от знака равенства может быть указана любая *константа времени компиляции*, в том числе выражения, которые компилятор может вычислить. Например:
```c
enum example {
	first = 100,
	second = first * 20,
	third = second + 1000,
	last = third
};
```
где:
* `second` получит значение `2000`
* `third` получит значение `3000`
* `last` получит значение `3000`
