Язык C позволяет в некотором роде смешивать различные типы данных в одном выражении. Это может быть как удобным, так и опасным явлением, по этому следует понимать базовые правила *автоматического* преобразования типов.

> [!NOTE] Внимание
> В большинстве случаев следует избегать автоматического преобразования типов.
> А когда это кажется удобным, стоит еще раз подумать над структурой кода.
> 
> Все же если решите применять автоматическое преобразование типов, то необходимо убедиться что приняты все меры предосторожности (и не допускается *понижение* типов)

Помимо автоматического преобразования типов их можно также указывать самостоятельно, то есть выполнять *приведение* типов. См. [[Иные (экзотические) операции#Операция приведения типов]]

## Повышение типов `char` и `short` в выражениях

Типы `char` и `short` (как `signed` так и `unsigned`), *находясь в выражении*, автоматически *преобразуются* *(повышаются)* до типа `int` (или `unsigned int`, если в системе `short` равен `int` по размеру, то `unsigned short` больше чем `int`)

## Повышение к более высокому типу

Базовое правило: Если в любую операцию вовлечены два типа, то оба значения приводятся к более высокому из этих двух типов

Порядок типов от высшего к низшему (см. также [[Целые числа]]):
* `long double`
* `double`
* `float`
* `unsigned long long`
* `long long`
* `unsigned long`
* `long`
* `unsigned int`
* `int`

Для `unsigned int` возможно исключение (преобразование сразу в `unsigned long`) если в системе `int` равен по размеру `long`

## Преобразование в операции присваивания

Результат вычислений преобразуется к типу переменной, которой присваивается значение. Процесс может привести как к *повышению* типа, так и к *понижению*.

## Преобразование  типов при передаче в качестве аргументов

При передаче значений в функции автоматически выполняется *повышение*:
* `char` и `short` преобразуются в `int`
* `float` преобразуется в `double` 
 
Яркий и знакомый пример -- `printf`

Помимо повышения возможны иные преобразования согласно описываемым ниже правилам (зависит от прототипа функции [[Programming/C/Функции#Объявление функции и прототипы]])
Например:
```c
void pound (int n);
...
int times = 5;
char ch = '!';  /* ASCII: 33 */
float f = 6.0f;
...
pound (times);  /* аргумент типа int */
pound (ch);     /* эквивалентно pound ((int) ch) */
pound (f);      /* эквивалентно pound ((int) f) */
```

## Возможные последствия понижения типов

Если с повышением типов обычно все происходит гладко и без последствий, то при *понижении* могут возникнуть проблемы, когда преобразованное значение не помещается в конечном типе. 

Поведение при *понижении* типа:
* целевой тип: целый тип без знака (`unsigned`)
* присваиваемое значение: целое число
* поведение: лишние биты *отсекаются (игнорируются)*. То есть происходит деление по модулю на максимальное значение целевого типа.

* целевой тип -- целый тип со знаком (`signed`)
* присваиваемое значение: целое число
* поведение: *зависит от реализации*

* целевой тип: целочисленный
* присваиваемое значение: число с плавающей точкой
* поведение: *не определено*

## Конвертация чисел с плавающей точкой в целочисленные

Если число с плавающей точкой *умещается* в целевой *целочисленный* тип, то выполняется следующая логика:

Поведение* *без понижения* типа:
* целевой тип: целочисленный
* присваиваемое значение: число с плавающей точкой
* поведение: *усечение* (округление в сторону нуля)
