Ключевое слово `const` из языка C++, используется в C (и C++), чтобы *запретить изменение некоторой области памяти*

## Использование `const` с переменными
Переменные, описанные этим модификатором остаются перменными (их просто "типа запрещено" изменять, на самом деле можно)
Пример описания константной переменной с инициализатором:
```c
const int iteration_count = 78;
```
введенное таким образом имя *не является константой* и не может быть использовано там, где требуется значение времени компиляции (например в `case-`метках, при указании размеров массивов и т. д.). Потому  подобные описания с `const` не очень полезные  
> [!NOTE] VLA
> С C99 такие "константы" разрешено использовать для указания размера массива. Его вообще стало возможно задавать выражением, вычисляемым во время исполнения (variable length arrays VLA)
> Рекомендуется не использовать этот функционал

## Использование `const` с указателями
Да, указатель тоже переменная, однако данный случай следует рассмотреть отдельно

Полезно использовать `const` в описаниях и объявлениях указателей:
```c
const int *p;
```
В таком случае описывается *не* константный указатель, а *указатель на константную область памяти*. То есть `const` относится не к переменной `p`, а к области памяти, на которую указывает `p` (при этом сам указатель менять можно)

В таком виде модификатор `const` активно применяется в описаниях параметров функций.
Пример:
```c
void suspicious_func(int *a);
void trustworthy_func(const int *a);
```
В первом случае *можно подозревать*, что функция *что-то сделает* с переданной ей областью памяти.
Во втором случае, *автор* функции *заявляет*, что и*зменять память функция не намерена*

Однако слово `const` не дает гарантий, всегда можно изменить тип адресного выражения на другой (например не предполагающий константности). Скорее `const` следует воспринимать как *обещание*, за которым постарается проследить компилятор

> [!NOTE] Неизменяемый указатель
> Задать неизменяемый указатель тоже можно, при этом нужно сразу же задать ему значение, так как присваивать ему ничего нельзя
> Пример:
> ```c
> int x;
> int * const q = &x;
> ```
> Такая конструкция практически не имеет никакого смысла
