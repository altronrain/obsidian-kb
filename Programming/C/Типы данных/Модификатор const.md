Ключевое слово `const` из языка C++, используется в C (и C++), чтобы *запретить изменение некоторой области памяти*

## Использование `const` с переменными
Переменные, описанные этим модификатором остаются перменными (их просто "типа запрещено" изменять, на самом деле можно обойти ограничение)

Пример описания константной переменной с инициализатором:
```c
const int iteration_count = 78;
```
введенное таким образом имя *не является константой* и не может быть использовано там, где требуется значение времени компиляции (например в `case-`метках, при указании размеров массивов и т. д.).

> [!NOTE] VLA
> С C99 такие "константы" разрешено использовать для указания размера массива. Его вообще стало возможно задавать выражением, вычисляемым во время исполнения (variable length arrays VLA)
> Рекомендуется не использовать этот функционал

## Использование `const` с указателями
Да, указатель тоже переменная, однако данный случай следует рассмотреть отдельно

Полезно использовать `const` в описаниях и объявлениях указателей:
```c
const int *p;
```
В таком случае описывается *не* константный указатель, а *указатель на константную область памяти*. То есть `const` относится не к переменной `p`, а к области памяти, на которую указывает `p` (при этом сам указатель менять можно).

При этом константный указатель допускает присваивание ему адресов как константных, так и неконстантных данных:
```c
double rates[3] = {88.9, 100.12, 59.54};
const double locked[4] = {0.08, 0.075, 0.0725, 0.07};

const double *pc = rates;   /* допустимо */
pc = locked;                /* допустимо */
pc = &rates[3];             /* допустимо */
pc++;                       /* допустимо */
```

Обычному указателю можно присвоить только адрес неконстантных данных:
```c
double rates[3] = {88.9, 100.12, 59.54};
const double locked[4] = {0.08, 0.075, 0.0725, 0.07};

double *pnc = rates;        /* допустимо */
pnc = locked;               /* не допустимо */
pnc = &rates[3];            /* допустимо */
pnc++;                      /* допустимо */
```
Следствем можно считать возможность передачи в функции с *константным* указателем *как константных данных, так и обычных*.

При объявлении константного массива (что суть тоже работа с указателями) мы не можем присваивать его элементам новые значения
```c
const int days[MONTHS] = {31,28,31,30,31,30,31,31,30,31,30,31};

days[9] = 44;  /* не допустимо */
```

При присвоении константному указателю имени массива мы также не сможем менять его элементы, обращаясь через указатель. Однако сможем по оригинальному имени массива:
```c
double rates[3] = {88.9, 100.12, 59.54};
const double *pb = rates;

*pd = 29.89;         /* не допустимо */
pd[2] = 22.22;       /* не допустимо */
rates[0] = 99.99;    /* допустимо */
pd++;                /* допустимо, pd указывает на rates[1] */
```
Собственно, такая же логика при передаче в функции по указателю. См. ниже.

В таком виде модификатор `const` активно применяется в описаниях параметров функций.
Пример:
```c
void suspicious_func(int *a);
void trustworthy_func(const int *a);
```
В первом случае *можно подозревать*, что функция *что-то сделает* с переданной ей областью памяти.
Во втором случае, *автор* функции *заявляет*, что и*зменять память функция не намерена*
Первая функция способна принимать только неконстантные данные в качестве фактических аргументов, а вторая как константные, так и нет.

Однако слово `const` не дает гарантий, всегда можно изменить тип адресного выражения на другой (например не предполагающий константности). Скорее `const` следует воспринимать как *обещание*, за которым постарается проследить компилятор

> [!NOTE] Неизменяемый указатель
> Задать неизменяемый указатель тоже можно, при этом нужно сразу же задать ему значение, так как присваивать ему ничего нельзя
> Пример:
> ```c
> int x;
> int * const q = &x;
> ```
> Такой указатель не может указывать на что-нибудь другое, так как присваивать ему ничего нельзя. Такая конструкция практически не имеет никакого смысла (хотя значение все еще можно менять в ячейке, на которую указатель указывает)
