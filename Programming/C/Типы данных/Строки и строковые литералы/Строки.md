## Строки как массив символов 
Строки имеют непредсказуемый размер и память под них обычно выделяется с некоторым запасом. Тем самым требуется способ указания *текущей длины строки*

В языке C строки представляют собой массив символов, то есть *массив элементов типа* `char`
*Корректной строкой* является только такой массив, в котором хотя бы один *элемент равен* `0`. Такой элемент выступает в роли ограничителя:
* все элементы *до нулевого* -- считаются составляющими строку
* все что находится *после нулевого* -- игнорируется (мусор)

Если нулевых элементов в массиве больше одного, то в качестве ограничителя рассматривается первый из них (с наименьшим индексом)

Длина строки -- разница между положением ограничителя (`0`) и началом строки
Минус подобного подхода -- для определения длины строки ее необходимо просматривать с начала до символа с кодом `0`

## Особенности передачи строк в функции
При передаче строк важно помнить несколько моментов:
* передача строки в функцию идет по адресу первого элемента (аналогично иным массивам), то есть в данном случае по `char *`
* так как компилятор размещает строковые литералы в *неизменяемой памяти*, то следует в вызове функции указать модификатор `const` для массива `char`

Пример передачи строкового литерала в функцию для подсчета длины строки:
```c
int string_length(const char *str)
```
где:
* `str` -- строковый литерал
* `int` -- возвращаемое целочисленное значение

## Примеры функций, обрабатывающих строки

### Подсчет длины строки
В стандартной библиотеке для этого есть функция `strlen`

В C задачу подсчета длины строки удобно решать с помощью цикла `for` (см. [[Programming/C/Контроль хода программы и иные операторы/Циклы#Цикл с конечным количеством итераций|Циклы с конечным количеством итераций]])
Пример:
```c
int string_length(const char *str)
{
	int i;
	for(i = 0; str[i]; i++)
		{}
	return i;
}
```
где `str[i]` выступает в качестве *логического* выражения условия выполнения цикла

Более *оптимизированный* вариант той же функции, который уменьшает общее количество операций *сложения* за счет использования в качестве *переменной цикла не индекса текущего элемента, а указателя на него*:
```c
int string_length(const char *str)
{
	const char *p;
	for(p = str; *p; p++)
		{}
	return p - str;
}
```

Как делать *не следует*:
```c
int striny_length(const char *str)
{
	const char *p = str;
	while(*p++);
	return p - str;
}
```
причины:
* пустое тело цикла `{}` на отдельной строке гораздо заметнее `;`
* побочный эффект загнан в условное выражение (операция `++` меняет `p`, что приводит к ошибке в данном коде)

### Копирование строки
В стандартной библиотеке для этого есть функция `strcpy`

При копировании строк будет два параметра:
* область памяти, куда производится копирование
* область памяти, откуда производится копирование (сам строковый литерал)
Второй параметр не изменяется. Функция ничего не возвращает.
Пример вызова функции:
```c
void string_copy(char *dest, const char *src)
```

Вариант "в лоб":
```c
void string_copy(char *dest, const char *src)
{
	int i;
	for(i = 0; src[i]; i++)
		dest[i] = src[i];
	dest[i] = 0;
}
```
последняя строка нужна, потому что при `src[i] = 0` тело цикла уже не будет выполнено

Оптимизированный пример через цикл `while`:
```c
void string_copy(char *dest, const char *src)
{
	while (*src) {
		*dest = *src;
		dest++;
		src++;
	}
	*dest = 0;
}
```
Оптимизированный пример через цикл `for`:
```c
void string_copy(char *dest, const char *src)
{
	for(; *src; dest++, src++)
		*dest = *src;
	*dest = 0;
}
```
