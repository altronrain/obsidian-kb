## Обычная обработка строковых литералов
Зная, как устроены строки в C, можно понять *последовательность действий компилятора* при виде строки в двойных кавычках. 
А именно, компилятор:
* отводит в неизменяемой области памяти (чаще всего в секции `.text`, см. [[Programming/NASM/Структура программы#Секции исполняемого файла|Секции исполняемого файла]]) нужное количество памяти по одному байту на символ + 1 байт на завершающий `0`
  Неизменяемая область выбирается из соображений эффективности
* разместив содержимое строкового литерала в генерируемом объектном модуле, компилятор подставляет вместо него *адрес первого элемента* с модификатором `const`:
  `const char *`

## Использование строковых литералов для инициализации массивов
Помимо этого, имеется иной способ применения/обработки строковых литералов -- *для инициализации массивов* типов `char, signed char, unsigned char`
В таком случае компилятор не станет размещать массив в неизменяемой памяти (даже с модификатором `const`)
Пример:
```c
char str[6] = "Hello";
```
что аналогично записи:
```c
char str[6] = {'H', 'e', 'l', 'l', 'o', 0 };
```
число в обоих случаях можно было опустить

При этом необходимо понимать, что выражение:
```c
char str[] = "Hello";
```
* описывает и инициализирует массив из `6` элементов
* `str` переменной не является
* массив не помещается в неизменяемую область памяти
* `sizeof(str)` выдаст размер области памяти, занятый массивом
* полученный массив можно изменять, так `str[4] = 0` превратит слово в `"Hell"`
При этом выражение:
```c
char *ptr = "Hello";
```
* описывает переменную типа "указатель"
* переменная инициализируется адресом строкового литерала
* сам литерал обрабатывается компилятором согласно общим правилам (находится в неизменяемой области памяти)
* указателю в дальнейшем можно присвоить другой адрес
* `sizeof(str)` выдаст размер указателя (4 или 8 в зависимости от разрядности)
* попытка изменить строковый литерал через указатель аварийно завершит программу (`ptr[4] = 0`)
