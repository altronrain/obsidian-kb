Структуры в C представляют собой *составной тип данных* и предполагают, что переменная этого типа состоит из *нескольких переменных других типов* (*разных*, что отличает структуры от массивов).
Элементы составного типа (переменные) называются *полями*

Структуры являются аналогом записей в Pascal (см. [[Запись]]). Также их не стоит путать с перечислениями enum (см. [[Перечислимый тип]])

## Описание структур
Структуры описываются ключевым словом `struct`, а подход описания похож с перечислимым типом:
* сначала ключевое слово
* необязательное имя (идентификатор), вместе с ключевым словом образующие *имя типа*
* в фигурных скобках информация о типе (перечисление полей)
* список переменных (необязателен)
То есть:
```c
struct [<имя>] {<список_полей>} [<список_переменных>]
```
Пример описания структуры:
![[Pasted image 20250301124223.png]]
Где `student` есть *имя структуры, которое не является именем типа*. Тип имеет имя `struct student`
Так, описание переменной этого типа будет выглядеть как:
```c
struct student st1;
```

Для доступа к элементам структуры используется точка (`.`). Например `st1.name`

Переменные структурного типа можно инициализировать при их описании. Однако подобные конструкции допустимы только при *инициализации* а *не при присваивании*
Присваивать переменные друг другу можно, в отличие от массивов. 
Можно и передавать в функции по значению, как и возвращать из функций (однако лучше так не делать)
> [!NOTE] Пространство имен
> В C структуры образуют отдельное пространство имен, так что можно использовать такие же идентификаторы для чего-нибудь еще. Однако не рекомендуется, так как например в C++ ситуация противоположная

## Размер структуры. Выравнивание
Размер структуры может оказаться *больше* чем суммарный размер всех составляющих ее элементов. 

Переменные, занимающие больше одного байта (точнее больше одной ячейки памяти) на многих *аппаратных архитектурах* могут начинаться только с четных адресов (например Sparc процессоры), либо выполняют эти операции значительно медленнее чем с четных (Intel).

Компилятор учитывает этот факт и вставляет в переменную структурного типа *выравнивающие байты* (padding bytes). В них не будет содержаться никакая информация, задача выравнивающих байтов -- занимать место таким образом, чтобы очередное поле начиналось с адреса кратного 2 или 4.
Этот процесс называется *выравниванием* (alignment). В общем случае мы не можем знать, сколько ячеек памяти будет занимать переменная структурного типа, кроме как применив `sizeof()`
Пример динамического выделения памяти под структуру:
```c
struct student *ptr;
ptr = malloc(sizeof(struct student));
```
Так как `sizeof` работает не только для имен типов, но и для произвольных выражений, выделение памяти можно переписать короче:
```c
ptr = malloc(sizeof(*ptr));
```

## Указатели на структуры. Обращение к полям структуры
Если требуется через указатель (а они используются почти всегда в C) обратиться к полю структуры, то сделать это выражением `*p.field` не получится.

Символ точки сам по себе не операция, а вот `.field` -- операция извлечения из поля `field`
Эта операция унарная и имеет суффиксную форму. Суффиксные унарные операции имеют приоритет над префиксными (см. [[Приоритет операций]])
Следовательно в записи `*p.field` будет получена ошибка, так как `.year` будет выполняться от указателя `p`, у которого нет полей.

Корректной формой будет запись вида `(*p).year`, но такая запись довольно громоздкая.
По этому была придумана операция *стрелка*
```c
p->field
```
то же самое что:
```c
(*p).field
```
