Синтаксис структур используется для описания *битовых полей*, а точнее -- для анализа и синтеза целых чисел, состоящих из отдельно рассматриваемых битов и групп битов.

Для целочисленных полей структуры язык C позволяет в явном виде указать количество битов, которое данное поле будет занимать. Как правило такие поля это беззнаковые (`unsigned int` или просто `unsigned`). Количество бит указывается сразу после имени поля через двоеточие
Например:
```c
struct flags {
	unsigned io_error:1;
	unsigned seen_a_digit:1;
	unsigned had_a_eol:1;
	unsigned signaled:1;
	unsigned count:4;
}
```
структура состоит из 4 именованных флагов, которые могут быть в значении `0` или `1`, и некоего счетчика на 4 бита (значения от `0` до `15`)

Подобное имеет смысл только если несколько битовых полей расположены подряд *друг за другом*. Если смешивать с полями других типов, то будет иметь место *выравнивание* (см. [[Структуры#Размер структуры. Выравнивание]]) Так, изначально в стандарте языка смешивать с другими типами вообще было нельзя. 

Битовые поля изначально были предназначены, чтобы разбить на отдельные биты *машинное слово*. Описывать эти поля можно только типами `int` или `unsigned int` (лучше второе) и тут многое зависит от компилятора. 
Группа идущих подряд битовых полей всегда занимает *целое число слов*. Под словом понимается столько байт, сколько занимает `int`, то есть структура выше будет занимать 4 байта.

Компилятор `gcc` позволяет описывать поля типов `char`, что является *расширением языка*. В иных компиляторах она *не поддерживается*.

Вопрос применения битовых полей является спорным. Многие считают что достаточно побитовых операций и битовые поля не нужны.
