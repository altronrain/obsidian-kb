## Основные понятия
В императивных языках программирования высокого уровня *переменная* на уровне машинного кода представляет собой *область памяти* (несколько ячеек памяти, расположенных подряд -- имеющих *последовательные* адреса)

*Адресом области памяти* является *наименьший* из адресов ячеек, составляющих область

*Указатель* -- это *переменная*, в которой хранится адрес

Утверждение вида "A указывает на B" означает "A содержит адрес B"

## Описание переменных-указателей
### Типизированные указатели
В С *адресный тип привязан к типу переменной*, адрес которой имеется ввиду.
Адресный тип описывается с помощью символа звездочки: 
* `int *`  -- адрес `int`
* `double *` -- адрес `double`

Пример описания переменных-указателей:
```c
int *p;
double *q;
```

При этом `*` хоть и имеет отношение к типу, воспринимается как атрибут описываемого имени, а не часть типа. По этому для множественного описания указателей используется следующий синтаксис:
```c
char *s1, *s2, *s3;
```
При том что типом является `char *`

При записи как
```c
char* s1, s2, s3;
```
тип `char*` получит только `s1`

Между собой присваивание отработает только между указателями на одинаковые типы:
```c
int n = 5;
double x;
int *p1 = &n;
double *pd = &x;

x = n;   /* неявное преобразование типа плохо, но со значениями отработает */
pd = p1; /* ошибка на этапе компиляции */
```

### Указатели на многомерные массивы
См. также [[Programming/C/Типы данных/Массивы#Многомерные массивы|Многомерные массивы]]

Чтобы создать типизированный указатель на массив из *n* элементов (что может быть полезно при работе с многомерными массивами) необходимо использовать следующее объявление указателя:
```c
int (*pz)[2];
```
Круглые скобки нужны, так как приоритет `[]` выше чем у `*`.
Без скобок мы создадим массив из двух указателей на `int`, а со скобками -- указатель на массив из 2 значений `int`.
Подобный указатель можно применять вместо имени массива, имеющего такую же структуру.

При этом такой указатель на `int[2]` это не то же самое что `int **ptr` (указатель на указатель типа `int`), так как количество элементов разное.

Общее правило преобразования адресации "через элементы массива" и "через указатели" одинакого как для имени массива так и для подобного рода указателей и выглядит следующим образом:
```c
zippo[m][n] == *(*(zippo + m) + n)
pz[m][n] == *(*(pz + m) + n)
```
По поводу присваивания указателей между собой, сохраняется требование совпадения типов и в подобных случаях c указателями на сложные массивы:
```c
int *pt;
int (*pa)[3];
int arr1[2][3];
int arr2[3][2];
int **p2;

pt = &arr1[0][0]; /* допустимо, оба -- указатели на int */
pt = &arr1[0];    /* допустимо, оба -- указатели на int */
pt = arr1;        /* недопустимо arr1 -- массив массивов int[3] */
pa = arr1;        /* допустимо, оба -- указатели на int[3] */
pa = arr2;        /* недопустимо, arr2 -- массив массивов int[2] */
pt = &pt;         /* допустимо, оба -- указатели на int */
*p2 = arr2[0];    /* допустимо, оба -- указатели на int */
p2 = arr2;        /* недопустимо, arr2 -- массив массивов int[2] */
```

### Нетипизированные (универсальные) указатели
В C можно работать с адресами не уточняя тип переменных, на которые эти адреса будут указывать. 
Соответствующий тип указателя будет называться:
```c
void *z;
```
Так:
* в переменную `z` можно занести *любой* адрес
* любому типизированному указателю можно присвоить нетипизированный адрес `z`


## Сложность многократного разыменования
Очень легко допустить ошибку при работе с указателями на указатели (многократные уровни *косвенности*). Особенно, если тут еще замешан модификатор `const` (см. [[Модификатор const]])

Напоминание:
```c
const int *p;
```
не константный указатель -- *указатель на константную область памяти*

Примеры:
```c
int x = 20;
const int y = 23;
int *p1 = &x;
const int *p2 = &y;
const int **pp2;

p1 = p2; /* небезопасно -- присваивание константного значения неконстантному */
p2 = p1; /* допустимо -- присваивание неконстантного значения константному */
pp2 = &p1 /* небезопасно -- присваивание вложенных типов указателей */
```
1. Мы присваиваем адрес из константного указателя в неконстантный. Это вызовет максимум *предупреждение* компилятора, так можно. Но мы более не гарантируем что значение по адресу `&y` не будет изменено.
2. Мы присваиваем адрес из неконстантного указателя в константный. Так можно. Таким образом далее в коде мы гарантируем, что по адресу через данный указатель значение не будет изменено.

Последний случай требует отдельного пояснения. Пример:
```c
const int **pp2;
int *p1;
const int n = 13;

pp2 = &p1; /* разрешено, но p1 сам по себе без const */
*pp2 = &n; /* допустимо, оба const, но из-за строки выше p1 теперь указывает на n */
*p1 = 10;  /* допустимо, однако мы пытаемся изменить константу n. UB */
```
В последней строке результат выполнения не определен (UB).

## Идентификатор `NULL`
Библиотека C предусматривает специальное обозначение для адресного значения, гарантированно не являющегося адресом какой-либо ячейки или области памяти (скрывая нюансы архитектуры и компиляторов, так как значения для такого указателя может разниться между ними)

Таким обозначением является идентификатор `NULL`, который всегда равен соответствующему значению для указанного компилятора

В Pascal для этих целей был предусмотрен идентификатор `nil`

Значение адресов можно использовать в качестве логических значений, при этом:
* `NULL` является "ложью"
* любой другой адрес является "истиной"
