## Основные понятия
В императивных языках программирования высокого уровня *переменная* на уровне машинного кода представляет собой *область памяти* (несколько ячеек памяти, расположенных подряд -- имеющих *последовательные* адреса)

*Адресом области памяти* является *наименьший* из адресов ячеек, составляющих область

*Указатель* -- это *переменная*, в которой хранится адрес

Утверждение вида "A указывает на B" означает "A содержит адрес B"

## Описание переменных-указателей
### Типизированные указатели
В С *адресный тип привязан к типу переменной*, адрес которой имеется ввиду.
Адресный тип описывается с помощью символа звездочки: 
* `int *`  -- адрес `int`
* `double *` -- адрес `double`

Пример описания переменных-указателей:
```c
int *p;
double *q;
```

При этом `*` хоть и имеет отношение к типу, воспринимается как атрибут описываемого имени, а не часть типа. По этому для множественного описания указателей используется следующий синтаксис:
```c
char *s1, *s2, *s3;
```
При том что типом является `char *`

При записи как
```c
char* s1, s2, s3;
```
тип `char*` получит только `s1`

### Нетипизированные (универсальные) указатели
В C можно работать с адресами не уточняя тип переменных, на которые эти адреса будут указывать. 
Соответствующий тип указателя будет называться:
```c
void *z;
```
Так:
* в переменную `z` можно занести *любой* адрес
* любому типизированному указателю можно присвоить нетипизированный адрес `z`

## Операции над указателями
Основные операции:
* операция взятия адреса -- `&`
* операция разыменования -- `*`

Пример работы с указателями:
```c
int x;    /* переменная x типа int */
int *p;   /* указатель на тип int p */

p = &x;   /* занесение адреса x в указатель p */
*p = 27;  /* *p -- синоним x. занесение 27 в переменную x  */
```
Пример для пояснения постановки символа `*`:
```c
int x, y;
int *p, *q, *r;

x = 25;
y = 36;
p = &x;
q = &y;
```
Тогда:
```c
r = p;
```
есть присвоение переменной `r` значения переменной `p`, то есть в `r` будет находиться *адрес* переменной `x`
В то время как:
```c
*r = *q;
```
есть присвоение переменной `x` значения переменной `y`, то есть числа `36`. Сами указатели при этом не изменяются

## Идентификатор `NULL`
Библиотека C предусматривает специальное обозначение для адресного значения, гарантированно не являющегося адресом какой-либо ячейки или области памяти (скрывая нюансы архитектуры и компиляторов, так как значения для такого указателя может разниться между ними)

Таким обозначением является идентификатор `NULL`, который всегда равен соответствующему значению для указанного компилятора

В Pascal для этих целей был предусмотрен идентификатор `nil`

Значение адресов можно использовать в качестве логических значений, при этом:
* `NULL` является "ложью"
* любой другой адрес является "истиной"
