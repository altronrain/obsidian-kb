## Передача по значению

В C существует *только один* способ передачи параметров -- *по значению*

В ситуации, когда требуется передать *переменную* в функцию, приходится передавать *адрес переменной*, а в теле функции использовать разыменование для обращения к переменной

Пример функции, меняющей значение двух переменных типа `int`:
```c
void swap(int *a, int *b)
{
	int t;
	t = *a;
	*a = *b;
	*b = t;
}
```
Вызов такой функции будет выглядеть следующим образом:
```c
swap(&x, &y);
```
На время работы функции в ее *локальных* переменных `a` и `b` окажутся *адреса* переменных `x` и `y`. Выражения `*a` и `*b` станут соответственно обозначать переменные `x` и `y`

## О передаче фундаментальных типов и массивов

Данные типов вроде `int` можно передать в функцию двумя способами -- по значению *переменной*, и передачей *указателя* на тип `int` (тоже по значению, как указано пунктом выше).

Передача *по значению переменной* -- обычно применяется, если программа не нуждается в изменении этого значения в *исходной функции*.

Передача *по адресу переменной* -- обычно применяется, если требуется изменить значение переменной из/в *исходной функции*, вернув его туда потом.

Как легко заметить, в C с массивами альтернативы нет. Мы в любом случае передаем его в функцию по адресу первого элемента через указатель (для быстродействия).
То есть мы *всегда работаем* в вызываемой функции с *исходным массивом*, а не с его копией. Для защиты исходных массивов от изменений, например, применяется ключевое слово-модификатор `const` в определении и прототипе вызываемой функции-обработчика.
```c
int sum (const int arr[], int n);
```
Подобный прототип (и объявление) сообщают, что функция `sum` обещает не модифицировать исходный массив `arr`. Если попытаемся изменить внутри функции содержимое массива -- компилятор выдаст ошибку.

При этом *не требуется* чтобы исходный массив был объявлен константным с данным модификатором.
