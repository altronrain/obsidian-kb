Исходя из названия класса такие переменные имеют:
* область видимости в пределах файла
* внешнее связывание
* статическую продолжительность хранения

Также данный класс называют *внешним классом хранения*, а переменные -- *внешними переменными*. Такие переменные могут применяться в *любом файле*, являющемся частью программы.

## Объявление внешних переменных
Объявляется такая переменная:
* за пределами всех функций (чаще всего)
* внутри функции, в которой используется (с ключевым словом `extern`) (дополнительное необязательное объявление)
* за пределами всех функций с ключевым словом `extern` (обязательно, если переменная определена в другом файле)

Примеры:
```c
int num;                /* внешне определенная переменная */
double up[100];         /* внешне определенный массив */
extern char letter;     /* обязательное объявление, если */
						/* letter определена в другом файле */
int main (void)
{
	extern int num;     /* необязательное объявления внешних */
	extern double up[]; /* переменных внутри функции */
	...                 /* размерность массива можно не указывать уже */
}
```

Ключевое слово`extern` внутри `main` можно было бы и опустить, но это служит для:
* демонстрации намерения
* для предотвращения замещения

 Иначе объявление `int num` создаст *автоматическую локальную* переменную в пределах функции `main`, которая *заместит* внешнюю. А в *других* функциях будет использоваться *исходная внешняя* `num`.

> [!NOTE] Место объявления играет роль
> Если объявить переменную посреди файла между функциями, то она будет известна только тем функциям, что следуют *после объявления*.

## Инициализация внешних переменных
Внешние переменные инициализируются *нулями*, если не инициализированы явно. В том числе и все элементы внешнего массива будут нулями.
См. [[Programming/NASM/Структура программы|Структура программы]] и [[Память пользовательской задачи. Сегменты]]

Для инициализация переменных с областью видимости в пределах файла можно применять только константные выражения. Нельзя использовать *переменные*, что отличает данный класс от автоматических переменных.

Примеры:
```c
int    x = 10;               /* ок. константа */
int    y = 3 + 20;           /* ок. константное выражение */
size_t z = sizeof (int);     /* ок. константное выражение */
int    w = 2 * x;            /* недопустимо. использование переменной */
```

## Объявление переменных и определение
Иная тонкость заключается в разнице между *определением* и *объявлением*.
Пример:
```c
int num = 1;          /* определение переменной num */

int main (void)
{
	extern int num; /* использование num, определенной где-то в другом месте */
	...
}
```
Оба выражения в листинге явялются *объявлениями* переменной `num`.
Однако лишь одно из них является *определением*.

Первое приводит к тому, что под переменную отводится место в памяти. Это *определение* переменной. *Определяющее объявление*.

Второе просто указывает компилятору на необходимость применения переменной, созданной ранее. Это не определение. Это *ссылочное объявление*.

Ключевое слово `extern` сообщает компилятору, что объявление не является *определением*. Это указание компилятору искать определение переменной *в другом месте*. 

Именно по этому не стоит использовать `extern` для *определения* переменных. Память под них не выделяется. Как не стоит и пытаться инициализировать переменные с данным ключевым словом:
```c
extern char yes = 'Y';   /* ошибка на этапе компиляции */
```
