* Область видимости: В пределах блока
* Связывание: Отсутствует
* Продолжительность хранения: Автоматическая

Также называются локальными.
## Инициализация переменных
Автоматические переменные не инициализируются, если этого не сделать явно, так как память под них выделяется динамически. Если переменную не проинициализировать -- там будет мусор.

## Поведение переменных во вложенных блоках и сокрытие
Переменная определена с момента входа в блок (за исключением массивов переменной длины) до момента выхода из блока. Доступна по имени только внутри  данного блока (косвенными средствами значение или адрес можно передать наружу).

При наличии вложенных блоков, переменная известна и доступна в блоке, в котором она объявленна, и всех нижележащих блоках:
```c
int loop (int n)
{
	int m;             /* m в области видимости */
    scanf ("%d", &m);
    {
        int i;         /* m и i в области видимости */
        for (i = m; i < n; i++)
            puts ("i is local to a sub-block");
    }
    return m;          /* m в области видимости, i исчезла */
}
```

Если внутри блока объявить переменную с таким же именем, как переменная снаружи, то конкретно внутри блока будет использоваться местная переменная, а наружняя будет тем самым *скрыта*. Так, например, в цикле `while` используя инкрементацию в условии, можно написать следующее:
```c
int x = 30;
printf ("x во внешнем блоке: %d по адресу %p\n", x, &x);
while (x++ < 33) /* исходная переменная x (важна инкрементация тут) */
{
    int x = 100; /* новая переменная x, скрывающая первую */
	x++;         /* инкрементация новой переменной */
    printf ("x в цикле while: %d по адресу %p\n", x, &x);
}
printf ("x во внешнем блоке: %d по адресу %p\n", x, &x);
```
Что приведет к выводу:
```
x во внешнем блоке: 30 по адресу 0x7fffe44af34c
x в цикле while: 101 по адресу 0x7fffe44af344
x в цикле while: 101 по адресу 0x7fffe44af344
x в цикле while: 101 по адресу 0x7fffe44af344
x во внешнем блоке: 34 по адресу 0x7fffe44af34c
```
Именовать переменные в таком стиле не следует никогда. Важен факт демонстрации.

То же верно и для расширения C99 с блоками без фигурных скобок для `for`, `while`, `if`.
С той лишь оговоркой, что тело цикла (будь оно со скобками или нет) -- подблок полного блока цикла (в который входит и объявление):
```c
int n = 8;

printf ("Первоначально n = %d по адресу %p\n", n, &n);
for (int n = 1; n < 3; n++)
	printf ("Цикл 1: n = %d по адресу %p\n", n, &n);
printf ("После цикла 1 n = %d по адресу %p\n", n, &n);
for (int n = 1; n < 3; n++)
{
	printf ("Индекс цикла 2: n = %d по адресу %p\n", n, &n);
	int n = 6;
	printf ("Цикл 2: n = %d по адресу %p\n", n, &n);
	n++;
}
printf ("После цикла 2 n = %d по адресу %p\n", n, &n);
```
Выведет:
```
Первоначально n = 8 по адресу 0x7ffc8abd40ec
Цикл 1: n = 1 по адресу 0x7ffc8abd40e8
Цикл 1: n = 2 по адресу 0x7ffc8abd40e8
После цикла 1 n = 8 по адресу 0x7ffc8abd40ec
Индекс цикла 2: n = 1 по адресу 0x7ffc8abd40e4
Цикл 2: n = 6 по адресу 0x7ffc8abd40e0
Индекс цикла 2: n = 2 по адресу 0x7ffc8abd40e4
Цикл 2: n = 6 по адресу 0x7ffc8abd40e0
После цикла 2 n = 8 по адресу 0x7ffc8abd40ec
```

* переменная `n` из первого объявления цикла `for` скрывает исходную
* переменная `n` из второго объявления цикла `for` скрывает исходную
* переменная `n` внутри тела цикла скрывает индекс цикла `for` (исчезает при выходе из тела, на момент подсчета индекса "восстанавливается" переменная из объявления цикла)

## Применение ключевого слова `auto`
Чтобы обозначить намерение, можно специально указать для переменной, что она относится к данному классу хранения, используя ключевое слово `auto`:
```c
int main (void)
{
    auto int plox;
...
}
```

> [!NOTE] Ключевое слово `auto`
> В C данное ключевое слово является спецификатором класса хранения.
> В C++ ключевое слово `auto` предназначено совершенно для другого.
