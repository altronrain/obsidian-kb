Краткая сводка по ключевым словам спецификаторам в однром месте.

В языке C имеется 6 ключевых слов, которые сгруппированы вместе как *спецификаторы классов хранения*:
* `auto`
* `register`
* `static`
* `extern`
* `_Thread_local`
* `typedef`

Иногда их интерпретация и значение может зависеть от контекста (как, к примеру `static` и `extern`).

В большинстве случаев нельзя использовать *более одного* спецификатора класса хранения. Исключением является `_Thread_local`, который можно исполдьзовать вместе со `static` и `extern`.

Ключевое слово `typedef` ничего не говорит о хранении в памяти, но относится к данной группе по синтаксическим причинам. Как раз потому что не получится создать alias для спецификатора классов хранения (?)

## Краткий обзор спецификаторов

Спецификатор `auto`:
* указывает переменную с автоматической продолжительностью хранения
* применяется в объявлении переменных с областью видимости в пределах блока
* служит по большей части для документирования

Спецификатор `register`:
* применяется в объявлении переменных с областью видимости в пределах блока
* указание-просьба поместить переменную в регистровый класс хранения (компилятор не обязан следовать указанию)
* запрещает операцию взятия адреса переменной, к которой применен
* в остальном идентична переменным с амтоматическим классом хранения

> [!NOTE] Modern C programming
> В настоящее время класс является скорее *deprecated* и его использование строго не рекомендуется. Компилятор сам оптимизирует код лучше среднестатистического программиста.

Спецификатор `static`:
* создает объект со статической продолжительностью хранения (появляется после загрузки программы в память и исчезает при завершении программы)
* при объявлении внутри блока область видимости переменной ограничена блоком
* при объявлении в файле вне блока указывает на внутреннее связывание а не на продолжительность хранения как таковую

Спецификатор `extern`:
* указывает, что объявляется переменная, которая была определена где-то в другом месте
* при объявлении переменной с областью видимости в пределах файла, такая переменная *должна* иметь внешнее связывание (и быть определена в другой единице трансляции)
* при объявлении переменной в пределах блока, такая переменная может иметь либо внешнее, либо внутреннее связывание.

## Различия автоматической и статической продолжительности хранения

Для переменных с автоматической продолжительностью хранения:
* память выделяется на момент входа в блок, где объявлена такая переменная
* память освобождается при выходе из блока
* неинициализорованные переменные получают случайное значение

Для переменных со статической продолжительностью хранения:
* память выделяется на этапе компиляции программы
* память сохраняется на все время выполнения программы
* неинициализоваронные переменные плдучают значение 0
