*Представляет* собой скорее "запрос", нежели указание к исполнению. Переменные такого класса хранения мы *хотим хранить в регистрах* CPU, как самой быстрой памяти, которая может быть доступна. Получить *адрес* такой переменной *не удастся* (эта операция запрещена для такого класса хранения).

В остальном не отличаются от автоматического класса хранения:
* Область видимости: В пределах блока
* Связывание: Отсутствует
* Продолжительность хранения: Автоматическая

Для объявления подобных переменных используется ключевое слово `register`:
```c
int main (void)
{
    register int quick;
...
}
```

В таком случае компилятор постарается выполнить запрос и разместить переменную в регистре, если хватит ресурсов (собственно, регистров). Но если не сможет, то переменная будет идентичной [[Автоматический]] классу хранения, за исключением того, что к ней все еще нельзя нельзя применять операцию взятия адреса.

Часто можно встретить подобные запросы в прототипах функций и заголовках:
```c
void somefunc (register int n);
```

> [!NOTE] Типы поддерживаемых переменных
> Не все типы переменных могут поддерживать данный класс хранения, так как они попросту могут не влезть в регистр. Особенно аккуратно нужно действовать с числами с плавающей точкой (`double` и `long double`)


> [!NOTE] Modern C programming
> В настоящее время класс является скорее *deprecated* и его использование строго не рекомендуется. Компилятор сам оптимизирует код лучше среднестатистического программиста.
