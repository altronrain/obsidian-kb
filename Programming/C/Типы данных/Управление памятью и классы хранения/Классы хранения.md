## Объекты в C
Данные хранятся в памяти, что обеспечивается аппаратным аспектом. Любое сохраненное значени будет находиться в каком-то участке физической памяти. Такой участок памяти в языке C называется *объектом*.

Объект может хранить одно или большее количество значений, может не содержать сохраненного значения, но иметь правильный размер под хранение такого значения.

Программный аспект -- способ доступа к объекту. Что может подойти для обеспечения доступа к объекту:
* *идентификатор* (имя переменной)
* некий адрес или адресное выражение
* массив с объектами
см. [[Леводопустимые выражения]]

К характеристикам объекта и идентификатора объекта (l-value) можно отнести:
* объект: продолжительность хранения (насколько долго он остается в памяти)
* идентификатор: область видимости
* идентификатор: связывание

Последние два вместе указывают, в какой части программы разрешено применять идентификатор объекта.

В языке C представлены различные *классы хранения*, обеспечивающие различный набор характеристик по указанным 3 характеристикам. Сначала следует разобрать их.

## Область видимости
Описывает участок (или участки) программы, где можно обращаться к идентификатору.
Переменная в C имеет одну из следующих *областей видимости* от меньшей к большей:
* в пределах блока
* в пределах функции
* в пределах прототипа функции
* в пределах файла

В указанных пределах переменная определена с момента объявления до конца указанного фрагмента программы.

### Видимость пределах блока
К ней относятся:
* формальные параметры функции (хоть они и до `{}`)
* переменные, определенные внутри любого блока `{}`

> [!NOTE] C99 расширение
> В стандарте C99 стало можно:
> * объявлять переменные в любом месте внутри блока (а не только в его начале)
> * понятие блока было расширено на конструкции управляющего кода циклами (`for`, `while`, `do while`) и ветвлениями `if`, даже если они однострочные без `{}`)

### Видимость в пределах функции
Применяется только к *меткам*, применяемым с операторами `goto`.
Если метка впервые появляется во внутреннем блоке функции, ее область видимости простирается *на всю* функцию. Это предотвращет путаницу при попытке использовать одинаковое имя метки внутри двух отдельных блоков.

### Видимость в пределах прототипа функции
Применяется к именам переменных, используемых в прототипах функций. Распространяется от места определения переменной до конца объявления прототипа.

В данном случае компилятор будет интересовать только тип аргумента, а имена роли не играют (кроме расширения C99 с VLA, где имена играют небольшую роль)

### Видимость в пределах файла
Переменные, объявленные за рамками любых функций, имеют *область видимости в пределах файла*.
Такая пременная видна от места ее определения до конца файла, который содержит ее определение. Такие переменные также называются *глобальными*.

## Связывание
Переменная в C имеет одно из следующих *связываний*:
* внешнее связывание
* внутреннее связывание
* отсутствие связывания

Переменные с областью видимости *в пределах блока, функции и прототипа функции* *не имеют связывания*. Они являются закрытыми для блока, (функции, прототипа), в котором определены.

Переменная с *областью видимости в пределах файла* может иметь либо *внутреннее*, либо *внешнее* связывание.
* переменная с *внешним* связыванием может применяться в *любом месте многофайловой программы*
* переменная с *внутренним* связыванием может применяться в *любом месте одной единицы трансляции* (в которой определена) (см. [[Единицы трансляции и файлы]])

Разница в типе связывания определяется наличием или отсутствием  спецификатора класса хранения `static`:
```c
int giants = 5; /* внешнее связывание, глобальная область видимости */

static int halflings = 3; /* внутреннее связывание, область видимости в пределах единицы трансляции */
```
Так, `giants` может применяться и в других файлах, являющихся составными частями той же программы. В то время как `halflings` может применяться в любом месте данного конкретного файла.

> [!NOTE] Альтернативные наименования областей видимости
> В стандарте используются следующие определения для описания области видимости:
> * ограниченной одной единицей трансляции -- "область видимости в пределах файла с внутренним связыванием"
> * потенциально распространяемой на другие единицы трансляции -- "область видимости в пределах файла с внешним связыванием"
> 
> В обиход вошли определения:
> * "область видимости в пределах файла" (для областьи видимости в пределах файла с внутренним связыванием)
> * "глобальная область видимости" или "область видимости в пределах программы" (для области видимости в пределах файла с внешним связыванием)

## Продолжительность хранения
Характеризует постоянство объектов, доступных через идентификаторы (их время жизни/существования).

Объект в C имеет одну из следующих 4 продолжительностей хранения:
* статическую
* потоковую
* автоматическую
* выделенную

### Статическая продолжительность хранения
Статические объекты существуют н*а протяжении всего времени выполнения программы*.
Такими, например, являютстя *все* переменные с областью видимости в пределах файла.

> [!NOTE] Нюансы `static`
> Ключевое слово `static` для переменных с областью видимости в пределах файла
> отвечает за *связывание*

### Потоковая продолжительность хранения
Важна в *параллельном* программировании, когда выполнение программы может быть разделено на несколько потоков.
Объект с потоковой продолжительностью хранения существует с момента его объявления и до завершения потока.

Такой объект создается, когда объявление объекта с областью видимости *в пределах файла* модифицировано с помощью ключевого слова `_Thread_local`.
Каждый поток получает свою собственную закрытую копию этой переменной.
