Характеристика идентификатора объекта.

Переменная в C имеет одно из следующих *связываний*:
* внешнее связывание
* внутреннее связывание
* отсутствие связывания

Переменные с областью видимости *в пределах блока, функции и прототипа функции* *не имеют связывания*. Они являются закрытыми для блока, (функции, прототипа), в котором определены.

Переменная с *областью видимости в пределах файла* может иметь либо *внутреннее*, либо *внешнее* связывание.
* переменная с *внешним* связыванием может применяться в *любом месте многофайловой программы*
* переменная с *внутренним* связыванием может применяться в *любом месте одной единицы трансляции* (в которой определена) (см. [[Единицы трансляции и файлы]])

Разница в типе связывания определяется наличием или отсутствием  спецификатора класса хранения `static`:
```c
int giants = 5; /* внешнее связывание, глобальная область видимости */

static int halflings = 3; /* внутреннее связывание, область видимости в пределах единицы трансляции */
```
Так, `giants` может применяться и в других файлах, являющихся составными частями той же программы. В то время как `halflings` может применяться в любом месте данного конкретного файла.

> [!NOTE] Альтернативные наименования областей видимости
> В стандарте используются следующие определения для описания области видимости:
> * ограниченной одной единицей трансляции -- "область видимости в пределах файла с внутренним связыванием"
> * потенциально распространяемой на другие единицы трансляции -- "область видимости в пределах файла с внешним связыванием"
> 
> В обиход вошли определения:
> * "область видимости в пределах файла" (для областьи видимости в пределах файла с внутренним связыванием)
> * "глобальная область видимости" или "область видимости в пределах программы" (для области видимости в пределах файла с внешним связыванием)
