## Описание массивов
В языке С массивов *как таковых* нет. 

Пример описания массива:
```c
int m[20];
```
описывает массив из `20` элементов типа `int`. Элементы доступны с помощью операции *индексирования* под номерами (индексами) от `0` до `19`

Элементы массива доступны *каждый в отдельности*, а *к самому массиву* как единому целому *обратиться невозможно* (в следствии чего в C отсутствует присваивание массивов, в том числе *строк*, как частного случая массивов)

*Введенное имя* `m` практически во всех ситуациях *обозначает* не сам массив как таковой, а *адрес* его *первого элемента*. Так как элементы имеют тип `int`, имя `m` представляет собой *константу* типа `int*`

Так, определив указатель:
```c
int *p;
```
можно выполнить присваивание 
```c
p = m;
```
и обращаться к элементам массива с помощью индексирования как через `p`, так и через `m`:
```c
p[0], p[1], p[2], ... p[19]
m[0], m[1], m[2], ... m[19]
```


> [!NOTE] Когда массив соответствует всему массиву
> Единственной осмысленной ситуацией, в которой имя массива соответствует всему массиву как единому объекту -- это применение к имени массива псевдофункции `sizeof(m)`
> Что для примера `m[20]` выдаст результат `80`
> 
> Еще пару ситуаций пред предусматривают "стандарты", но это сложно объяснить и для самостоятельного изучения

## Индексирование
При этом, исходя из теории про адресацию в C, к первому элементу массива (`m[0]`) можно также обратиться как `*m`

Индексирование -- операция не над массивами (над массивами нет операций), а над *адресами*

В C к типизированному адресному выражению (любому кроме `void*`) можно *прибавить целое число*, при этом *адрес изменяется на размер элемента* того *типа*, на который указывает исходный адрес.

Адрес типа `char*` при добавлении к нему `1` увеличится на `1` (1 байт)
Адрес типа `int*` при добавлении к нему `1` увеличится на `4` (4 байта)

Таким образом, если в памяти размещен массив элементов одного типа,  добавление единицы к адресу одного из этих элементов даст адрес следующего элемента массива

Это и выполняет операция индексирования
Пример:
```c
int m[20];
m     /* адрес первого элемента */
m+1   /* адрес второго элемента */
m+19  /* адрес последнего элемента */
```
Это *адресная арифметика*. Если выражения вида `m, m+1, ..., m+19` являются адресами, то их можно разыменовывать:
```c
*m, *(m+1), *(m+2), ..., *(m+19)
```
то же самое что:
```c
m[0], m[1], m[2], ..., m[19]
```

В общем виде, каковы бы ни были `a` и `b` выполняется равенство:
```c
a[b] == *(a+b)
```
Более того, возможна запись:
```c
17[m]
```
ибо от перемены местами слагаемых сумма не меняется, но так *делать не стоит никогда*

## Вычитание типизированных адресов
В C определена операция *вычитания типизированных адресов*, результатом которой является целое число

Операция имеет смысл только при применении к адресам элементов *одного массива*
*Результат* операции -- *расстояние между двумя элементами* (то есть разность индексов)

Пример:
```c
p = &m[13]    /* p = m + 13 */
p - m         /* результат будет равен 13 */
```

## Инициализация массивов
При описании массива язык C позволяет задать начальные значения его элементов
Пример синтаксиса:
```c
int m[10] = { 2, 3, 5, 7, 11, 13, 17, 19, 21, 23 };
```
Символ `;` обязателен -- это часть синтаксиса описания
Все элементы инициализатора *обязаны* быть *константами времени компиляции*

Если для массива предусмотрен инициализатор, то размер в явном виде можно не задавать, он будет вычислен компилятором самостоятельно:
```c
int m[] = { 2, 3, 5, 7, 11, 13, 17, 19, 21, 23 };
```
размерность такого массива тоже будет равна `10` по количеству элементов в инициализаторе
Такая запись позволяет программисту не подсчитывать количество вручную

Узнать длину массива можно, поделив размер массива на размер элемента массива:
```c
sizeof(m)/sizeof(m[0])
```
такое выражение можно использовать в циклах по обходу элементов массива

Массив *можно проинициализировать*, но *присваивать* массиву *ничего никаким способом нельзя*, когда он уже существует
Конструкция из фигурных скобок и списка значений не является *выражением* и не может встречаться в других выражениях, в том числе в присваиваниях. 

Инициализировать можно и *локальные* массивы, однако такая возможность не бесплатна: *данные будут копироваться в область памяти в стековом фрейме каждый раз в начале выполнения такой функции*, так что чаще всего так делать не стоит
```c
int f(int x)
{
	int m[] = { 2, 3, 5, 7, 11, 13, 17, 19, 21, 23 };
	/* ... */
}
```
