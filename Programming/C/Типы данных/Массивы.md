## Описание массивов
В языке С массивов *как таковых* нет. 

Пример описания массива:
```c
int m[20];
```
описывает массив из `20` элементов типа `int`. Элементы доступны с помощью операции *индексирования* под номерами (индексами) от `0` до `19`

Элементы массива доступны *каждый в отдельности*, а *к самому массиву* как единому целому *обратиться невозможно* (в следствии чего в C отсутствует присваивание массивов, в том числе *строк*, как частного случая массивов)

*Введенное имя* `m` практически во всех ситуациях *обозначает* не сам массив как таковой, а *адрес* его *первого элемента*. Так как элементы имеют тип `int`, имя `m` представляет собой *константу* типа `int*`

Так, определив указатель:
```c
int *p;
```
можно выполнить присваивание 
```c
p = m;
```
и обращаться к элементам массива с помощью индексирования как через `p`, так и через `m`:
```c
p[0], p[1], p[2], ... p[19]
m[0], m[1], m[2], ... m[19]
```


> [!NOTE] Когда массив соответствует всему массиву
> Единственной осмысленной ситуацией, в которой имя массива соответствует всему массиву как единому объекту -- это применение к имени массива псевдофункции `sizeof(m)`
> Что для примера `m[20]` выдаст результат `80`
> 
> Еще пару ситуаций пред предусматривают "стандарты", но это сложно объяснить и для самостоятельного изучения

## Индексирование
При этом, исходя из теории про адресацию в C, к первому элементу массива (`m[0]`) можно также обратиться как `*m`

Индексирование -- операция не над массивами (над массивами нет операций), а над *адресами*

В C к типизированному адресному выражению (любому кроме `void*`) можно *прибавить целое число*, при этом *адрес изменяется на размер элемента* того *типа*, на который указывает исходный адрес.

Адрес типа `char*` при добавлении к нему `1` увеличится на `1` (1 байт)
Адрес типа `int*` при добавлении к нему `1` увеличится на `4` (4 байта)

Таким образом, если в памяти размещен массив элементов одного типа,  добавление единицы к адресу одного из этих элементов даст адрес следующего элемента массива

Это и выполняет операция индексирования
Пример:
```c
int m[20];
m     /* адрес первого элемента */
m+1   /* адрес второго элемента */
m+19  /* адрес последнего элемента */
```
Это *адресная арифметика*. Если выражения вида `m, m+1, ..., m+19` являются адресами, то их можно разыменовывать:
```c
*m, *(m+1), *(m+2), ..., *(m+19)
```
то же самое что:
```c
m[0], m[1], m[2], ..., m[19]
```

В общем виде, каковы бы ни были `a` и `b` выполняется равенство:
```c
a[b] == *(a+b)     /* то же значение */
&a[b] == *a+b      /* тот же адрес */
```
Более того, возможна запись:
```c
17[m]
```
ибо от перемены местами слагаемых сумма не меняется, но так *делать не стоит никогда*

## Вычитание типизированных адресов
В C определена операция *вычитания типизированных адресов*, результатом которой является целое число

Операция имеет смысл только при применении к адресам элементов *одного массива*
*Результат* операции -- *расстояние между двумя элементами* (то есть разность индексов)

Пример:
```c
p = &m[13]    /* p = m + 13 */
p - m         /* результат будет равен 13 */
```

## Инициализация массивов
При описании массива язык C позволяет задать начальные значения его элементов
Пример синтаксиса:
```c
int m[10] = { 2, 3, 5, 7, 11, 13, 17, 19, 21, 23 };
```
Символ `;` обязателен -- это часть синтаксиса описания
Все элементы инициализатора *обязаны* быть *константами времени компиляции*

Массив с указанием размера можно проинициализировать не полностью, а указав лишь часть элементов. При этом остальные элементы будут проинициализированы нулями:
```c
int some_data[SIZE] = {1492, 1066};

$ ./some_data
i no_data[i]
0 1492
1 1066
2 0
3 0
```

Если для массива предусмотрен инициализатор, то размер в явном виде можно не задавать, он будет вычислен компилятором самостоятельно:
```c
int m[] = { 2, 3, 5, 7, 11, 13, 17, 19, 21, 23 };
```
размерность такого массива тоже будет равна `10` по количеству элементов в инициализаторе
Такая запись позволяет программисту не подсчитывать количество вручную

Узнать количество элементов массива без указания можно, поделив размер массива на размер элемента массива:
```c
sizeof(m)/sizeof(m[0])
```
такое выражение можно использовать в циклах по обходу элементов массива

Массив *можно проинициализировать*, но *присваивать* массиву *ничего никаким способом нельзя*, когда он уже существует
Конструкция из фигурных скобок и списка значений не является *выражением* и не может встречаться в других выражениях, в том числе в присваиваниях. 

Инициализировать можно и *локальные* массивы, однако такая возможность не бесплатна: *данные будут копироваться в область памяти в стековом фрейме каждый раз в начале выполнения такой функции*, так что чаще всего так делать не стоит
```c
int f(int x)
{
	int m[] = { 2, 3, 5, 7, 11, 13, 17, 19, 21, 23 };
	/* ... */
}
```

В стандарте C99 появилась возможность применять *назначенные инициализаторы*, что позволяет выбрать, какие элементы массива будут инициализированы.
Так, для инициализации только последнего элемента в массиве синтаксис будет следующий:
```c
int arr[6] = {0, 0, 0, 0, 0, 212};  /* ANSI C (C90) (или без указания размера */

int arr[6] = {[5] = 212};           /* С99 инициализация последнего элемента */
```
При этом оставшиеся элементы будут проинициализированы нулями, так как данный случай соответствует частичной инициализации.
Стоит учесть что:
* при указании нескольких элементов в одном таком инициализаторе (`[4] = 31,30,31`), будут проинициализированы указанными значениями и последующие *n* элементов;
* если встретится повторная инициализация ранее определенного элемента, то актуальной будет последняя директива инициалиизации;
* если не указать размер массива и применить назначенный инициализатор, то компилятор сделает массив такой длины, чтобы уместить указанный элемент.
```c
int days[MONTHS] = {31,28, [4] = 31,30,31, [1] = 29};

17:54 $ ./designate    
1 31  
2 29  
3  0  
4  0  
5 31  
6 30

9  0  
10  0  
11  0

```
```c
int stuff[] = {1, [6] = 4,9,10};

$ ./stuff
0 1

```

## Индексирование массивов в цикле `for`

Для перебора элементов массива в цикле `for` следует использовать следующую конструкцию:
```c
for (index = 0; index < SIZE; index++)
```
где SIZE -- размер массива, указанный при объявлении.

Так цикл будет выполняться от первого элемента с индексом `0` до последнего с индексом `SIZE - 1`. А следующая итерация для `index = SIZE` выполнена не будет.

## Массивы переменной длины (C99)

До стандарта C99 при объявлении массива позволялось использовать *константное целочисленное выражение* (оно же -- константа времени выполнения, переменные с ключевым словом `const` сюда не относятся, а sizeof да).
При этом такое выражение должно быть больше нуля.

В C99 добавили возможность создания *массивов переменной длины*. При их объявлении количество элементов задается *переменной*, объявленной заранее.
```c
int n = 5;                  /* ок */
float a1[5];                /* ок */
float a2[5 * 2 + 1];        /* ок */
float a3[sizeof(int) + 1];  /* ок */
float a4[-4];               /* ошибка */
float a5[0];                /* ошибка */
float a6[2.5];              /* ошибка */
float a7[(int)2.5];         /* ок */
float a8[n];                /* C90 -- ошибка; C99 -- ок */
```

С такими массивами связан ряд дополнительных ограничений. Так, например, такие массивы *нельзя инициализировать* при объявлении.

В стандарте C11 их сделали дополнительной, а не обязательной языковой возможностью).

## Многомерные массивы

Объявления многомерных массивов выглядит следующим образом:
```c
float rain[5][12];
```
где *в объявлении*:
* `rain[5]` -- массив из 5 элементов, каждый из которых
* `float [12]` -- сам по себе является массивом из 12 элементов типа `float`

Для визуализации удобно представлять себе двумерные массивы в виде таблицы со строками и столбцами. В памяти же храенение будет плоским подряд.

Обращение к различным элементам:
```c
rain[0];    /* обращение к первому массиву из 12 элементов */
rain[2][3]; /* обращение к 4 элементу третьего массива из 12 элементов */
```

Пример инициализации такого массива:
```c
int some_data[2][3] =
{
    {1, 2, 3},
    {4, 5, 6}
}
```

При инициализации таких массивов можно не указывать внутренние фигурные скобки, однако, если количество элементов меньше необходимого, то способ заполнения оставшихзся элементов будет зависеть от постановки внутренних фигурных скобок:
![[Pasted image 20260315194102.png]]

Все это также распространяется и на большее количество "измерений", например:
```c
int box[10][20][30];
```

## Передача массивов в качестве аргументов функции для обработки

### С передачей количества элементов
Для решения задачи передачи массива на обработку в другую функцию следует помнить, что массив представляет собой адрес первого элемента. Таким образом описание формального аргумента функции в прототипе и объявлении должно представлять собой указатель соответствующего типа.

Помимо этого, такой функции с вероятностью 100% потребуется передать количество элементов массива в качестве второго аргумента:
```c
int sum (int *arr, int n);    /* типовой подход передачи массивов */
```

*Только* для *прототипа* и *объявления* можно использовать и такую форму:
```c
int sum (int arr[], int n);    /* подчеркивает что arr -- массив */ 
```

С учетом того, что в *прототипе* (и только в прототипе) позволяется не указывать имена формальных аргументов, *допустимы* все перечисленные ниже варианты:
```c
int sum (int *arr, int n);
int sum (int *, int);
int sum (int ar[], int n);
int sum (int [], int);
```

Если попытаться внутри такой функции вывести размер переданного первого аргумента, то получим не размер массива, а размер указателя (обычно 8 байт).

Пример функции подсчета суммы элементов:
```c
int sum (int arr[], int n)
{
	int i;
	int total = 0;

	for (i = 0; i < n; i++)
		total += arr[i];

	return total;
}
```

### С передачей двух указателей (начало и конец массива)

Другой вариант заключается в передаче указателей на первый элемент массива и на первую ячейку после завершения массива.

Прототип:
```c
int sum (int *start, int *end);
```
Реализация функции подсчета суммы элементов:
```c
int sum (int *start, int *end)
{
	int total = 0;

	while (start < end) {  /* end -- первая ячейка за массивом */
		total += *start;   /* добавить значение к total */
		start++;           /* переместить указатель на следующий элемент */
	}

	return total;
}
```
Но ни в коем случае не следует пытаться взять значение из ячейки за массивом --`marbles[SIZE]`, так как ее содержимое не гарантируется.

Помня о ассоциативности унарных операций `*` и `++` можно свернуть внутренность цикла в одну строку:
```c
total += *start++;
```
Гарантируется, что `++` будет применена к `start`, а не `*start`. См. [[Приоритет операций#Ассоциативность операций]]
