Подход к использованию динамической памяти в C существенно отличается от Pascal (см. [[Динамические переменные]])

В самом языке нет средств работы с динамической памятью -- они вынесены в библиотеку

Основные функции работы с динамической памятью:
* `malloc` -- выделяет память
* `free` -- освобождает память
Дополнительные функции работы с динамической памятью:
* `calloc`
* `realloc`

Заголовки функций:
```c
void *malloc(int size);
void free(boid *p);
```

Функции для работы с динамической памятью располагаются в заголовочном файле `stdlib.h`
## Функция `malloc`
Действия функции:
* принимает в качестве параметра размер необходимой области памяти.
* выделяет такую область из "кучи"
* возвращает адрес начала выделенной области

Возвращает нетипизированный адрес, так как не знает, для каких целей затребована память

Пример использования на практике:
```c
double *k;
k = malloc(360*sizeof(double));
```
выделяет область памяти, достаточной для размещения `360` элементов типа `double`

Так как [[Programming/C/Типы данных/Массивы#Индексирование|Индексирование]] есть *операция над адресами*, с этой областью памяти можно работать как с массивом, обращаясь к его элементам по индексам
Пример заполнения массива с обходом элементов в цикле:
```c
for(i = 0; i < 360; i++)
	k[i] = sin((2*M_PI/360.0) * (double)i);
```

## Функция `free`
Действия функции: освобождает ранее выделенную память, делая ее вновь доступной для выделения
В качестве *параметра* принимает *адрес, ранее возвращенный функцией* `malloc`

Пример освобождения массива из предыдущего пункта:
```c
free(k);
```
при этом размер области памяти указывать *не нужно* (его библиотека хранит сама)

Если функции `free` передать что-то отличающееся от адреса, возвращенного `malloc`, то авария практически гарантирована
