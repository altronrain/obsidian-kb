Помимо текстовых файлов программы на C могут обрабатывать и файлы бинарного формата. Например, средства посимвольного ввода-вывода `fgetc` и `fputc` справляются с *потоками байтов*, не являющихся текстом (см. [[Посимвольный ввод-вывод]])

## Блочный принципы работы с бинарными файлами
Зачастую, нетекстовые файлы *не рассматриваются как потоки*. 
Работа с бинарными файлами часто требует использования *попеременно* операций *чтения* и *записи* в разные места файла. С текстовым файлом такая работа невозможна из-за непостоянства размера строк и текстовых фрагментов.

Файлы, не являющиеся текстовыми можно рассматривать как последовательность записей фиксированного размера (этакий массив), что позволит вычислять размер и местонахождение каждой записи.

Подобный принцип работы называется *блочным*, так как мы оперируем блоками информации.

## Почему средства работы с бинарными файлами из стандартной библиотеки не популярны
Чаще всего для работы с файлами, имеющими нетекстовый формат, используются *низкоуровневые функции* ввода вывода. В семействе Unix-подобных ОС эти функции представляют собой *системные вызовы*, а точнее -- их обертки.

Преимущества, которые дают высокоуровневые средства ввода-вывода (работающие с переменными типа `FILE *`):
* перевод чисел из внутреннего представления в текстовое (при записи в потоки ввода-вывода) и обратно (при чтении из потоков) -- что полезно при работе с текстом
* буферизация -- что полезно при работе с потоками

Оба данных преимущества оказываются не нужны  при рассмотрении файлов бинарного формата, когда мы прыгаем по файлу чередуя операции чтения-записи.

## Средства стандартной библиотеки для работы с файлами как массивами произвольных данных
Функции блочного чтения и записи стандартной библиотеки требуют открытия файла как обычно, с помощью `fopen` (см. [[Работа с файлами и потоками|Работа с файлами и потоками]])

### Чтение и запись блоков
Сами функции имеют названия `fread` и `fwrite`. Их заголовки:
```c
int fread(void *ptr, int size, int n, FILE *stream);
int fwrite(const void *ptr, int size, int n, FILE *stream);
```
Для обеих функций:
* `stream` задает открытый файл, с которым работаем
* `n` указывает *количество элементов*, которые необходимо прочитать или записать
* `size` задает *размер каждого* такого *элемента*
* `ptr` передает адрес области памяти:
	* для `fwrite` -- содержащей информацию для записи
	* для `fread` -- куда следует поместить прочитанную информацию
В обеих случаях размер области памяти должен быть не менее `size * n`

Чаще всего указанные функции работают с байтами, то есть `size` устанавливается в `1`

Функции возвращают *количество* успешно записанных или прочитанных *элементов* (не байтов). Если возникла ошибка, то возвращается `0`. 
Также `0` возвращается если `fread`, не прочитав ни одного элемента обнаружила ситуацию "конец файла". Ситуация конца файла может возникнуть, когда уже несколько элементов прочитано, тогда `fread` возвращает число, меньшее, чем указанный ей параметр `n`, а следующее обращение к `fread` вернет `0`

### Диагностирование ошибок и ситуации "конец файла"
Чтобы отличить ошибочную ситуацию от штатной ситуации "конец файла", используются функции `feof` и `ferror`:
```c
int feof(FILE *stream);
int ferror(FILE *stream);
```
Они работают как булевы функции:
* возвращают ненулевое значение (*истину*) в случае возникновения ситуации "конец файла" (`feof`) или ошибки (`ferror`)
* возвращают ноль (*ложь*) если не возникло указанных ситуаций.

Так, если `fread` вернула `0`, можно сразу вызвать `feof` для определения, наступила ли ситуация "конец файла"

### Изменение текущей позиции в файле
Для изменения текущей позиции в файле используется функция `fseek`. Ее заголовок:
```c
int fseek(FILE *stream, long offset, int whence);
```
где:
* `stream` задает открытый файл, с которым работаем
* `offset` указывает, на сколько *байтов* следует сместиться
* `whence` определяет, *от какого места эти байты отсчитывать*

* при `whence`, равном константе `SEEK_SET`, отсчет пойдет от начала файла (`offset` должен быть не отрицательным)
* при `whence`, равном константе `SEEK_CUR`, отсчет пойдет от текущей позиции (`offset` может быть как положительным, так и отрицательным)
* при `whence`, равном константе `SEEK_END`, отсчет пойдет от конца файла (`offset` может быть любым, что забавно)

Функция возвращает новое значение текущей позиции, считая от начала файла.

Не все потоки допускают изменение текущей позиции. Например, потоки, связанные с терминалом такой возможности не дают (как на ввод так и на вывод), ибо данные нигде не хранятся, так что понятие текущей позиции для них не существует.
