Стандартная библиотека предоставляет возможность обрабатывать целые *строки*, разделенные символом перевода строки, как единицы информации.

## Функции с явным указанием потока ввода-вывода
Функции называются `fputs` и `fgets`
* `fputs` выдает заданную строку в заданный поток вывода
* `fgets` прочитывает строку из заданного потока ввода и размещает прочитанное в указанной области памяти
Заголовки указанных функций:
```c
int fputs(const char *s, FILE *stream);
char *fgets(char *s, int size, FILE *stream);
```

### Функция `fputs`
Пример вывода строки на стандартный поток вывода с помощью `fputs` вместо `printf`:
```c
fputs("Hello, world\n", stdout)
```
Главное отличие от `printf` и `fprintf` состоит в том, что содержимое строки *не интерпретируется*. Функция `fputs` *не обращает* никакого *внимания* на никакие специальные символы в строке

Функция возвращает неотрицательное число при успехе (чаще всего длину строки, но стандарт не регламентирует) и `-1` в случае ошибки

### Функция `fgets`
В качестве параметров функции имеются:
* `s` -- адрес области памяти, в которую хотим поместить прочитанную строку (соответственно, чаще всего -- массив `char` статический или динамический)
* `size` -- количество элементов в массиве для записи
* `stream` -- поток для считывания строки

Со считыванием строки *невозможно* достоверно *предсказать длину* строки, которая будет введена. Всегда *может не хватить массива* для размещения строки.
По этому и имеется второй параметр `size`, указывающий функции *сколько памяти* находится в ее распоряжении (размер массива для записи)

Функция `fgets` читает из потока вывода *не более чем* `size -1` символов.
В основном чтение прекращается в трех случаях:
* очередной символ оказался символом перевода строки
* очередной символ прочитать не удалось (ситуация "конец файла")
* прочитано максимально возможное количество символов (`size-1`)
Сам перевод строки в данном случае тоже заносится в массив.
Если хотя бы один символ был прочитан, функция также занесет нулевой символ в следующий элемент массива (за последним прочитанным), превратив массив в корректную строку.

Функция возвращает тот же адрес, что она получила через параметр `s`
Если ни одного символа прочитать не удалось -- возвращает `NULL` никак не изменяя содержимое массива `s`

## Функции для стандартных потоков
Весьма сомнительные в применении.
* `puts` -- выдает заданную строку в стандартный поток вывода
* `gets` -- прочитывает строку из стандартного потока ввода и размещает прочитанное в указанной области памяти
Заголовки указанных функций:
```c
int puts(const char *s);
char *gets(char *buf);
```

### Функция `puts`
Работает не так как `fputc` (см. [[Работа с файлами и потоками#Операции над файловым потоком|Работа с файлами b потоками]])
После вывода строки выдает еще символ перевода строки. То есть вывод `Hello, world` выглядел бы так:
```c
puts("Hello, world");
```
странно, но иногда использовать можно.

### Функция `gets`
Функцию `gets` *использовать нельзя ни в коем случае*. Функция принимает один параметр -- адрес массива для размещения считываемой строки.
Не имеется *никакого способа* сообщить *размер* массива. Невозможно гарантировать, что работа данной функции не приведет к выходу за пределы памяти.

> [!NOTE] Причина: buffer overflow exploit
> Вспоминая устройство стекового фрейма:
> * *локальные переменные* размещаются в стеке
> * стек растет в направлении уменьшения адресов (в сторону противоположную той, куда мы перемещаемся увеличивая индекс в массиве)
> 
> Соответственно, заполняя массив и выйдя за его пределы можно затереть адрес возврата из функции произвольным значением, или подменить его с целью передачи управления на произвольный код (главное в передаваемой строке не передавать байт `10`, но это просто)
> Такая техника называется *buffer overflow exploit*
> 
> P.S.: Направление роста стека зависит от процессора, но крайне часто стек "растет вниз" (в направлении убывания адресов). В любом случае с функцией `gets` грозит переполнение
 
На самом деле при инициализации массива *придется указать его размер*. Без этого его просто-напросто невозможно создать. 
Все отличие `gets` от `malloc`, `fgets` и т. д. в том, что *она спокойно позволяет выход за пределы массива, никак этому не препятствуя.*

Штатного способа *прочитать строку*, никак *не ограничивая ее длину*, в библиотеке C *нет ни в каком виде*. При необходимости такое способ придется изготовить самостоятельно, например:
* посимвольное чтение в элементы динамически выделенного массива, с ручным увеличением его размера по мере надобности
* использование списков как динамической структуры данных
