Функция `scanf` выполняет форматированный ввод.
Пример формального прототипа (описания):
```c
int scanf(const char *format, ...);
```
параметров у функции может быть сколько угодно.

Фактически функция `scanf()` считывает текстовые данные и преобразует их в желаемые указанные типы данных (различные числа, строки).

Как и `printf` функция представляет собой *интерпретатор* форматной строки.

Форматная строка состоит из *форматных директив*, которые делятся на:
* директивы преобразования (начинаются с символа `%` и заканчиваются *спецификатором преобразования*) (во многом аналогичны таковым из `printf`)
* последовательности *пробельных символов* произвольной длины (пробелы. табуляции, переводы строки и т.д.)
* обычные символы (все остальные символы)

Количество спецификаторов в первом аргументе (строке) определяет, сколько же еще аргументов ожидается в данном конкретном вызове.

Функция `scanf()` сама по себе больше предназначена для считывания одного слова или значения. Если введено несколько через пробельные символы, то базово `%s` считает только первое слово до пробела.

### Поведение scanf() с различными спецификаторами при считывании ввода 

Функция `scanf()` читает поток ввода посимвольно, в общем случае пропуская пробельные символы, пока не натолкнется на непробельный символ.

При обработке спецификатора `%d` `scanf()` в качестве непробельного символа ожидает *цифровой символ* или *знак*. Если встречает иное -- возвращает последний прочитанный *неподходящий* символ обратно в  поток ввода.
Если до этого успела прочитать какие-то цифры, то преобразует их в число и заносит его в переменную. Если ничего не прочитала, то так и будем считывать первый некорректный символ.

При обработке спецификаторов других типов чисел просто расширяется набор "корректных" символов (так например для `%x` будет еще `a-z A-Z`)

При обработаке спецификатора `%s` допускается любой символ, отличный от пробельного (опять же в общем случае). Как только появляется пробельный символ, вся последовательность прочитанных непробельных сохраняется в переменную (массив). В конец массива добавляется символ `\0`. 

## Дополнительные параметры в директиве преобразования
Внутри директивы преобразования, *между символом* `%` *и символом спецификации формата*
могут указываться дополнительные параметры форматирования (необязательные):
* флаг `*`
* ширина ввода
* *модификатор разрядности*

### Флаг `*`
Указывается сразу после символа `%`. Запрещает присваивание, то есть значение читается, но никакой параметр из стека не берется и никакая информаци в память не заносится:
```c
%*i   /* прочитать целое число и никуда не заносить */
```

### Ширина ввода
Для чисел обычно не используется, однако можно указать. В таком случае для формирования числа будет использовано *не более чем заданное количество символов из потока ввода*

Для строк, то есть при использовании:
```c
%s, %[...], %[^...]
```
если не указать ширину поля при чтении, то каков бы ни был размер массива, созданного для размещения строки, его может не хватить и произойдет переполнение.

*Указание ширины поля при чтении строк абсолютно обязательно*

Массив, в который строка помещается, должен быть на единицу больше по размеру (для помещения в конец ограничителя `\0`)

Взять эти значения из параметров, в отличие от `printf` функция `scanf` не может, потому область применения сильно сужается

### Модификатор разрядности
По умолчанию `scanf` при чтении *целых* чисел, в зависимости от спецификаторов, предполагает следующий тип считываемого параметра:
* `int*` -- для `%i` и `%d`
* `unsigned int*` -- для `%o, %x, %u`
Поведение можно изменить указав *модификатор разрядности*. Записывается непосредственно перед символом-спецификатором. Может принимать значения:
* `hh` -- означает тип `signed char` или `unsigned char` целочисленного параметра
* `h` --  означает тип `short` или `unsigned short` целочисленного параметра
* `l` -- означает тип `long` или `unsigned long` целочисленного параметра
* `ll` -- означает тип `long long` или `unsigned long long` целочисленного параметра

В C90 отсутствовали модификаторы для `char` и . В C99 добавили (см. `hh`).

Также добавили (C99+):
* `j` -- для целочисленного параметра для сохранения в `intmax_t` или `uintmax_t` (`"%jd"`, `"%8jX"`)
* `t` -- для целочисленного параметра для сохранения значений разницы указателей `ptrdiff_t` (`"%td"`, `"%12ti"`)
* `z` -- для целочисленного параметра для сохранения значений типа `size_t` (`"%zd"`, `"%12zx"`)

При чтении *чисел с плавающей точкой* по умолчанию предполагается тип `float`
Поведение тоже можно изменить указав *модификатор разрядности*. Записывается непосредственно перед символом-спецификатором. Может принимать значения:
* `l` -- означает тип `double` для чисел с плавающей точкой
* `L` -- означает тип `long double` для чисел с плавающей точкой

Соответственно, в отличе от `printf` разрядность имеет большое значение, ибо влияет на то, *какие типы* параметров мы *записываем в память*

## Последовательности пробельных символов
Последовательности *пробельных символов* (пробел, табуляция, перевод строки и так далее) произвольной длины, если встречаются в форматной строке, рассматриваются как указание выбирать из потока ввода пробельные символы и игнорировать их до тех пор, пока очередной символ окажется *не пробельным*

Полезно для спецификатора `%c`, в то время как остальные сами игнорируют пробельные символы.
Для спецификатора `%c`:
* без указания пробела в форматной строке будет считан первый значащий символ
* с указанием пробельного символа в форматной строке -- первый непробельный символ

Пример:
```c
printf("Введите пробельный символ и букву\n");
if (scanf("%c%*c", &literal1) != 1)
{
	printf("Передано некорректное значение.\n");
	return 1;
}
printf("Введенное значение: %c\n", literal1);
printf("Введите ту же последовательность еще раз\n");
if (scanf(" %c%*c", &literal2) != 1)
{
	printf("Передано некорректное значение.\n");
	return 1;
}
printf("Введенное значение: %c\n.", literal2);

return 0;
```
Результат выполнения:
```
$ ./scanf_whitespaces
Введите пробельный символ и букву
	d
Введенное значение:
Введите ту же последовательность еще раз
	d
Введенное значение: d
```

## Все остальные символы
Рассматриваются как директивы особого рода. Если `printf` их просто печатает, то `scanf` *требует*, *чтобы*, если он дошел в интерпретации форматной строки до обычного символа, то *в потоке ввода в этот момент встретился точно такой же символ*

Если символы в потоке ввода и в форматной строке не совпадают, `scanf` прекращает свою работу, оставив "неправильный" символ в потоке.

Аналогичное происходит если `scanf` *не смогла трактовать последовательность символов как текстовую запись значения, которое ожидается* (целое, строка и т.д.)

## Возвращаемое значение
Функция `scanf` *возвращает количество успешно преобразованных и помещенных в память значений*. То есть -- количество успешных директив преобразования, предполагающих запись по адресам, заданным в списке параметров.

Также `scanf` может вернуть:
* `-1`, если достигнут конец файла или произошла ошибка при чтении до попытки прочитать первый параметр.
* `0` при ошибке во время анализа первого параметра
* `1` при ошибке во время анализа второго параметра
и так далее

Крайне полезно *всегда анализировать возвращаемое значение* `scanf`, если она используется.
