Стандартная библиотека ввода-вывода также позволяет организовать работу с файлами, открывая их на чтение и запись, тем самым формируя новые потоки ввода-вывода

Для идентификации потоков ввода-вывода в файл в стандартной библиотеке предусмотрен тип `FILE*`, который *является адресом* чего-то, имеющего тип `FILE`
> [!NOTE] 
> Саму реализацию типа `FILE` скрывает библиотека, и данный тип может варьироваться от версии стандартной библиотеки. Потому программист оперирует типом `FILE*`
> То есть библиотека сама заботится о хранении всей необходимой для работы с файлами информации, а пользователю предоставляет лишь адрес для идентификации потока ввода-вывода

Указатели типа `FILE*` можно присваивать, передавать в функцию по значению, возвращать из функций, так как копируется просто адресное значение.

## Открытие файла
Для открытия файлов используется функция `fopen`. Ее заголовок имеет следующий вид:
```c
FILE* fopen(const char *name, const char *mode);
```
где:
* параметры -- строки, которые гарантируется не изменять:
	* первый параметр -- имя файла;
	* второй параметр -- режим открытия файла.

Имя файла может представлять собой:
* относительный (относительно *текущего* каталога)
* абсолютный (если начинается с символа `/`)

Возможные режимы открытия файла:
* `r` -- только чтение
* `r+` -- чтение и запись (работа начинается с начала файла)
* `w` -- только запись (если файла не было, он создается пустым; если существовал -- старое содержимое уничтожается);
* `w+` -- запись и чтение (можно что-то записав в файл, спозиционироваться на записанные данные и прочитать их)
* `a` -- открытие файла на добавление данных в конце (открытие на запись после старого конца файла). Если файла не существовало -- он создается
* `a+` -- чтение и запись (чтение с начала файла, запись в конец)

> [!NOTE] binary
> Unix-системы подразумевают еще режим *binary* буквой `b`, которую можно комбинировать с другими режимами (при этом можно ставить как перед `+` так и после). Но сам по себе в Unix-системах этот режим *ничего не делае*т, а в Windows может привести к трудноуловимым ошибкам и странному поведению

Открытие файла может вернуть и `NULL` (специальный указатель "ничего"), что говорит об ошибке во время выполнения. Поводов в случае функции `fopen`, может быть более чем предостаточно. Для определения причины можно воспользоваться глобальной переменной `errno` (см. [[Обработка ошибок при обращениях к ОС]])

## Операции над файловым потоком
Над открытым потоком ввода-вывода в файл можно выполнять тот же набор действий, что и над стандартными потоками. То есть, для потоков типа `FILE*` предусмотрен набор функций, аналогичный подобному для стандартных потоков. 

Так, для ввода-вывода отдельных символов имеются функции:
* `fgetc`, `int fgetc(FILE *stream);`
* `fputc`, `int fputc(int c, FILE *stream);`
аналогичны `getchar` и `putchar`, но принимающие еще в качестве аргумента поток файла (см. [[Посимвольный ввод-вывод]])

Имеются и функции форматированного ввода-вывода:
* `fprintf`
* `fscanf`
аналогичны `printf` и `scanf`, отличаются только наличием параметра типа `FILE*`, указываемого первым (перед форматной строкой) (см. [[Форматированный ввод `scanf`]] и [[Форматированный вывод `printf`]])

*Любую функцию, как библиотечную, так и собственную, рассчитанную на работу с потоком типа `FILE*` можно применить к одному из стандартных потоков ввода-вывода.*
Для этого в заголовочном файле `stdio.h` объявлены три глобальные переменные:
* `stdin`
* `stdout`
* `stderr`

## Закрытие файла
Для закрытия файла используется функция `fclose`, которая принимает один единственный параметр типа `FILE*`. Ее заголовок имеет следующий вид:
```c
int fclose(FILE *f);
```
она закрывает переданный поток ввода вывода, делая дальнейшие операции с ним *невозможными*. При этом высвобождаются ресурсы ОС, обеспечивавшие работу данного потока ввода-вывода.
В случае успеха функция возвращает `0`, а если произошла ошибка -- `-1` (поток при этом закрывается в любом случае)

Ошибки могут возникнуть по различным причинам, например из-за *буферизации*. Библиотечная функция сама организует буфер и сливает в файл данные не сразу. Не гарантируется, что на момент записи перед закрытием файла не произойдет ошибка (если в буфере были какие-то данные).

Аналогичная ситуация и при использовании системных вызовов `open` и `close` напрямую. Буфер организует и сама ОС. Подробнее см. [[Буферизация ввода-вывода]]
