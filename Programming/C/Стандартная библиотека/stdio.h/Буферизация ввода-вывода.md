Библиотечные функции ввода-вывода помимо выполнения своих задач обеспечивают *буферизацию* данных.

В реальности при вызове функций считывания с потока ввода, считывается не один символ, а порция данных (не больше определенного количества -- *размера буфера*). Например, при вводе с клавиатуры может считаться все. что пользователь успел ввести на текущий момент, после чего `getchar` отдаст один байт (остальные останутся в буфере)

Такой режим работы введен для оптимизации, так как накладные расходы на постоянное обращение к ОС (*переключение контекстов*) велики.
