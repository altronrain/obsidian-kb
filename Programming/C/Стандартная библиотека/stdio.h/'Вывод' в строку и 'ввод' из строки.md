Возможности форматного ввода-вывода, рассмотренные ранее (см. [[Форматированный ввод `scanf`]] и [[Форматированный вывод `printf`]]) можно задействовать для:
* формирования отформатированного текстового представления *в строковом буфере*
* анализа информации, содержащейся *в строковом буфере*

Для этого предусмотрены функции `sprintf` и `sscanf`. 

## Функция `sprintf`
Ведут они себя во многом как `printf`, однако результат преобразования в текстовое представление *никуда не выводится*, а *складывается в* предоставленный *строковый буфер*
Заголовок данной функции:
```c
int sprintf(char *buf, const char *format, ...);
```
Буфер для формирования результата передается через первый параметр, остальные параметры аналогичны `printf`

Функция не знает размера буфера и не производит никаких проверок на эту тему. Обеспечить буфер нужного размера -- обязанность вызывающего.
По этому при вставке через директиву `%s` некоей строки, когда ее размер неизвестен точно, следует *обязательно ограничивать пространство*, которое строка может занять в результате. Например, указав `%.20s`
> [!NOTE] Современные версии библиотеки
> В современных версиях библиотеки (начиная с C99) имеется функция `snprintf`, принимающая в качестве еще одного параметра *размер буфера*
> 
> При указании опций компилятора, фиксирующих версию языка, изменения затрагивают не только сам язык, но и возможности библиотек. Если зафиксировать ANSI C, `-ansi -pedantic`, функция `snprintf` окажется недоступной

Функция `sprintf` часто используется для перевода числа того или иного типа в его текстовое представление:
```c
char str[32];
int n;
/*...*/
sprinf(str, "%d", n);
```
Вызов `sprintf` занесет в массив `str` текстовое представление числа в переменной `n`

Также может быть полезна при передаче текстовых данных с использование ввода вывода *низкого уровня* (через системные вызовы и дескрипторы потоков, минуя высокоуровневую библиотеку). Данные сначала формируются в массиве типа `char`, а потом передаются ОС.

Функция `sprintf` возвращает количество *выведенных символов* (длину строки, сформированной в буфере *без учета нулевого байта*)

## Функция `sscanf`
Проделывает противоположную операцию функции `sprintf` -- *анализирует строку* в соответствии с заданным форматом и *раскладывает результаты по адресам* из списка параметров. 
Заголовок данной функции:
```c
int sscanf(const char *buf, const char *format, ...);'
```
Правила интерпретации полностью совпадают с `scanf`. Роль ситуации конца файла играет преждевременное окончание строки `buf`.
Если строка кончилась раньше, чем функция успешно проанализировала хотя бы одну директиву преобразования, возвращается `-1`
