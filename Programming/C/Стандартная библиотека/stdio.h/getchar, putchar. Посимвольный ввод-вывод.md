Для посимвольного ввода и вывода используются функции `getchar` и `putchar`

## Посимвольный ввод (`getchar`)
Функция `getchar` выполняет операцию ввода (считывания) символа со стандартного потока ввода, а точнее, ввод порции данных, соответствующей минимальному адресуемому на данной машине (в реальности всегда `1` восьмибитный байт)

Она не принимает параметров, а возвращает значение типа `int`:
```c
int getchar();
```
Почему *функция возвращает* `int` *вместо* `char` (`signed` или `unsigned`) -- потому что диапазона `0..255` *недостаточно* для обработки чего-либо еще кроме вводимых символов. Во время штатной работы встречается как минимум одна ситуация, не являющаяся символом -- *ситуация конца файла*

Функция `getchar` при возникновении ситуации конца файла возвращает `-1`, который для ясности обозначают идентификатором `EOF` (end of file) заглавными буквами.

Пример функции, которая считывает ввод и печатает `OK` на каждую строку:
```c
#include <stdio.h>

int main()
{
	int c;
	while((c = getchar()) != EOF){
		if(c == '\n')
			printf("OK\n");
	}
	return 0;
}
```
Редкий случай, когда побочный эффект в условном выражении уместен и оправдан -- вызов `getchar` помещен прямо в условие цикла (так как присваивание это операция, это становится возможным, однако необходимы скобки ввиду низкого приоритета операции)

Значение, возвращенное функцией `getchar`, нельзя сразу присваивать переменной типа `char`. Такое присваивание возможно лишь после того, как проверили, что возвращено значение, отличное от `EOF`

## Посимвольный вывод (`putchar`)
Функция `putchar` производит вывод символа на стандартный поток вывода.
Пример заголовка функции:
```c
int putchar(int c);
```
Параметр задает код выводимого символа. 
Формально параметр имеет тип `int`, но числа за пределами диапазона `unsigned char` (в реальной жизни `0..255`) будут превращены в числа этого диапазона путем отбрасывания "лишних" битов.

Функция возвращает код выведенного символа, если все в порядке, а если произошла ошибка, то вернет константу `EOF`

## Возврат прочитанного символа в поток (`ungetc`)
Возвращает *только что* прочитанный символ *обратно* в поток.
Пример заголовка функции:
```c
int ungetc(int c, FILE *stream);
```
Следующая операция *чтения* получит *этот символ* первым, как если бы он *еще не был прочитан*.
Ограничения:
* вернуть можно только один символ
* это реально должен быть символ, только что прочитанный из этого же потока
в остальных случаях результат неопределен.

Функция полезна при *лексическом анализе*, хотя можно обойтись и без нее. В ситуациях, когда невозможно понять, не прочитав *очередной символ*, является ли он *частью лексемы* или нет. Например:
```
756+27
```
Понять, что число `756` кончилось, можно лишь прочитав `+`. Вероятнее всего обработка чисел реализована отдельной функцией, в которой не хочется описывать все разделители. В такой ситуации функция `ungetc` сэкономит много времени
