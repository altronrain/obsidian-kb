Функция `printf` выполняет форматированный вывод.
Пример формального прототипа (описания):
```c
int printf(const char *format, ...);
```
параметров у функции может быть сколько угодно, что обозначено `...` Такие функции называются *вариадическими* (см. [[Вариадические фунции]])

Работа функции *управляется* форматной строкой. По сути функция представляет собой *интерпретатор* форматной строки.

Форматная строка состоит из *форматных директив*, которые делятся на:
* обычные символы (любые байты кроме нулевого и символа `%`)
* директивы преобразования (начинаются с символа `%` и заканчиваются *спецификатором преобразования*)

Количество спецификаторов в первом аргументе определяет, сколько же еще аргументов ожидается в данном конкретном вызове.

## Спецификаторы преобразования
Список символов, которые могут выступать в качестве спецификатора преобразования:
```
diouxXeEfFgGcsp%
```
Спецификатор преобразования указывает:
* какого типа значение следует извлечь из списка фактических параметров функции (из очередной позиции стека)
* в каком виде представить это значение на печати

## Дополнительные параметры в директиве преобразования
Внутри директивы преобразования, *между символом* `%` *и символом спецификации формата*
могут указываться дополнительные параметры форматирования (необязательные):
* флаги: набор из символов `- + 0 # <пробел>`
* целое число, задающее ширину поля (количество знакомест для вывода параметра)
* десятичная точка и целое число, задающее *точность представления*
* *модификатор разрядности*

### Ширина и флаги
Число, задающее ширину означает *минимальное* количество знакомест, которое будет использовано в выводе. Если представление занимает меньше знаков, то оно будет дополнено пробелами слева (в простейшем случае)
Пример:
```c
printf("[%5d]", 12)    -> "[   12]"
```
Ширина *не задает максимальное* количество знакомест. Пример:
```c
printf("%3d", 1234)    -> "1234"
```

Дополнительно вывод можно модифицировать с помощью флагов:
* `-` -- выравнивание по левому краю, а не по правому: `"[12   ]"`
* `0` -- дополнение не пробелами а нулями: `"[00012]"`
* `+` -- обязательно печатать знак числа (даже если положительное, отрицательные и так выводятся со знаком)
```c
printf("%+d,%+d", 12, -3)    -> "+12,-3"
```
* `<пробле>` -- для знака нужно оставить место, но печатать его только для отрицательных, для положительных предварять пробелом:
```c
printf("[% d],[% d]", 12, -3)    -> "[ 12],[-3]"
```
* `#` -- задает не требование к ширине поля, а требование *модифицировать формат*
	* для `oct` требует вывести лидирующий `0`
	* для `hex` требует вывести лидирующую комбинацию `0x` или `0X` (зависит от спецификатора)
	* для спецификаторов `e, E, f, F` требует обязательной выдачи десятичной точки, даже если  дробная часть равна нулю
	* для спецификаторов `g, G` запрещает удалять незначащие нули в дробной части

### Точность
*После* ширины или *вместо* нее можно задать *точность*, записанную с точки. Точность выполняет разные действия в зависимости от спецификатора (типа параметра):
* для чисел с плавающей точкой -- количество цифр после запятой
* для строк -- *максимальное* количество символов строки (остальные будут *отброшены*)
```c
printf("[%7.5s]", "abrakadabra")   -> "[  abrak]"
```
* для целого числа означает *минимальное количество цифр*. Обычно применяется совместно с шириной:
	* сначала число дополняется нулями до требуемой точности
	* затем число дополняется пробелами до требуемой ширины
```c
printf("[%6.4d]", 12)   -> "[  0012]"
```

### Модификатор разрядности
Записывается непосредственно перед символом-спецификатором. Может принимать значения:
* `h` --  означает тип `short` или `unsigned short` целочисленного параметра
* `l` -- означает тип `long` или `unsigned long` целочисленного параметра
* `ll` -- означает тип `long long` или `unsigned long long` целочисленного параметра
* `L` -- означает тип `long double` для параметра с плавающей точкой

Для `float` отдельного модификатора не требуется, так как при вызове любой функции значение выражения типа `float`, в списке фактических параметров, всегда преобразуется к `double`.  

По аналогичной причине отсутствует модификатор для `char`, ибо минимально что умеем выводить, это `short int`.

Аналогично и `short` при передаче расширяется до `int` (для удобства), потому модификатор `h` в основном может потребоваться, чтобы вывести на экран значение, которое примет число, усеченное до `short`

### Символ `*` для ширины и точности
Ширину и точность можно не указывать в явном виде, а взять из очередного целочисленного параметра, использовав вместо числа символ `*`
Пример:
```c
printf("%*.*d", w, p, n)
```
напечатает число `n` с использованием ширины `w` и точности `p`

## Возвращаемое значение
Обычно не используется, однако `printf` возвращает количество напечатанных символов (тип `int`)

## Буферизация вывода
Выходные данные не отправляются на терминал напрямую при каждом поступлении. Они пересылаются в промежуточный *буфер*, откуда затем уже выводятся на терминал.
Данный механизм реализован с целью сокращения накладных расходов на системные вызовы в том числе.

Критерии отправки данных из буфера на терминал довольно очевидны:
* заполнение буфера
* встреча символа новой строки
* наступает время ввода данных (например, используя `scanf()`)
