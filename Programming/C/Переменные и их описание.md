Объявлять и описывать можно не только функции, но и *переменные*

## Basic
### Описание переменных
Описания переменных могут находиться:
* *внутри тела функции* --  такая переменная является *локальной* (и видна только в пределах блока, в которой описана)
* *вне тел функций* --  такая переменная будет *глобальной*, то есть видимой от места своего описания до конца единицы трансляции
тем самым *глобальные* и *локальные* переменные имеют разную *область видимости*

Пример описания переменных:
```c
int dangerous_global_variable;

int f()
{
	int local_var;
	/*...*/
}
```
Нужно руководствоваться правилом, что: *использовать глобальные переменные крайне нежелательно*

Помимо области видимости переменные характеризуются еще *временем существования* (периодом времени от создания до высвобождения памяти):
* *глобальные переменные* существуют все время выполнения программы
* локальные переменные начинают существовать при входе в блок, в котором они описаны, и исчезают при выходе из этого блока (возникают и исчезают в момент изменения регистра `ESP`)

### Описание переменных в блоках (функций)

#### ANSI C
Стандарт ANSI C требует объявления переменных в начале блока. Не допускается наличие каких-то других операторов до объявления переменной.
Пример:
```c
int f()
{
	int var1;
	int var2;
	/*...*/
}
```

#### С99 и выше
Начиная со стандарта C99 позволяется размещать объявления переменных в любом месте блока. При этом переменную обязательно нужно объявить до первого использования по телу блока.
Пример:
```c
int f()
{
	/* какие-то операторы по телу функции */
	int var1;
	var1 = 42;
	/* еще операторы */
	int var2;
	var2 = 334;
	/*...*/
}
```
При этом рекомендуется придерживаться традиционных правил с целью обеспечения обратной совместимости.

### Допустимые имена переменных
Имя переменной называется также *идентификатором*. Для указания имен допустимо использовать:
* буквы нижнего и верхнего регистров;
* цифры;
* символ подчеркивания `_`;

Первым символом должна быть буква или символ подчеркивания. Не рекомендуется применять буквы верхнего регистра и использовать CamelCase. (предпочтение отдается snake_case).
Пример:
```
/* допустимые имена переменных */
wiggles
cat2
Hot_Tub
taxRate
_kcab

/* недопустимые имена переменных */
$Z]**
2cat
Hot-Tub
tax rate
don't
```

Имена, начинающиеся с одного или двух символов подчеркивания `_` считаются зарезервированными (например, библиотечные функции). Можно получить конфликт имен, следует использовать с осторожностью.

## Advanced
### Инициализация  переменных и присваивание
Описание переменной с явным указанием ее начального значения называется *инициализацией*
Пример:
```c
int some_global_var = 42;
```
Выражение справа от знака равенства -- *инициализатор*

Инициализация не имеет ничего общего с *присваиванием* (хотя обозначается тем же знаком)

*Присваивание* является *деструктивной* операцией:
* разрушается имеющееся значение
* записывается вместо него новое

*Инициализация* *ничего не разрушает*, так как до момента описания переменной не было.

Если в описании переменной нет инициализатора, то ее исходное значение зависит от того, локальная она или глобальная (за исключением *локальных статических переменных*)
* глобальные переменные без указания начальных значений заполняются *нулями*
* локальные переменные без указания начальных значений содержат произвольный *мусор*

### Корреляция с языком ассемблера
Расположение переменных в секциях программы:
* *глобальные* переменные с заданным начальным значением компилятор помещает в секцию `.data`
* *глобальные* переменные без задания начального значения компилятор помещает в секцию `.bss`

Собственно, при загрузке программы в память ОС из секций `.data` и `.bss` формирует сегмент данных и заполняет нулями область, соответствующую `.bss`
Подробнее см. [[Programming/NASM/Структура программы|Структура программы]] и [[Память пользовательской задачи. Сегменты]]

*Локальные* переменные располагаются в *стековых фреймах*, которые создаются при вызове функции, и уничтожаются при ее завершении.

Если заполнение `.bss` нулями выполняется один раз при загрузке программы в память, то обращения к функциям совершаются часто. По этому было бы очень накладно по времени работы реализовывать очистку стекового фрейма.
Именно по этому в *неинициализированных* локальных переменных находится какой-то мусор от предыдущих фреймов на стеке.

*Использовать* значения *неинициализированных переменных* до внесения значений -- *грубая ошибка*, которую сложно отловить компилятору.

Инициализация *глобальных* переменных ничего по ресурсозатратам не стоит кроме размера пары байтов, так как секция `.data` целиком размещается в исполняемом файле и затем записывается в память.

Инициализация *локальных* переменных *на уровне машинного кода* реализована так же, как *присваивание*. 
Если в самом начале функции мы присваиваем локальной переменной какое-то значение, то не следует для этой переменной задавать еще и инициализатор (экономия пары тактов)

### Локальные "статические" переменные
Представляют собой странный гибрид локальных и глобальных переменных.
Свойства таких переменных:
* *область видимости* соответствует *локальной* переменной (внутри блока с описанием)
* *время существования* соответствует *глобальной* переменной (все время выполнения программы)
Описываются также в начале блока.

Размещаются в секциях такие переменные аналогично *глобальным*:
* в секции `.data` если *указан* инициализатор
* в секции `.bss` если инициализатора *не указано*

Описание локальной статической переменной отличается от обычной локальной переменной наличием слова `static`:
```c
int my_function(int x, int y)
{
	int tt = 25;
	static int zz = 75;

	/*...*/
}
```
где:
* `tt` -- обычная локальная переменная
* `zz` -- локальная статическая переменная

*Обычная локальная переменная*:
* при рекурсивных вызовах функции может одновременно существовать в *нескольких экземплярах* (по числу вызовов функции, которые состоялись, но не завершились)
* каждому такому экземпляру при входе будет присвоено значение, указанное в инициализаторе
* при завершении работы функции экземпляр локальной переменной исчезает вместе со всем стековым фреймом

*Статическая локальная переменная*:
* существует вне зависимости от вызова функции с начала работы программы  и имеет значение, указанное в инициализаторе , и готова к работе
* по окончании работы функции *не исчезает*
* *сохраняет свое значение между вызовами функции*

Пример:
```c
void print_next_number()
{
	static int n = 0;
	printf("%d\n", n);
	n++;
}
```
функция при каждом вызове печатает *следующее целое число* (`0, 1, 2, ...)

Локальные статические переменные обладают большинством *недостатков* глобальных переменных, так что их следует *использовать с осторожностью*

Слово `static` может применяться не только к локальным, но и к *глобальным* описаниям (как к переменным так и к функциям). Там смысл его меняется. Для глобальных описаний оно будет означать, что они не *видны из других модулей программы*. См. [[Раздельная трансляция]]
