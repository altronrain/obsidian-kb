В стандартной библиотеке C вопрос выяснения причины неуспешного вызова для функций, *предполагающих обращение к операционной системе*, решается единообразно:
*Код ошибки заносится в глобальную переменную* `errno`

## Просмотр и хранение ошибок
Для доступа к этой переменной нужно подключить заголовочный файл `errno.h`
Возможные значения переменной `errno` имеют символические имена. 
Например функция открытия файла может иметь следующие ошибки:
* `ENOENT` -- отсутствует сам файл или каталог, в котором он должен был быть расположен
* `EACCESS` -- для открытия файла программе не хватило полномочий
* `EROFS` -- файл находится на диске, который можно только читать, при попытке открытия на запись
и так далее.
Для изучения всех кодов ошибок для библиотечных функций или системных вызовов можно посмотреть `man`.
## Средства обработки ошибок
Обычно в программах либо вообще не анализируют значение `errno`, либо проверяют один-два особых значения, а остальные ошибки обрабатывают однотипно в одном месте, используя *предназначенные для этого средства*.

Так, функция `strerror` принимает на вход *код ошибки*, а возвращает *адрес строки*, содержащей соответствующее этому коду диагностическое сообщение.
На вход `sterror` можно подать саму переменную `errno`, а возвращаемую строку -- напечатать.

Если выводить сведения об ошибке на стандартный поток диагностики `stderr`, то лучше использовать функцию `perror`. Она имеет один параметр -- строку, описывающую то, с чем произошла проблема (для файлов обычно указывается имя файла, в иных случаях часто указывают имя функции).
В поток диагностики выдается параметр `perror`, затем двоеточие и диагностическое сообщение в зависимости от значения `errno`. 
Пример:
```c
FILE *f;
f = fopen("file.txt", "r");
if(!f) {
	perror("file.txt");
	exit(1);
}
```
При отсутствии указанного файла будет выведено сообщение:
```c
file.txt: no such file or directory
```

Можно вывести и произвольное сообщение на стандартный поток диагностики, например с помощью функции `fprintf` и глобальной переменной `stderr`:
```c
fprintf(stderr, "I think there's an error\n");
```
где:
* `fprintf` -- функция форматированного вывода на указанный поток
* `stderr` -- идентификатор стандартного потока диагностики
