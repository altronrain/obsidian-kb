Программа на Си состоит из функций. *Нет* никакого аналога *головной программы*. Любые операторы могут находиться только в телах функций.
Вне функций могут располагаться:
* описания типов
* описания глобальных переменных и т.п.
они не являются операторами и ничего сами по себе делать не могут

Как ранее было указано, в файле на C могут присутствовать:
* глобальные объявления
* глобальные описания
подробнее см. [[Programming/C/Структура программы|Структура программы]]

## Описание функций
Описание функций состоит из:
* заголовка функции
* тела функции

В *заголовке* указывается:
* тип значения, которое возвращает функция (перед именем функции, в редких случаях наоборот)
* имя функции
* список параметров функции в круглых скобках (для каждого указывается тип)

Тело функции, заключенное в фигурные скобки, включает в себя:
* *необязательная* секция описания локальных имен
* последовательность операторов

Пример описания функции:
```c
char case_up(char c)
{
	if(c >= 'a' && c <= 'z')
		return c - ('a' - 'A');
	else
		return c;
}
```

Компилятор по такому описанию:
* сгенерирует секцию кода в объектном модуле (создаст функцию *здесь и сейчас*)
* в том числе создаст указанные в объявлении функции переменные

## Объявление функции и прототипы
Ввести функцию можно и без описания, ограничившись ее *объявлением*.
Пример объявления функции:
```c
char case_up(char c);
```
Подобную форму объявления функций еще называют *прототипом*, так как в данном случае указывается и тип возвращаемого значения и типы принимаемых. Эта форма появилась со стандарта ANSI C (C90) и является основной.

До C90 использовалась форма без указания принимаемых значений:
```c
char case_up();
```
то есть это объявление функции, но его нельзя считать прототипом.

В общем случае такие упоминания функций не будут откомпилированы сразу в объектный код, для этого недостаточно данных. 

Компилятор будет знать что такая функция *где-то есть*, и *при встрече ее вызова*, сверит типы параметра и возвращаемого значения. Это необходимо, чтобы компилятор при встрече незнакомой ему функции не выдал ошибку.

## Функции без возвращаемых значений
Функции/подпрограммы могут не возвращать никакого результата по завершению своей работы (что аналогично процедурам в Pascal). Для такого случая предусмотрено ключевое слово `void` *вместо* типа функции
Пример:
```c
void print_n_chars(char c, int n)
{
	int k;
	for(k = 0; k < n; k++)
		printf("%c", c);
}
```
где `%c` означает печать символа с заданным кодом, код задается параметром `c`
