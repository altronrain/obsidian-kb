### Прямая адресация
*Прямая адресация* -- та, при которой адрес задается явно. 
Так, например, мы можем создать метку, и потом обратиться по ее адресу:
```nasm
section .text
;...
count   dd 0

mov edx, [count]
```

### Косвенная адресация
*Косвенная адресация* -- та, при которой *адрес*, используемый для доступа к памяти, *будет вычисляться во время исполнения программы*.
Процессор i386 для косвенной адресации позволяет использовать только значения в регистрах процессора.
Так, например, если вычисляемый адрес записывать в регистр `EAX`, то пример работы с такими адресами будет выглядеть следующим образом:
```nasm
mov ebx, [eax]
```

### Исполнительный адрес
Адрес, по которому очередная машинная команда произведет обращене к памяти (неважно каким образом он задан) называется *исполнительным адресом*.
Процессор i386 позволяет задавать исполнительный адрес так, чтобы он вычислялся *в ходе выполнения команды*:
![[Pasted image 20241124152855.png]]
Пример:
```nasm
mov eax, [array+ebx+2*edi]
```
Каждое слагаемое является необязательным, что-то можно упустить

> [!NOTE] Особенности NASM
> NASM позволяет немного нарушить структуру исполнительного адреса.
> Так, если требуется пройти в цикле с регистром `ECX` все элементы от конца к началу, то можно использовать запись:
> ```nasm
> mov ecx, 1000
> mov eax, 0
> lp: add eax, [array+4*ecx-4] 
> loop lp
>```
>Что соответствует записи `array+4*(ecx-1)`  -- ведь если счетчик пробегает значения от начального (n) до 1 (в нуле уже не выполняется), то элементы массива пробегают значения от (n-1) до 0
>Так как записать `ecx-1` не можем, то просто вычитаем `4`
>NASM *перед* трансляцией сам вычтет 4 из `array` и в таком виде оттранслирует

### Команда `lea`
Для вычисления адреса без непосредственного обращения к памяти можно использовать команду `lea` (load effective address). Ограничения аналогичны [[Команда пересылки]]

Команда `lea` используется для вычисления адреса. Данная команда только вычисляет адрес и заносит его в регистр, не выполняя обращения к памяти (по вычисленному адресу).
```nasm
lea var1, [var2]
```
Имеет два операнда:
* регистр (32 или 16 бит), в который будет занесен результат
* исполнительный адрес
Применять команду можно и для целочисленных рассчетов, несмотря на операнд типа "память"
