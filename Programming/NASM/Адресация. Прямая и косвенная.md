### Прямая адресация
*Прямая адресация* -- та, при которой адрес задается явно. 
Так, например, мы можем создать метку, и потом обратиться по ее адресу:
```nasm
section .text
;...
count   dd 0

mov edx, [count]
```

### Косвенная адресация
*Косвенная адресация* -- та, при которой *адрес*, используемый для доступа к памяти, *будет вычисляться во время исполнения программы*.
Процессор i386 для косвенной адресации позволяет использовать только значения в регистрах процессора.
Так, например, если вычисляемый адрес записывать в регистр `EAX`, то пример работы с такими адресами будет выглядеть следующим образом:
```nasm
mov ebx, [eax]
```

### Исполнительный адрес
Адрес, по которому очередная машинная команда произведет обращене к памяти (неважно каким образом он задан) называется *исполнительным адресом*.
Процессор i386 позволяет задавать исполнительный адрес так, чтобы он вычислялся *в ходе выполнения команды*:
![[Pasted image 20241124152855.png]]
Пример:
```nasm
mov eax, [array+ebx+2*edi]
```
Каждое слагаемое является необязательным, что-то можно упустить

Для вычисления адреса без непосредственного обращения к памяти можно использовать команду `lea` (load effective address) [[Команды. Машинные коды]]
