## Директивы для отведения памяти
Написанные условные обозначения машинных команд ассемблер транслирует в *образ области памяти* с данными. При запуске программы в эту область памяти будет передано управление (адрес будет записан в `EIP`) и CPU начнет *выполнение* программы.

Ассемблер можно использовать для создания образа области памяти, содержащей данные, а не команды. Для этого ассемблеру нужно:
* сообщить, сколько памяти потребуется под те или иные нужды
* задать значения, которые в этму память буду помещены перед стартом программы

### Указание секций
Для указания секций программы используется директива `section`. См. [[Programming/NASM/Структура программы#Секции исполняемого файла|Структура программы#Секции исполняемого файла]]
* секция исполняемого кода (`.text`)
* секция областей памяти с заданным начальным значением (инициализированные данные) (`.data`)
* секция областей памяти без задания начальных значений (неинициализированные данные) (`.bss`) (содержит только размер)

Стоит отметить, что содержимое секции `.data` *влияет* на итоговый *размер исполняемого файла*, в то время как содержимое секции `.bss` *не оказывает влияния* на размер исполняемого файла

Константы (неизменяемые данные) выгодно и удобно располагать в секции `.text`, так как всегда будет храниться один экземпляр в памяти независимо от количества запущенных копий программы

### Резервирование памяти
Сообщить о потребностях в оперативной памяти можно с помощью *директив резервирования памяти*, которые делятся на два типа:
* директивы резервирования неинициализированный памяти
* директивы задания исходных данных
Обычно перед этими директивами ставится метка, чтобы можно было ссылаться с ее помощью на адрес в памяти, где ассемблер отвел требуемые ячейки

Директивы резервирования памяти в NASM используют в качестве единиц измерения объема памяти байты и слова (а также двойные и учетверенные слова)
> [!NOTE] Слово и машинное слово
> Тут есть нюанс, что 32-битный процессор имеет *машинное слово* 32-бит
> Однако устоявшаяся терминология и размер у программистов:
> * слово `word` -- 2 байта (16 бит)
> * двойное слово `dword` -- 4 байта (32 бит)
> * учетверенное слово `qword` -- 8 байт (64 бита)

### Резервирование неинициализированной памяти
Обычно используются для секции `.bss`

*Директивы резервирования неинициализированной памяти* приказывают ассемблеру выделить заданное количество ячеек памяти. Ничего кроме количества не уточняется
Для резервирования используются директивы:
* `resb` -- для однобайтных ячеек
* `resw` -- для слов (2 байта)
* `resd` -- для двойных слов (4 байта)
После директивы указывается число ячеек указанного размера, которое необходимо зарезервировать.
С указанием меток (для возможности перехода к указанной области памяти) резервирование памяти может выглядеть следующим образом:
```nasm
string resb 20     ; 20-bite reserved for string "var"
count  resw 256    ; 256-words reserved for counters
x      resd 1      ; 1-dword for number or address
```

### Задание исходных данных
Обычно используются для секции `.data`

*Директивы задания исходных данных* не только резервируют память, но и указывают какие значения должны в этой памяти находиться к моменту запуска программы
Значения указываются после соответствующей директивы через запятую. Памяти будет выделено столько, сколько указано значений.
Директивы:
* `db` -- для задания однобайтных значений
* `dw` -- для задания слов (2 байта на значение)
* `dd` -- для задания двойных слов (4 байта на значение)

С указанием меток (для возможности перехода к указанной области памяти) задание значений может выглядеть следующим образом:
```nasm
fibon  dw 1, 1, 2, 3, 5, 8, 13, 21
```
