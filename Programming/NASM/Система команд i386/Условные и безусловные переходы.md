Последовательное выполнение команд можно нарушить, выполнив *передачу управления* (переход)
Передача управления *принудительно* записывает новый адрес в регистр `EIP`, заставляя процессор продолжить выполнение с другого места.

Различают:
* безусловный переход (передача управления в другое место без проверок)
* условный переход (передача управления в зависимости от результата проверки)

## Дальность переходов
Все команды передачи управления в процессоре i386 делятся на три типа в зависимости от "дальности" такой передачи:
* *дальние* (far) переходы -- передача управления во фрагмент программы в *другом сегменте* (для "плоской" модели памяти не применимо, в т.ч. в Unix-подобных ОС)
  См. [[Особенности разработки в мультизадачных ОС#Виртуальная память]]
  
* *близкие* (near) переходы -- передача управления в произвольное место внутри одного сегмента, (явное изменение значения `EIP`). В "плоской" модлели памяти позволяет "прыгнуть" в *произвольное место в адресном пространстве*

* *короткие* (short) переходы -- используются (для оптимизации), в случае если точка, куда надлежит "прыгнуть" отстоит от текущей команды на 127 байт вперёд или 128 байт назад (в машинном коде такой команды смещение задается одним байтом)

Тип перехода можно указать явно поставив после команды `short` или `near`
По умолчанию (без указания):
* для безусловных переходов используется расстояние `near` (что ок)
* для условных переходов используется `short` (что может стать проблемой)

## Безусловный переход
Безусловный переход выполняется программой `jmp`, которая принимает *один операнд* определяющий адрес, куда передать управление
*Прямые* переходы:
* непосредственный операнд (метка) (переход по адресу метки)
*Косвенный* переход:
* регистровый операнд (переход по адресу из регистра)
* операнд типа "память" (переход по адресу, содержащемуся в памяти по указанному адресу)
Примеры:
```nasm
jmp cycle  ; прямой переход на метку cycle
jmp eax    ; косвенный переход по адресу из регистра EAX
jmp [addr] ; косвенный переход по адресу, содержащемуся в памяти,
           ; помеченной как метка addr
jmp [eax]  ; косвенный переход по адресу, прочитанному из памяти,
           ; расположенной по адресу, взятому из регистра EAX
```

> [!NOTE] Особенность прямых переходов
> Прямые переходы под капотом выполняются с *относительной адресацией*. Ассемблер вычисляет разницу адресов межу меткой и а адресом команды, следующей за `jmp` 
> Косвенные переходы выполняются по абсолютному адресу без подобных вычислений

## Условный переход
Команд условных переходов довольно много. Решение о переходе может зависеть от флага, комбинации флагов или даже от значения регистра

*Все* команды условных переходов поддерживают *только* непосредственный операнд (обычно метку)

Для условных переходов под капотом используется *относительная адресация*

### Простейшие переходы
Простейшие переходы зависят от значения *одного* флага
![[Pasted image 20241125230238.png]]
Обычно такие флаги стоят сразу после арифметических операций, например сравнений (`cmp`)
Примеры:
```nasm
; Проверка на равенство
cmp eax, ebx
jz are_equal

; проверка на неравенство a < b для знаковых
cmp a, b
jl less_then
; или синоним
jnge not_greater_or_equal

; проверка на неравенство a < b для беззнаковых
cmp a, b
jc carry_flag
; или синонимы
jb below
jnae not_above_or_equal
```
Слова-конструкты `less` и `greater` применяются в *знаковых* условных переходах
Слова-конструкты `below` и `above` применяются в *беззнаковых* условных переходах
![[Pasted image 20241125231335.png]]

### Регистровые переходы
Регистровые условные переходы представлены командами `jcxz` и `jecxz`
* команда `jcxz` производит переход если в регистре `CX` ноль
* команда `jecxz` производит переход если в регистре `ECX` ноль
обе команды не учитывают флаги. Обе команды имеют короткую (short) дальность

Полезно при организации циклов (см. [[Ветвления и циклы]]) для предотвращения *перевыполнения*
