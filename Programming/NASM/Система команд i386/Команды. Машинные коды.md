Краткая справка по мнемоникам 32-bit Intel процессоров:
* `mov var1, var2` поместить содержимое/значение `var2` в `var1`
* `inc var1` увеличить значение `var1` на единицу
* `dec var1` уменьшить значение `var1` на единицу
* `cmp var1, var2` сравнить значение `var1` с `var2`
  результат сравнения записывается в *регистр флагов*

### `mov`
Команда `mov` -- команда пересылки данных из одного места в другое. Данная комнада только копирует данные, не выполняя никаких преобразований.
```nasm
mov var1, var2
```
Имеет два операнда:
* первый задает назначение (место, куда будут помещены данные)
* второй задает источник (место, откуда будут взяты данные)
> [!NOTE] Мнемоники и машинные команды
> Важно помнить, что одной мнемоникой могут обозначаться различные машинные команды, выполняющие схожие действия (для удобства программиста)
> Так на примере `mov`:
> ```nasm
> mov eax, 2
> mov eax, [x]
> mov [x], eax
> mov [x], al
> ```
> указанные команды обозначают разные машинные команды, имеют разные значения машинного кода и даже занимают разное количество байт в памяти


### `lea`
Команда `lea` используется для вычисления адреса. Данная команда только вычисляет адрес и заносит его в регистр, не выполняя обращения к памяти (по вычисленному адресу).
```nasm
lea var1, [var2]
```
Имеет два операнда:
* регистр (32 или 16 бит), в который будет занесен результат
* исполнительный адрес см. [[Адресация. Прямая и косвенная#Исполнительный адрес]]
Применять команду можно и для целочисленных рассчетов, несмотря на операнд типа "память"
