## Особенности построение ветвлений и циклов
В отличие от ЯП высокого уровня, на ассемблере при реализации конструкций ветвления (if-else) или циклов (while, for) *часто* приходится сторить их по *противоположному условию*
Так например цикл `while a = 0 do` на логике ассемблера будет выглядеть как:
```nasm
cmp dword [a], 0
jnz cycle             ; jump if not zero
```

В общем же случае цикл c предусловием:
```
while <условие> do <тело>
```
средствами ассемблера реализиуется как:
```nasm
cycle:  <вычисление условия>
        JNx cycle_quit           ; выход
        <выполнение тела>
        JMP cycle                ; повтор
cycle_quit:
```

Ветвление в полном варианте:
```
if <условие> then <ветвь1> else <ветвь2>
```
средствами ассемблера реализиуется как:
```
       <вычисление условия>
       JNx else_branch      ; на ветку else
       <выполнение ветви1>
       JMP if_quit          ; обход ветки else
else_branch:
       <выполнение ветви2>
if_quit:
```

Где в обоих случаях `JNx` обоначен условный переход по *невыполнению* условия (который выполняется, если условие оказалось ложно)

В случаях перехода по невыполнению лучше применять мнемоники содержащие `n`, чтобы лишний раз подчеркнуть этот факт. см. [[Условные и безусловные переходы#Условный переход]] 

## Циклы и регистр `ECX`
Регистр `ECX` имеет особую роль как *счетчик цикла* (в системе команд i386 имеются специальные команды для построения циклов с `ECX` в роли счетчика)

Команда `loop` используется для организации циклов с заранее известным количеством итераций. В качестве счетчика она использует регистр `ECX`, куда перед началом цикла заносится число нужных итераций
Команда выполняет два действия:
* уменьшает на единицу значение в регистре `ECX`
* производит переход на заданную метку если значение в регистре `ECX` не стало равным нулю
Команда может совершать только короткие переходы (см. [[Условные и безусловные переходы#Дальность переходов]])

Аналогичные действия можно выполнить и с помощью любого другого регистра общего назначения (и даже самого `ECX`) двумя командами:
```nasm
dec eax
jnz lp
```
Однако команда `loop` занимает меньше памяти
При этом команда `loop` на многих процессорах работает медленнее

Команды `loope` и `loopz` работают одинаково. Принцип работы:
* уменьшает на единицу значение в регистре `ECX`
* производит переход на заданную метку если значение в регистре `ECX` не стало равным нулю и установлен флаг `ZF`
Команды `loopne` и `loopnz` работают одинаково. Принцип работы:
* уменьшает на единицу значение в регистре `ECX`
* производит переход на заданную метку если значение в регистре `ECX` не стало равным нулю и *не* установлен флаг `ZF`
Таким образом, любые версии команды `loop` в любом случае уменьшают значение регистра `ECX` на единицу

## Пропуск тела цикла по регистровому условию
Также с регистрами `CX` и `ECX` связан ряд команд условных переходов (см. [[Условные и безусловные переходы#Регистровые переходы]])
Переход с командами `jcxz` и `jecxz` выполняется если в соответствующих регистрах `CX` или `ECX` находится `0`, что полезно при организации циклов.

Так, если до начала цикла в регистре был `0`, то без данных переходов сначала выполнится тело цикла, затем `loop` уменьшит значение на единицу, в результате чего цикл выполнится ~ $2^{32}$ раз пока дойдет до нуля (а предполагалось что цикл не выполнится ни разу). Условный переход по проверке на ноль позволит обойти тело цикла
