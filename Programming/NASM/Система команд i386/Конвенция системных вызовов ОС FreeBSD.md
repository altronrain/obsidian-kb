Для системных вызовов на платформе i386 ядро FreeBSD использует:
* программное прерывание с номером `80h`
* номер системного вызова предается через регистр `EAX`
* если системный вызов принимает параметры, то они передаются через стек
все параметры системных вызовов 4-х байтные, либо целочисленные, либо адресные

Результат выполнения вызова возвращается через регистр `EAX`, при этом если произошла ошибка, то устанавливается флаг `CF`, а в `EAX` будет код ошибки, если же ошибки не было, флаг `CF` не выставляется

Ядро FreeBSD предполагает передачу управления путем обращения к процедуре следующего вида:
```nasm
kernel:
		int 80h
		ret
```
в таком случае код возврата при вызове окажется на вершине стека, а параметры будут под ним. Ядро FreeBSD учитывает это и ничего не делает с числом на вершине стека, извлекая параметры ниже.
Совершить системный вызов можно и без такой процедуры, однако нужно не забыть перед выполнением команды `int` занести в стек дополнительное двойное слово, которое проигнорирует ядро FreeBSD.

После выполнения системного вызова и возврата из него необходимо очистить стек смещением указателя (увеличением регистра `ESP`) командой `add`

С некоторыми вызовами нужно быть предельно внимательным, так для `read`, параметры вызова которого аналогичны `write`, очень важно анализировать возвращаемое значение (в `EAX`). 
Если там положительное число, то оно означает количество прочитанных байтов (которое может оказаться меньше "заказанного к прочтению" в параметрах, но не может его превышать)
Если там `0`, то это означает ситуацию "конец файла", больше читать нечего. 
Всегда стоит анализировать возвращаемое значение вызова `read`
