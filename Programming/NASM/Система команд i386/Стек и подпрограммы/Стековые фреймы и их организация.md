См. [[Programming/NASM/Система команд i386/Стек и подпрограммы/Подпрограммы|Подпрограммы]]

Область стековой памяти, содержащая связанные с *одним* вызовом *значения параметров, адрес возврата и локальные переменные называется* *стековым фреймом*

Еще раз о процессе вызова подпрограммы:
* параметры вызова в стеке размещает *вызывающая* программа
* при вызове в стек заносится адрес возврата
* место под локальные переменные резервирует в стеке *вызванная подпрограмма*

К содержимому стекового фрейма можно обращаться используя в качестве точки отсчета *адрес возврата*
Так адрес возврата оказывается в `ESP`, к параметрам можно обращаться как `[esp+4],[esp+8]...`, а к локальным переменным как `[esp-4],[esp-8]...`

Но `ESP` использовать неудобно, так как стек еще может потребоваться самой процедуре, или вызову другой подпрограммы. По этому обычно первым делом после получения управления значение из `ESP` копируется в другой регистр (чаще в `EBP`)
и используют для доступа к данным фрейма именно его.
Перед возвратом управления обычно его возвращают в исходное положение, указывающим на адрес возврата стекового фрейма (загоняя значение из `EBP`)

Но регистр `EBP` один, а вызовов подпрограмм может быть множество. Для решения вопроса этого вопроса используется простое правило:

**Каждая подпрограмма должна сама хранить старое значение `EBP` и восстановить его перед возвратом управления**

Для хранения `EBP` используется тот же стек. Так, сразу после передачи управления обычно выполняется команда:
```nasm
push ebp
```
таким образом *старое* значение `EBP` помещается в стек непосредственно *после адреса возврата* (выше).
Так, значение `EBP` в стеке становится адресом "привязки" и точки отсчета. 
Для этого выполняется
```nasm
mov ebp, esp
```
в результате чего регистр `EBP` указывает на то место в стеке, где находится его сохраненное значение (`ESP` указывал на `EBP`, мы поместили в `EBP` значение `ESP`)

Далее обращения к элементам стекового фрейма можно строить по принципу `[ebp+4]... [ebp-4]...`
Адрес возврата будет находиться по адресу `[ebp+4]`

Память под локальные переменные отсчитывается вычитанием нужного числа из текущего значения `ESP`, так после `push` и `mov` выделить 16 байт можно командой:
```nasm
sub esp, 16
```

Итого, каждая подпрограмма в начале своей работы должна выполнить:
```nasm
push ebp            ; сохранение предыдущего значения регистра ebp
mov ebp, esp        ; значение текущего указателя стека (на ebp) в регистр ebp
sub esp, X          ; выделение X байт памяти под локальные переменные
```
а в завершение своей работы:
```nasm
mov esp, ebp        ; указатель стека указывает на предыдущее значение ebp
pop ebp             ; восстановление предыдущего значения ebp
ret                 ; возврат управления
```

> [!NOTE] Специальные команды обработки стекового фрейма
> У процессора также есть специальные команды `enter` и `leave`, выполняющие эти действия и сокращающие запись (а также меньше расходующие память), но они работают медленнее, потому *практически не используются*
> C ними запись была бы вида:
> ```nasm
> ; вход
> enter 16, 0
> 
> ; выход
> leave
> ret
> ```

В Unix-подобных ОС стек сздается автоматически при запуске любой задачи, а во время ее исполнения при необходимости увеличивает размер доступной для стека памяти.
После появления в Unix-системах легковесных процессов (тредов) на размер стека было наложено ограничение в 8 MB, при превышении процесс будет уничтожен.
