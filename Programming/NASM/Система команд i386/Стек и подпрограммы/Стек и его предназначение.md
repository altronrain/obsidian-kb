## Строение и применение стека
В низкоуровневом программировании понятие стека *уже* определен для структуры данных в программировании
Здесь стек это *непрерывная область памяти*, для которой в специальном регистре хранится *адрес вершины стека*
* память выше вершины (с адресами меньшими адреса вершины) считается *свободной*
* память от вершины до конца области (включая саму вершину) -- считается занятой

Также определены операции добавления в стек и извлечения из стека:
* добавление в стек уменьшает адрес вершины (сдвигая яя вверх), в новую вершину записывают добавляемое значение
* извлечение считывает значение вершины стека и сдвигает вершину вниз, увеличивая ее адрес

> [!NOTE] Направление роста стека
> НАправление роста стека вообще зависит от реализации CPU. Стек может:
> * расти вверх (в направлении увеличения адресов)
> * расти вниз (в направлении убывания адресов)
> * а может и вовсе контролироваться/выбираться направление
> * или стек организуется цикличекски

## Работа со стеком в системе команд i386
Регистр, хранящий адрес вершины, называется *указателем стека* (`SP`, а фактически `ESP`). Хоть он и относится к регистрам общего назначения, фактически он всегда содержит указатель на вершину стека.
Стек растет в сторону уменьшения адресов, то есть при помещении очередного элемента в стек адрес в регистре `ESP` *уменьшается*, и обратное при извлечении элементов
* в стек можно помещать и извлекать *слова* и *двойные слова* (рекомендуется последнее)
* отдельные байты помещать в стек запрещено

Команда добавления элемента в стек (`push`) и извлечения элемента из стека (`pop`) имеют *один* операнд:
* регистровый
* типа "память" с указанием размера (`word` или `dword`)
Обе команды совмещают операции:
* копирования данных (на вершину или из нее)
* сдвиг самой вершины (изменение значения `ESP`)

Для обращения к значению на вершине стека без извлечения можно использовать команду `mov`:
```nasm
mov eax, [esp]
```
кроме того подобным образом можно обратиться не только к данным на вершине стека (не стоит использовать ячейки выше вершины, ибо ОС может затереть данные там)

Пример работы со стеком на задаче переворота строки символов в обратном порядке, где:
* `ESI` и `EDI` -- указатели на начало строки (оба)
* `ECX` используется для подсчета элементов и как счетчик цикла
* `EBX` (его младшая часть `BL`) содержит текущий символ
* `0` -- конец строки
задача решается за два цикла:
* запись строки в стек пока не дойдем до конца строки
* чтение из стека с записью с начала строки (получаем обратный порядок)
```nasm
        xor ebx, ebx      ; обнуляем ebx
        xor ecx, ecx      ; обнуляем ecx
lp:     mov bl, [esi+ecx] ; очередной байт из строки
        cmp bl, 0         ; проверка на конец строки
        je lpquit         ; если конец строки -- конец цикла
        push ebx          ; запись bl в составе ebx в стек
        inc ecx           ; следующий индекс строки
        jmp lp            ; повторить цикл
lpquit: jecxz done        ; если строка пустая -- конец
        mov edi, esi      ; опять начинаем с начала строки
lp2:    pop ebx           ; извлекаем элемент с вершины стека
        mov [edi], bl     ; записываем символ в место строки
        inc edi           ; следующий адрес в строке
        loop lp2          ; повторить цикл ecx раз
done:
```

## Дополнительные команды работы со стеком
В стек можно занести значение всех регистров общего назначения одной командой.
Команда `pushad` (push all doublewords) заносит в стек содержимое регистров:
* `EAX, ECX, EDX, EBX, ESP, EBP, ESI, EDI` в указанном порядке (см `EBX`)
* значение `ESP` заносится такое, какое было *до выполнения* команды

Аналогичным образом можно извлечь из стека содержимое всех регистров общего назначения.
Команда `popad` (pop all doublewords) извлекает из стека восемь четырёхбайтных значений и заносит их в регистры общего назначения:
* `EDI, ESI, EBP, ESP, EBX, EDX, ECX, EAX` в указанном порядке
* значение `ESP` извлекается но *не записывается* -- игнорируется

Регистр флагов может быть занесен в стек командой `pushfd` и извлечен командой `popfd`. При работе в ограниченном (пользовательском) режиме некоторые флаги (только для привелегированного режима) командой `popfd` будут проигнорированы, а изменены только те, которые можно изменять в пользовательском режиме.
