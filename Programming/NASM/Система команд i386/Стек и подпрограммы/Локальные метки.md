По меткам см. [[Виды операндов#Адресный операнд]]

В подпрограммах, пусть даже на языке ассемблера скорее всего понадобятся метки. При этом возникает неободимость исключить пересечение имен меток с основным телом программы и другими подпрограммами, чтобы не вспоминать все имена меток в программе и использовать имена эффективно.

Ассемблер NASM предусматривает для решения этой задачи *локальные метки*
Они синтаксически отличаются от обычных меток тем, что начинаются с символа точки `.`
Это особенность ассемблера NASM

Ассемблер локализует такие метки во фрагменте программы, *ограниченном с обеих сторон обычными (нелокальными) метками*. То есть он рассматривает ее не саму по себе, а как нечто *подчиненное*/*вложенное* последней (ближайшей сверху) нелокальной метке.

Так, появляется возможность для следующего наименования циклов в подпрограммах:
```nasm
first_proc:    ; один фрагмент программы
      ; ...
.cycle:        ; локальная метка цикла
      ; ...
second_proc:   ; второй фрагмент программы
      ; ...
.cycle:        ; локальная метка цикла
      ; ...
```


> [!NOTE] Как достигается этот эффект
> На самом деле NASM поступает тут довольно хитро. Видя метку, начинающуюся с точки, он подставляет к ней спереди имя первой метки *без* точки выше по тексту. 
> 
> Таким образом на самом деле имеют место быть две *разные* метки в обособленных фрагментах кода с именами:
> * `first_proc.cycle`
> * `second_proc.cycle`
>   
> Об этом желательно помнить, а также *крайне желательно не использовать* имена меток, содержащих где-то внутри себя точку не на стартовой позиции
