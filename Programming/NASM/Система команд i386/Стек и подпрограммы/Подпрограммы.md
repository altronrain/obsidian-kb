## Принцип работы с подпрограммами
Подпрограммой называется обособленная часть програмного кода, которая может быть *вызвана* из главной программы (или из другой подпрограммы)
Под *вызовом* понимается временная передача управления подпрограмме. По завершению своей работы подпрограмма *возвращает* управление *в точку, откуда ее вызвали*

При вызове подпрограммы нужно запомнить *адрес возврата*, то есть адрес машинной команды, следующей *за командой вызова*, при этом сделать это так, чтобы сама подпрограмма могла воспользоваться этим адресом для передачи управления обратно

Помимо адреса возврата подпрограммам часто требуются:
* параметры
* локальные переменные

Под всё это необходимо выделить оперативную память или регистры
Можно было бы каждой подпрограмме выделять свою область памяти заранее, но с таким подходом невозможно использовать *рекурсию* (вызов подпрограмм из самих себя, порождение новых экземпляров той же подпрограммы)
Рекурсивыне вызовы подчиняются правилу LIFO, по этому логично использовать *стек* при вызовах подпрограмм

Последовательность действий при вызове подпрограмм:
* перед вызовом в стек помещаются параметры вызова
* производится вызов подпрограммы с передачей управления, в стек сохраняется адрес возврата
* подпрограмма сама резервирует в стеке определенное количество ячеек для локальных переменных, сдвигая вершину

Область стековой памяти, содержащая связанные с одним вызовом значения параметров, адрес возврата и локальные переменные называется *стековым фреймом*
см. [[Стековые фреймы и их организация]]

## Вызов подпрограмм и возврат из них
Для вызова подпрограммы в процессоре i386 предусмотрена команда `call`
Аналогично команде `jmp` у команды `call` предусмотрен *один* аргумент:
* непосредственный, обычно метка а не адрес вручную
* регистровый (адрес передачи управления находится в регистре)
* типа "память" (адресный) (переход по адресу, прочитанному из заданного места памяти)

В плане расстояния перехода для `call` не существует "короткой" формы, а дальняя не требуется в виду концепции "плоской памяти" и отсутствия сегментов, итого всегда используется "близкая" дальность (`near`) (см. [[Условные и безусловные переходы#Дальность переходов]])

Для возврата из подпрограммы предусмотрена команда `ret` (return)
В простейшей форме команда *не имеет* операндов
При выполнении команды процессор извлекает четыре байта с вершины стека и записывает их в регистр `EIP`, в результате чего управление передается по адресу, который находился в вершине стека.

Пример подпрограммы по заполнению области памяти однобыйтным значением (символом), где:
* `EDI`  содержит адрес области памяти который нужно заполнить
* `ECX` содержит количество ячеек для заполнения
* `AL` содержит само значение для записи

Процессор i386, когда точно известно сколько параметров передано, имеет особую форму команды `ret` с *одним* операндом
Он может быть только непосредственным и имеет длину два байта (слово), задает количество памяти в байтах, занятой параметрами функции.
Пример:
```nasm
ret 12
```
Эта команда извлечет из стека адрес возврата и передаст по нему управление, но одновременно с этим увеличит `ESP` на заданное число избавляя *вызвавшего* от необходимости *очистки стека*
см. для компиляторов каких ЯП это применимо [[Конвенции вызовов подпрограмм]]
