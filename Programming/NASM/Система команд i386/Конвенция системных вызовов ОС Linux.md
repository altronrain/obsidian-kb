Для системных вызовов на платформе i386 ядро Linux использует:
* программное прерывание с номером `80h`
* номер системного вызова предается через регистр `EAX`
* если системный вызов принимает параметры, то они располагаются в регистрах `EBX, ECX, EDX, ESI, EDI` и в очень редких случаях `EBP`
все параметры системных вызовов 4-х байтные, либо целочисленные, либо адресные

Результат выполнения вызова возвращается через регистр `EAX`, причем значения между `fffff000h` и `ffffffffh` сигнализируют об ошибке и представляют собой условный код ошибки

Пример вызова `write` для вывода информации на поток вывода:
* номер вызова (`4`) помещается в `EAX`
* номер стандартного потока вывода (`1`) помещается в `EBX`
* адрес строки в памяти помещается в `ECX`
* длина строки помещается в `EDX`
* команда программного прерывания `int 80h`
Пример вызова `_exit` для завершения работы программы:
* номер вызова (`1`) помещается в `EAX`
* код завершения помещается в `EBX`

С некоторыми вызовами нужно быть предельно внимательным, так для `read`, параметры вызова которого аналогичны `write`, очень важно анализировать возвращаемое значение (в `EAX`). 
Если там положительное число, то оно означает количество прочитанных байтов (которое может оказаться меньше "заказанного к прочтению" в параметрах, но не может его превышать)
Если там `0`, то это означает ситуацию "конец файла", больше читать нечего. 
Всегда стоит анализировать возвращаемое значение вызова `read`
