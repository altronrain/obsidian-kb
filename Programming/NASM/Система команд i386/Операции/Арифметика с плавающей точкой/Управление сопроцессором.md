Еще раз напоминание про регистры:
* через `CR` можно управлять режимом работы сопроцессора
* сопроцессор по результатам выполнения операций устанавливает содержимое `SR`
* состояние регистров стека отражено в регистре `TW`
подробнее см. [[Устройство сопроцессора]]

## Работа с регистром CR
Для работы с регистром `CR` имеется три команды: `fstcw, fnstcw, fldcw` (при этом `fstcw` на самом деле `wait` + `fnstcw`, см. [[Обработка исключительных ситуаций#Исключения и команда `wait`]])
Все три команды принимают *один* операнд *только* типа "память" (адресный) размера `word`

Команды `fstcw` и `fnstcw` записывают содержимое регистра `CR` в заданное место памяти.
Команда `fldcw` загружает содержимое регистра `CR` из памяти.

## Работа с регистром SR
Для получения содержимого регистра `SR` предусмотрены команды `fstsw` и `fnstsw` (при этом `fstsw` на самом деле `wait` + `fnstsw`, см. [[Обработка исключительных ситуаций#Исключения и команда `wait`]])
Команды принимают *один* операнд либо регистр `AX` либо типа "память" размера `word`

Загрузки не предусмотрено, так как это регистр *состояния*

Значение `TOP` в регистре `SR` можно увеличить на единицу командой `fincstp`, а уменьшить на единицу командой `fdecstp`
Действие при этом обратное, как и с обычным стеком:
* при увеличении вершины размер стека уменьшается
* при уменьшении вершины размер стека увеличивается
При этом "занятости" они не меняют, так что следует использовать осторожно. Так при уменьшении вероятно вершиной `ST0` станет пустой регистр, а при увеличении `ST7` вероятно окажется занят, так как это бывший `ST0`

Для принудительной очистки флагов исключительных ситуаций используется команда `fclex` и эквивалент без `wait` -- `fnclex` (см. [[Обработка исключительных ситуаций#Исключения и команда `wait`]])

> [!NOTE] Рекомендации
> Перед загрузкой новой конфигурации/состояния в управляющий `CR` стоит всегда выполнять очистку состояния в `SR` командой `fclex`, так как в новой конфигурации `CR` мы можем что-то демаскировать и возникнет исключение

## Работа с регистром TW
Регистр `TW` не может быть напрямую считан или записан, но существует одна команда, непосредственно влияющая на его состояние. Это команда `ffree`, имеющая *один* операнд, представляющий собой регистр `STn`.
Данная команда помечает регистр как *свободный* (пустой). Так, например, убрать число с вершины стека "в никуда" можно двумя командами:
```nasm
ffree st0
fincstp
```

## Работа со всеми регистрами
Для сброса регистров сопроцессора в "исходное состояние" используется команда `finit` или `fninit` (где `finit` есть `wait` + `fninit`, см. [[Обработка исключительных ситуаций#Исключения и команда `wait`]])

При сбросе:
* в регистр `CR` заносится значение `037Fh`:
	* округление в ближнюю сторону
	* наибольшая точность
	* все исключения замаскированы
* регистр `SR` обнуляется:
	* `TOP` = 0
	* флаги сброшены (в том числе флаги исключительных ситуаций)
* регистры `FIP` и `FDP` обнуляются
* регистр `TW` заполняется единицами, что соответствует пустому стеку
* регистры стека (хранения чисел) никак не изменяются но благодаря `TW` считаются пустыми

Командой `fsave` можно сохранить все состояние сопроцессора, то есть содержимое всех его регистров, в области памяти, чтобы затем восстановить его.
Для сохранения всего состояния сопроцессора потребуется область памяти длиной 108 байт.
Команда `fsave` имеет *один* операнд типа память, указывать его размер *не нужно*
На самом деле `fsave` представляет собой также `wait` + `fnsave`

После сохранения состояния выполняется сброс к состоянию по умолчанию (т.е. нет необходимости давать команду `finit)

Для восстановления состояния сопроцессора используется команда `frstor`. Команда так же имеет *один* операнд типа "память", и указывать размер *не нужно*

Для сохранения и восстановления только вспомогательных регистров сопроцессора используются команды `fstenv, fnstenv, fldenv`, которым требуется 28 байт памяти для сохранения/загрузки

Как и для основного процессора, для сопроцессора существует команда `fnop`, *не делающая ничего*
