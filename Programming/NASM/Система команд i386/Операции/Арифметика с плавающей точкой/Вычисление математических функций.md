Существует ряд команд для вычисления математических функций. 
## Тригонометрические функции
К ним относятся:
* `fsin` -- вычисляет синус числа, лежащего в `ST0` и помещает результат в `ST0`
* `fcos` -- вычисляет косинус числа, лежащего в `ST0` и помещает результат в `ST0`
* `fsincos` -- извлекает из стека (`ST0`) число, вычисляет его синус и косинус, кладет их в стек (размер стека по итогу увеличивается на 1):
	* синус в `ST1`
	* косинус в `ST0`
* `fptan` -- вычисляет тангенс числа, лежащего в `ST0` и помещает результат в `ST0`, после чего *добавляет* в стек `1`, в итоге:
	* размер стека увеличивается на 1
	* на вершине стека в `ST0` лежит `1`
	* в `ST1` лежит результат вычисления тангенса

Для вычисления котангенса можно после вычисления тангенса выполнить `fdivrp`, если котангенс не нужен, то можно избавиться от единицы разделив на нее командой `fdivp` или просто выкинуть из стека командой `fstp st0`

* `fpatan` -- вычисляет `arctg(y/x)`, где `x` -- значение `ST0`, а `y` -- значение `ST1`:
	* оба числа изымаются из стека
	* результат записывается в стек (размер стека по итогу становится меньше на 1)
	* знак результата совпадает со знаком `y`
	* модуль результата не превосходит числа Пи

## Логарифмические функции
Команды сопроцессора, вычисляющие различные логарифмические функции:
* `fyl2x` -- вычисляет `y*log(2)x`, где `x` -- значение `ST0`, а а `y` -- значение `ST1`:
	* оба числа изымаются из стека
	* результат записывается в стек (размер стека по итогу становится меньше на 1)

Операнд `y` требуется для того, что в реальной жизни приходится часто вычислять логарифмы не только по основанию 2, и тут помогает формула:
![[Pasted image 20241215161055.png]]
Сопроцессор умеет вычислять только логарифмы по основанию 2, что в формуле будет представлять `b`. В таком случае можно подготовиться заранее и рассчитать `y` как `1/log(2)a` и использовать его в качестве второго операнда

* `fyl2xp1` -- вычисляет `y*log(2)(x+1)`, где `x` -- значение `ST0`, а а `y` -- значение `ST1`:
	* оба числа изымаются из стека
	* результат записывается в стек (размер стека по итогу становится меньше на 1)
	* `x` не должен превосходить по модулю `1 - sqrt(2)/2`

Применяется для расчетов, когда `x` очень близок к `1`, а `log(a)1 = 0`, чего стараемся избежать. Тогда в таких близких окрестностях единицы обычная команда вычисления логарифма не применима, и логарифмируется не `x`, а *его отличие от единицы*. Результат, хоть и будет близок к нулю, но будет не нулевой.

## Иные вычисления функций
* `fsqrt` -- вычисляет квадратный корень числа, лежащего в `ST0` и помещает результат в `ST0`
* `f2xm1` -- вычисляет `2^x - 1`, где `x` -- значение `ST0`, результат заносится обратно в `ST0`
	* аргумент не должен по модулю превосходить `1`

Ситуация с последней примерно повторяет суть с вычислением логарифма при значениях близких к единице. Так `a^0 = 1` для любого положительного основания `a`, для крайне малых `x` вычисление `2^x` довольно быстро будет давать `1`, и такие вычисления как
`1 + 2^-65`
довольно быстро будет не отличить от единицы. По этому эффективнее вычислять не `2^x`, а *его отличие от единицы*
