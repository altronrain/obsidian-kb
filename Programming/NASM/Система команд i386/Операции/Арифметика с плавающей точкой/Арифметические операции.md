Все обозначения команд, имеющих отношение к арифметическому сопроцессору, начинаются с буквы `f` (от floating point)
Большинство команд не имеет операнда или принимает один операнд (и только в редких случаях 2)
В качестве операнда могут выступать:
* регистры сопроцессора (через обозначения `STn`, т.е. как стек)
* операнды типа "память" (адресные операнды)
*непосредственные операнды*, то есть числа с плавающей точкой прямо в команде, *не поддерживаются*

## Простейшие арифметические операции
Четыре действия арифметики на сопроцессоре можно выполнить командами:
```
fadd, fsub, fsubr, fmul, fdiv, fdivr
```
Что касается операндов, можно использовать форму с *одним* операндом, или с *двумя*, или *без операндов*

С форме с одним операндом он должен представлять собой адресный операнд размера `dword` или `qword`
Подробнее:
* `fadd` и `fmul` выполняют сложение и умножение регистра `ST0` с операндом
* `fsub` вычитает *операнд из* `ST0`
* `fsubr` вычитает `ST0` *из операнда*
* `fdiv` делит `ST0` *на операнд*
* `fdivr` делит *операнд на* `ST0`
результат вычислений записывается в `ST0`

В форме без операндов в качестве такового выступает содержимое `ST1`

В форме с двумя операндами роль операндов выполняют регистры сопроцессора, причем одним из них должен обязательно быть `ST0`, но он может быть как на первом так и на втором месте.
В такой форме записи команды выполняют заданное действие над первым и вторыми операндами и заносят результат *в первый операнд*.

## Выталкивающая форма команд
Каждая команда арифметической операции имеет также "выталкивающую форму"
```
faddp, fsubp, fsubrp, fmulp, fdivp, fdivrp
```
В такой форме всегда указывается *два операнда* (регистры сопроцессора), причем второй из них *должен быть* `ST0`

После выполнения операции и *занесения результата в первый операнд* эти команды убирают из стека `ST0`, то есть он помечается как пустой (и значение `TOP` увеличивается на единицу). Вытесненное значение никуда не записывается, а *результат операции* оказывается *в новой вершине стека* (таки в `ST0`)

Также можно записать команды без операндов, в таком случае используются регистры `ST1` и `ST0`

Данная форма команд используется чаще всего, ибо позволяет вычислить любое арифметическое выражение, если только оно не содержит слишком много вложенных скобок (иначе не хватит глубины стека). Для этого выражение нужно представить в *польской инверсной записи* (ПОЛИЗ, RPN reverse polish notation) (см. [[Польская инверсная запись. Алгоритм Дейкстры]])

Так, выражение `(x + y)*(1 - z)` в ПОЛИЗе будет иметь вид: `x y + 1 z - *`
И тогда выражение можно следующим образом записать командами ассемблера (представив что `x, y, z` лежат в памяти и имеют длину `qword` и содержат числа с плавающей точкой):

## Форма команд для операций с целым числом
Иногда бывают полезны команды
```
fiadd, fisub, fisubr, fimul, fidiv, fidivr
```
имеющие *один* операнд, выполняющие соответствующее арифметическое действие над `ST0` и своим операндом (типа "память" размера `word` или `dword` и рассматриваемый как *целое* число)

## Иные арифметические операции
У данных команд *нет* операндов:
* команда `fabs` вычисляет модуль `ST0`
* команда `fchs` меняет знак `ST0` на противоположный
* команда `frndint` округляет `ST0` до целого в соответствии с установленным режимом округления (результат записывается обратно в `ST0`)

Также имеются команды:
```
fprem, fprem1, fscale, fxtract
```
