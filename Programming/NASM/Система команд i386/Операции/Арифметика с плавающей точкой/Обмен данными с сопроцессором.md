Все обозначения команд, имеющих отношение к арифметическому сопроцессору, начинаются с буквы `f` (от floating point)
Большинство команд не имеет операнда или принимает один операнд (и только в редких случаях 2)
В качестве операнда могут выступать:
* регистры сопроцессора (через обозначения `STn`, т.е. как стек)
* операнды типа "память" (адресные операнды)
*непосредственные операнды*, то есть числа с плавающей точкой прямо в команде, *не поддерживаются*

## Занесение и извлечение чисел с плавающей точкой
Команда занесения значения в регистровый стек сопроцессора `fld` (float load):
* имеет один операнд вида: 
	* типа "память" размера `dword, qword, tword`
	* или регистр `STn`
Операнд при этом *должен* содержать число с плавающей точкой

Пример:
```nasm
fld st0                   ; создание копии вершины стека
fld qword [matrix+ecx*8]  ; загрузка в стек восъмибайтового числа из массива
```
При помещении элементов в стек в регистре `SR` уменьшается значение `TOP`, то есть сдвигается вершина стека.

Команда извлечения результата из сопроцессора (с *вершины* регистрового стека) `fst` и `fstp` :
* имеют один операнд вида: 
	* типа "память" размера `dword, qword, tword` (последний только для `fstp`)
	* или регистр `STn`
При этом:
* `fst` просто считывает число с вершины стека (`SR0`)
* `fstp` извлекает число из стека (`SR0` помечается свободным, увеличивается `TOP`)

Команда `fst` не умеет работать с 80-битными операндами типа память

Последовательность операций `fstp` следующая:
* сначала в принимающий операнд записывается результат
* затем затирается значение в вершине стека
потому для *уничтожения* значения на вершине стека можно применять команду:
```nasm
fstp st0
```

## Занесение и извлечение целых чисел
Используется для перевода целых чисел в формат с плавающей точкой и наоборот

Команда занесения целого числа в стек сопроцессора `fild` имеет *один* операнд типа память размера `word, dword, qword` (в данном случае 8-байтное целое)
При этом число записывается и сдвигается вершина (оно становится вершиной стека)

Команды извлечения целого числа из сопроцессора `fist` и `fistp` имеют *один* операнд типа память размера `word, dword, qword` (последнее только для `fistp`).
Данные команды берут число из `ST0`, округляют его до целого в соответствии с установленным режимом округления и записывают результат по адресу, указанному операндом.
При этом:
* `fist` просто считывает число с вершины стека (`SR0`)
* `fistp` извлекает число из стека (`SR0` помечается свободным, увеличивается `TOP`)

## Обмен содержимым регистров сопроцессора
Команда `fxch` позволяет обменять местами содержимое *вершины* (`ST0`) и любого другого регистра `STn`, который передается в качестве операнда.

Регистры *не должны быть пустыми*. 

Если требуется поменять местами `ST0` и `ST1`, то операнд можно не указывать.

## Загрузка часто употребляемых констант
Имеется ряд команд для загрузки в стек часто употребляемых констант:
* `fld1` загружает `1.0`
* `fldz` загружает `+0.0`
* `fldpi` загружает число Пи
* `fldl2e` загружает `log2_e`
* `fldl2t` загружает `log2_10`
* `fldln2` загружает `ln2`
* `lfdlg2` загружает `lg2`
данные команды не имеют операндов, помещают указанное значение на вершину стека (`TOP` сдвигается, то есть добавляемое число становится вершиной)

Какое значение будет занесено зависит также от выставленного способа округления
