Рассмотрим команды сложения и вычитания над целыми числами

## Команды `add` и `sub`
Для выполнения данных операций в ассемблере предусмотрены команды `add` и `sub` для сложения и вычитания соответственно

Эти команды имеют два операнда:
* первый задает как одно из чисел и одновременно назначение (место, куда будут помещены данные)
* второй задает второе число (место, откуда будут взяты данные)

Аналогично команде `mov`:
Первый операнд может быть регистровый или типа "память" (адресный).
Второй операнд может быть любого типа.
Также не может быть двух операндов типа память

Примеры:
```nasm
add eax, ebx
add edx, 12
add dword [x], 12
sub [x], ecx
```

В соответствии с результатом команды `add` и `sub` выставляют значения *флагов*:
```
OF SF CF ZF
```
см. [[Система регистров#Регистры специального назначения]]
Флаги `OF`  и `SF` имеют смысл для знаковых чисел
* `SF` устанавливается если результат отрицательный, иначе сбрасывается
* `OF` устанавливается если произошло переполнение
Флаг `CF` имеет смысл для беззнаковых чисел (устанавливается если произошел перенос в или заем из несуществующего разряда)
Флаг `ZF` выставляется если в результате операции получился `0`

Важно отметить, что процессор при выполнении операций `add` и `sub` не различает знаковые и беззнаковые числа, он не знает с какими числами работает. Помнить об этом -- обязанность разработчика. По этой причине `add` и `sub` устанавливают все флаги из перечисленных, а программист должен использовать только те, которые соответствуют смыслу выполняемой операции (знаковые или беззнаковые)

## Команды `adc` и `sbb` (сложение и вычитание с учетом переноса)
Наличие (выставление) флага переноса позволяет организовать *сложение и вычитание с переносом*.
Для этого предусмотрены команды `adc` и `sbb`, по принципу работы и свойствам аналогичные командам `add` и `sub`, но учитывающие значение флага переноса `CF` на момент начала выполнения операции.
* команда `adc` *добавляет* к своему итоговому результату значение флага переноса
* команда `sbb` *вычитает* из своего итогового результата значение флага переноса
после формирования результата команды заново выставляют все флаги, включая `CF` уже в соответствии с новым результатом.

Пример, имеются два 64-битных числа, первое записано в `EDX` (старшие 32-бит) и `EAX` (младшие 32-бит), а второе записано в `EBX` и `ECX` аналогично
Тогда сложение двух числе можно выполнить командами:
```nasm
add eax, ecx  ; складываем младшие части (выставляются флаги)
adc edx, ebx  ; складываем старшие части с учетом переноса
```
Пример с вычитанием:
```nasm
sub eax, ecx  ; вычитаем младшие части (выставляются флаги)
sbb edx, ebx  ; вычитаем старшие части с учетом займа
```

## Команды `inc` и `dec` (инкремента и декремента)
Специальные команды предусмотрены для увеличения и уменьшения целого числа на единицу -- `int` и `dec`
Обе команды:
* имеют *один* операнд (регистровый или типа "память")
* устанавливают флаги `ZF`, `OF`, `SF`
* *не затрагивают* флаг `CF`
При использовании с операндом типа память указание размера обязательно

## Команда `neg`
Команда для *смены знака* (унарный минус). Имеет один операнд
*Обычно* применяется к знаковым числам, однако команда устанавливает все четыре флага `ZF`, `OF`, `SF`, `CF` (так как если бы выполнялось *вычитание из нуля*)

## Команда `cmp`
Команда *сравнения*. Имеет два операнда
Производит вычитание также, как команда `sub`, только результат *никуда не записывается*. Команда вызывается для установки флагов (и обычно после нее идет команды условного перехода).
Как и для команды `sub`, *первый операнд не может быть непосредственным*
