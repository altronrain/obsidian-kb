*Макропроцессор* -- программное средство, которое получает на вход текст и, пользуясь указаниями, данными в данном тексте, частично преобразует его, давая на выходе текст, уже не имеющий указаний к преобразованию.
Применительно к ЯП результатом работы макропроцессора является текст на языке программирования, который уже можно далее компилировать в соответствии с правилами языка

## Макросы
*Макросом* называют правило, в соответствии с которым фрагмент программы, содержащий определенное слово, должен быть преобразован. Само слово называется *именем макроса* (хотя часто просто говорят макрос)

Прежде чем применять макрос, нужно его определить. Для этого необходимо:
* сообщить макропроцессору, что некий идентификатор теперь является именем макроса
* задать правила обработки этого имени
такой фрагмент программы, определяющей макрос, называется *макроопределением*

Встречая имя макроса по тексту программы процессор совершает *макровызов*, заменяя имя макроса и его параметры (при наличии) ранее заготовленным фрагментом текста. Такая замена называется *макроподстановкой*, а полученный текст -- *макрорасширением*

Вместо имен макросов можно проводить преобразования путем *макродиректив* (так, например `%include` является макродирективой, подставляя вместо себя содержимое указанного файла)

## Пример многострочного макроса
Такие макросы всегда разворачиваются в одну или несколько строк на языке ассемблера

Пример объявления макроса:
```nasm
%macro pcall1 2      ; 2 -- кол-во параметров макроса
		push %2
		call %1
		add esp, 4
%endmacro
```
где:
* `pcall1` -- имя макроса
* строки между `%macro` и `%endmacro` -- тело макроса, шаблон для подстановки
* `%1` и `%2` -- параметры вызова (вплоть до `%9`) (`%0` -- общее количество параметров)
Пример вызова макроса:
```nasm
	pcall1 myproc, dword 517
```

### Локальные метки в многострочных макросах
См. также [[Programming/NASM/Система команд i386/Стек и подпрограммы/Локальные метки|Локальные метки]]
Следует помнить о том, что макровызов разворачивается в последовательные команды в теле программы. Если внутри макроса использовать метку, то уже при втором вызове ассемблер выдаст ошибку, так как обнаружит *одинаковые* метки по телу программы.
Для решения этой проблемы предусмотрены локальные метки в макросах.
Перед именем метки в теле макроса необходимо указать два символа `%`, например:
`%%lp`. Тогда при каждом вызове макроса NASM будет заменять `%%lp` на уникальное имя.

## Пример однострочного макроса
Такие макросы заменяют собой часть строки. Так например для обращения к параметрам и локальным переменным можно определить более понятные имена:
Пример:
```nasm
%define arg1 ebp+8
%define arg2 ebp+12
%define local1 ebp-4
%define local2 ebp-8

; или в более общей форме:
%define arg(n) ebp+(4*n)+4
%define local(n) ebp-(4*n)
```
Пример обращения к параметру процедуры:
```nasm
mov eax, [arg1]
```

С такими макросами есть ряд особенностей. Их можно переопределять по ходу программы, или же вовсе "забывать" директивой `%undef`
Можно внутри одного макроса вызвать другой при определении, и тут необходимо быть осторожным в плане того, какие значения будут подставлены при разворачивании.

### Ленивая макроподстановка
Если внутри макроса `A` использовать макровызов `B`, то этот макровызов *не раскрывается на момент объявления*. Он будет раскрываться на момент макроподстановки `A`, соответственно, будет использовано то *значение* макроса `B`, которое было *актуально на момент подстановки* `A`
Пример:
```nasm
%define    thenumber    25
%define    mkvar        dd thenumber

var1       mkvar        ; var1 dd thenumber -> var1 dd 25

%define    thenumber    36

var1       mkvar        ; var1 dd thenumber -> var1 dd 36
```
такая макроподстановка называется *ленивой* (lazy), что связано с тем что макропроцессор как бы ленится выполнять макроподстановку (макроса `thenumber`) пока его к этому не вынудят

### Энергичная макроподстановка
Для *энергичной* подстановки используется директива `%xdefine`
Разница с обычным `%define` в том, что если в теле описания макроса встречаются макровызовы, то они подставляются (разрешаются) *незамедлительно*, то есть *в момент объявления*, не дожидаясь вызова пользователем.
Пример:
```nasm
%define    thenumber    25
%xdefine    mkvar        dd thenumber   ; -> dd 25

var1       mkvar        ; var1 dd 25

%define    thenumber    36

var1       mkvar        ; var1 dd 25
```
переопределение макроса `thenumber` не влияет на работу макроса `mkvar` (так как под капотом там уже не содержится вызова `thenumer`, а содержится `25`)

### Макропеременные
Помимо директив `%define` и `%xdefine` существует еще директива `%assign`.
Она используется для *связи с именем макроса числа*, получаемого в результате вычисления *арифметического выражения*
То есть `%assign` не только выполняет все подстановки в теле макроопределения, но и пытается *вычислить* тело как целочисленное арифметическое выражение (в случае неудачи фиксируется ошибка)
Пример:
```nasm
%assign      var 25
; ...
%assign      var+1  ; var -> 26
```
далее в тексте программы ассемблер будет подставлять значение `26` если встретит слово `var`
Такие макроимена, вводимые директивой `%assign` называются *макропеременными*
