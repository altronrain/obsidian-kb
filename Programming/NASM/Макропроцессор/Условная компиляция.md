*Условная компиляция* (включение определенных фрагментов кода в компилируемую программу *по условию*) в NASM выполняется средствами макропроцессора.

## Условная компиляция по наличию/отсутствию макросимвола
Для условной компиляции используются директивы:
* `%define`
* `%ifdef`
* `%endif`
Код, помещенный в блок директив `%ifdef <однострочный_макрос>`, `%endif` будет выполнен, только если этот однострочный макрос определен директивой `%define <однострочный_макрос>`
Пример:
```nasm
%define DEBUG_PRINT
;...
%ifdef DEBUG_PRINT
	PRINT "Entering section"
%endif
;
; исследуемая секция
;
%ifdef DEBUG_PRINT
	PRINT "Leaving section"
%endif
```
Код, печатающий текст, будет выполнен. Если же закомментировать `%define`, то код не выполнится

Можно вообще не добавлять в код `%define`, а воспользоваться ключом `-d` при вызове компилятора:
```shell
nasm -f elf -dDEBUG_PRINT prog.asm
```

Также существует директива `%elifdef` все так же для проверки *наличия* макросимвола, и
* `%ifndef`
* `%elifndef`
для проверок *отсутствия* макросимвола
Действуют они как и в ветвлениях конструкции `if..elif..else` в иных ЯП

## Условная компиляция по выражению
Существует более общий класс условных директив, в которых для принятия решения вычисляется арифметико-логическое выражение на этапе компиляции (соответственно, оно является [[Критические выражения]], то есть должно быть вычислимым)
К таким директивам относятся:
* `%if`
* `%else`
* `%elif`
К поддерживаемым операциям при вычислении относятся:
* арифметические: `=, >, <, >=, <=, !=` (вместо `!=` можно указать `<>`, вместо `=` -- `==`)
* логические: `&&, ||, ^^` (и, или, исключающее или)

## Условная компиляция по совпадению строк
Директивы `%ifidn` и `%ifidni` принимают два аргумента -- строки, и сравнивают их.
Код, следующий за директивами, будет выполнен только в случае равенства строк.
* `%ifidn` требует точного совпадения
* `%ifdni` игнорирует регистр
Противоположное условие (неравенства строк) проверяется директивами `ifnidn` и `%idnidni`
Существуют так же `elif-` формы:
`%elifidn, %elifidni, %elifnidn, %elifnidni`

## Условная компиляция по наличию многострочного макроса
Действие аналогично предыдущим. Проверяет наличие указанного многострочного мароса. Директивы:
`%ifmacro, %ifnmacro, %elifmacro, %elifnmacro`

## Условная компиляция по проверке аргумента
Дийствие аналогично предыдущим. Проверка переданного аргумента на соответствие  определенному типу:
* `%ifid` -- аргумент идетификатор
* `%ifstr` -- аргумент строка
* `%ifnum` -- аргумент число
А также формы вида: `%ifnXXX, %elifXXX, %elifnXXX`

## Вызов ошибки при компиляции
Помимо этого существует директива `%error`, при нахождении которой ассемблер NASM выдаст ошибку. Полезно применять при невыполнии всех поставленных условий
