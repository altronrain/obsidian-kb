## Общие свойства систем счисления
Система счисления определяет, *как будет записано число*. Сам *число* и его свойства никак *не зависят от системы счисления*.

Число, обозначающее одновременно число используемых цифр и то, во сколько раз каждый следующий разряд числа больше предыдущего, называется *основанием системы счисления*.
Это свойство следует из [[Элементы комбинаторики#Размещения с повторениями]]
Разместить *n* цифр по *k* разрядам дает нам *n^k* чисел от 0 до *n^k -1*. Дальше придется добавлять разряд.

Из этого следует первое свойство (разложения на сумму по степеням основания):
Если число представлено цифрами ![[Pasted image 20241029220603.png]]
То его численно значение будет равняться сумме: ![[Pasted image 20241029220652.png]]
Пример: Число 3275 в десятичной системе счисления
```
3*10^3 + 2*10^2 + 7*10^1 + 5*10^0 = 3000 + 200 + 70 + 5 = 3275
```

Второе свойство: 
Последовательное деление числа на основание до тех пор пока частным от деления не окажется 0 даст остатки от деления, составляющие данное число в *обратном порядке*.
Пример: Число 3275 в десятичной системе счисления
```
3275/10 -- 327 (остаток 5)
327/10  -- 32  (остаток 7)
32/10   -- 3   (остаток 2)
3/10    -- 0   (остаток 3)

Остатки в обратном порядке: 3275
```

Третье свойство:
Если основание одной системы счисления представляет собой натуральную степень *n* основания другой системы счисления, то *одна* *цифра* первой системы счисления соответствует *n* *цифрам* второй системы счисления
## Преобразование из 2- в 8- и 16- системы счисления
На третьем свойстве основан перевод из двоичной (bin) в восьмиричную (oct) и шестнадцатиричную (hex) системы счисления. В первос случае одна цифра будет транслироваться в 3, во втором в 4 цифры.
![[Pasted image 20241029222415.png]]
Шестнадцатиричная система неимоверно удобна для записи байтов, которые представляют собой 8 бит, таким образом байт можно записать двумя шестнадцатиричными цифрами.

Важно, что пари переводе числа разбивают на группы *справа налево*.

## Перевод дробной части
На примере преобразований из двоичной в десятичные системы счисления и обратно.
### Перевод из 2- в 10-
Числа после точки (то есть дробная часть числа), соответствуют дробям, знаменатель которых является основанием системы счисления  в степени, равной позиции после точки. 
Пример:
```
101.01101 (2)
= 
1*2^2 + 0*2^1 + 1*2^0 + 0*(1/2^1) + 1*(1/2^2) + 1*(1/2^3) + 0*(1/2^4) + 1*(1/2^5) (10)
=
4 + 1 + 1/4 + 1/8 + 1/32 (10)
```
### Перевод из 10- в 2-
Дробная часть числа постепенно умножается на основание системы счисления. Если после умножения целая часть равна 0, то продолжаем умножение (0 вписывается в получаемое число). Если после умножения получаем 1, то вписываем ее также, потом отбрасываем и продолжаем умножение.
Пример:
```
0.40625 (10)
0.40625 * 2 = 0.8125 -- 0
0.8125 * 2  = 1.625  -- 1
0.625 * 2   = 1.25   -- 1
0.25 * 2    = 0.5    -- 0
0.5 * 2     = 1.0    -- 1
= 
0.01101 (2)
```
В виде *конечной* дроби заданной системы счисления представимы такие и только такие рациональные числа, у которых в знаменателе *степень основания системы счисления*.
Рациональное число, предствавленное в виде несократимой дроби *m/n*, *представимо* в виде *конечной дроби* в системе по основанию N тогда и *только тогда*, когда существует целая степень числа N, делящаяся нацело на n
Пример в десятичной системе счисления:
```
49/50 -- 10^2/50 -- OK
77/80 -- 10^4/80 -- OK
```
### Периодические дроби
Перевод из десятичной в двоичную будет осуществляться тем же принципом, пока не получим "рабочее" повторяющееся число при умножении дробной части на 2. Тогда можно говорить о том что мы нашли *период*.
## Умножение в двоичной системе
В двоичной системе умножение выполняется крайне легко. Также столбиком. Однако стоит заметить, что умнолжение на 0 можно и не писать вовсе. А при умножении на 1 получим тоже самое число, однако записать его необходимо со смещением согласно позиции.
![[Pasted image 20241029225521.png]]
В дальнейшем позиционные значения складываются. Принцип как и в любой другой системе счисления.
