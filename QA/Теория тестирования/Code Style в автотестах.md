Ниже перечислены принципы, которых стоит придерживаться при разработке автоматизированных тестов:

1. Максимальная линейность кода тестов. Избегайте ветвлений и циклов. Если в тест хочется добавить if -- возможно такой тесть следует разбить на два.
2. Избегайте зависимых тестов, которые могут быть выполнены только в строго определенном порядке. Тесты должны быть независимыми друг от друга.
3. Следует стремиться к тому, чтобы тест не полагался на "контент", а готовил данные самостоятельно (и удалял по завершению теста). Так например следует использовать чистые браузеры и новых пользоваетелей.
4. Линейность проверок. В assert утверждениях не должно быть ветвлений и циклов. Только линейная логика проверок.
5. Следует придерживаться одинакового стиля при именовании тестов. Имя теста должно четко отражать суть содержимого.
6. Одинаковые тесты, отличающиеся одним компонентом, стоит параметризировать, а не копировать.
7. Стремиться писать максимально атомарные и неделимые тесты.
