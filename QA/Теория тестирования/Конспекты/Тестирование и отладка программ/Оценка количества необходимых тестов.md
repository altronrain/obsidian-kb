Сколько тестов необходимо, чтобы обнаружить ошибки в программе?

Утверждается, что вероятность успешности очередного теста зависит от процента оставшихся ошибок. Последние ошибки обнаружить сложнее.
Пусть:
* *Tn* -- среднее количество тестов, необходимых для обнаружения *n* ошибок.
* *N* -- число ошибок в программе.
Для оценки *Tn* используется формула:
				*Tn = a\*sum(k=0, n-1)1/(N-k)*
где a -- некий экспериментально определяемый коэффициент.

Напрямую посчитать *TN* (т.е. количество тестов для обнаружения всех ошибок) мы не можем из-за коэффициента. Однако мы можем выразить соотношение:
				*P = Tn / TN = sum(k=0, n-1)1/(N-k) / sum(k=0, N-1)1/(N-k)*
В такую формулу можно подставлять N и n.

Что стоит вынести, в реальных программах с множеством ошибок ~ в процентах получается:
* ~13-15% тестов для обнаружения 50% ошибок;
* ~23-30% тестов для обнаружения 70% ошибок; т.е. примерно удвоение от предыдущего
* ~44-49% тестов для обнаружения 90% ошибок.
* последние 10% ошибок потребуют удвоения тестовой базы.
