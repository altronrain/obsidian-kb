Можно рассматривать как еще один критерий белого ящика, который является неким компромиссом, призванным обеспечить как достаточную надежность тестирования, так и приемлемую сложность.

Минимально грубое тестирование (МГТ) -- критерий покрытия решений/условий, усиленный дополнительными требованиями по проверке циклов.

Правила, по которым организуется проверка циклов:
1. Для каждого цикла с предусловием должна быть проверена правильность при:
	* нулькратном выполнении
	* однократном выполнении
	* многократном выполнении (более 1 раза)
2. Для каждого цикла с постусловием должна быть проверена правильность при:
	* однократном выполнении
	* многократном выполнении (более 1 раза)
3. Проверка цикла со счетчиком зависит от того, фиксированы ли границы счетчика, или вычисляются динамически. В общем случае, требуется проверка при:
	* нулькратном выполнении
	* однократном выполнении
	* многократном выполнении (более 1 раза)

Преимуществом данного метода можно считать табличную запись, что позволяет удобно отслеживать степень выполнения критерия (покрытия) и прицельно придумывать тесты.
Строки таблицы соответствуют проверяемым условиям. Столбцы -- тестам.

* для ветвления (условного оператора `if`) создаются 2 строки: для веток "то" и "иначе".
* для каждого цикла с предусловием -- 3 строки по количеству повторений тела цикла (0, 1, many)
* для каждого цикла с постусловием -- 3 строки по количеству повторений тела цикла (1, many)
* для каждого цикла со счетчиком -- 3 строки по количеству повторений тела цикла (0, 1, many)
* для каждого оператора выбора -- X строк по количеству ветвей (включая "инначе")

Если какая-то из ветвей не имеет смысла для программы (*например:* цикл выполняется всегда хотя бы 1 раз), то строка все равно помещается в таблицу, но в строке дается пояснение, что такая ситуация невозможна.

**Важно!** Если в условном операторе, в цикле с пред- или постусловием стоит *сложное* условие, то к строкам, соответствующим самому условию добавляются **по 2 строки** на каждое простое условие.

В итоговой таблице, каждая строка соответствует одной ситуации, которую *необходимо* проверить.
